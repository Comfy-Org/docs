**[SeedVR2](https://iceclear.github.io/projects/seedvr2/)** is a one-step diffusion-based video restoration model from ByteDance Seed. It performs adversarial VR training against real data to restore and upscale images and videos with high fidelity. SeedVR2 is natively supported in ComfyUI (PR [#14424](https://github.com/Comfy-Org/ComfyUI/pull/14424)).

[Paper (arXiv)](https://arxiv.org/abs/2506.05301) | [ByteDance Seed Codebase](https://github.com/ByteDance-Seed/SeedVR) | [Comfy-Org Model Repository](https://huggingface.co/Comfy-Org/SeedVR2)

### Key strengths

- **Conservative upscaling** ‚Äî preserves original structure and detail while enhancing clarity, ideal for archival restoration
- **One-step inference** ‚Äî single forward pass for both image and video upscaling
- **Multiple model sizes** ‚Äî 3B and 7B variants with FP16, FP8, INT8, and NF4 quantizations
- **Both image and video** ‚Äî supports single-image upscaling and video resolution enhancement

> **Limitations:** SeedVR2 may not be robust to heavy degradations or very large motions. On inputs with very light degradations (e.g., 720p AIGC content), it can occasionally produce oversharpened results due to its strong generation ability.

## Model downloads

The SeedVR2 model files are available from the [Comfy-Org SeedVR2 repository](https://huggingface.co/Comfy-Org/SeedVR2). Both 3B and 7B variants are available in multiple quantization formats.

| File | Size | Notes |
|------|:----:|:------|
| `seedvr2_3b_fp16.safetensors` | 6.0 GB | 3B FP16 base model |
| `seedvr2_3b_fp8_e4m3fn.safetensors` | 3.0 GB | 3B FP8 quantized |
| `seedvr2_3b_int8_convrot.safetensors` | 5.6 GB | 3B INT8 with convrot |
| `seedvr2_3b_mxfp8.safetensors` | 3.0 GB | 3B MXFP8 quantized |
| `seedvr2_3b_nvfp4.safetensors` | 1.5 GB | 3B NVFP4 quantized |
| `seedvr2_7b_fp16.safetensors` | 13.5 GB | 7B FP16 base model |
| `seedvr2_7b_fp8_e4m3fn.safetensors` | 6.8 GB | 7B FP8 quantized |
| `seedvr2_7b_int8_convrot.safetensors` | 12.7 GB | 7B INT8 with convrot |
| `seedvr2_7b_mxfp8.safetensors` | 6.8 GB | 7B MXFP8 quantized |
| `seedvr2_7b_nvfp4.safetensors` | 3.4 GB | 7B NVFP4 quantized |
| `seedvr2_7b_sharp_fp16.safetensors` | 13.5 GB | 7B FP16 sharp variant |
| `seedvr2_ema_vae_fp16.safetensors` | 168 MB | VAE for SeedVR2 |

> The sharp variant (`_sharp`) is trained with additional sharpness enhancement. For most use cases, the EMA weights (without `_sharp`) provide better balance.

Place the files in the following directories:

```
ComfyUI/
+-- models/
    +-- diffusion_models/
        +-- seedvr2_<variant>.safetensors
    +-- vae/
        +-- seedvr2_ema_vae_fp16.safetensors
```

### Using image upscale

1. **Load an image** ‚Äî Place your image in the `ComfyUI/input/` folder and select it in the `Load Image` node
2. **Select the model** ‚Äî Choose your SeedVR2 model checkpoint in the SeedVR2 upscaler subgraph
3. **Run the workflow** ‚Äî Click `Queue` or use `Ctrl(Cmd) + Enter`

The output image is saved to the `ComfyUI/output/` folder with the prefix `ComfyUI-upscaled`.

### Tips

For better results, downscale the input image to approximately 0.35 megapixels using the `ImageScaleToTotalPixels` node before feeding it to the SeedVR2 upscaler. This is especially helpful for higher-resolution source images.

The workflow is organized into a few parts:

2. **SeedVR2 Video Upscaler (Subgraph)**: The main upscaling engine with configurable resolution and model selection
3. **Frame batch control**: `ImageFromBatch` clips the first 96 frames for faster testing (disable it with Ctrl-B for full video)
4. **Output**: Combines upscaled frames into a video with `CreateVideo` and `SaveVideo`

### Using video upscale

1. **Load a video** ‚Äî Place your video in the `ComfyUI/input/` folder and select it in the `Load Video` node
2. **Set the resolution** ‚Äî The `Resolution` parameter controls the output height. Standard values:
   - `1080` for 1920x1080 (1080p)
   - `1440` for 2560x1440 (1440p)
   - `2160` for 3840x2160 (4K)
3. **Select the model** ‚Äî Choose your SeedVR2 model checkpoint. The default is `seedvr2_ema_7b_fp16.safetensors`
4. **Adjust frame range** ‚Äî The `ImageFromBatch` node limits processing to 96 frames by default. Select it and press `Ctrl-B` to disable for full-length video processing
5. **Run the workflow** ‚Äî Click `Queue` or use `Ctrl(Cmd) + Enter`

### Resolution guide

The Resolution setting in the SeedVR2 upscaler is based on the output height. The workflow automatically adjusts the width to maintain the source aspect ratio:

| Resolution Setting | Example Output (16:9) |
|:------------------:|:---------------------:|
| 1080 | 1920x1080 (1080p) |
| 1440 | 2560x1440 (1440p) |
| 2160 | 3840x2160 (4K) |

If your source is 1280x720 and you set Resolution to 1080, the output will be 1920x1080. Use the `Image Scale` node before the upscaler to downscale your input to the correct starting size ‚Äî the upscaler center-crops the input to match the target aspect ratio.

### Performance

Higher target resolutions and larger models (7B) require significantly more processing time. Start at lower resolutions and use the frame batch limit when iterating on settings before committing to a full render.

| Model | Resolution | Approx. Time (10s 720p clip) |
|:----:|:----------:|:---------------------------:|
| SeedVR2 (local, 5090) | 1080p | ~312s |

## Additional Notes

- Make sure your ComfyUI is up to date. SeedVR2 support is available in recent versions.
- The VAE checkpoint `seedvr2_ema_vae_fp16.safetensors` is shared across all model variants.
- For image upscaling, best results are achieved when the input is around 0.35 megapixels before upscaling.
