# ComfyUI SeedVR2 Introduction

[SeedVR2](https://iceclear.github.io/projects/seedvr2/) (arXiv:2506.05301, from ByteDance Seed) is a one-step diffusion-based video restoration model. It performs adversarial training against real data to restore and upscale images and videos with high fidelity. SeedVR2 is natively supported in ComfyUI (PR [#14424](https://github.com/Comfy-Org/ComfyUI/pull/14424)).

Key capabilities:

- **Conservative upscaling**: preserves original structure and detail while enhancing clarity
- **One-step inference**: single forward pass for both image and video upscaling
- **Multiple model sizes**: 3B and 7B variants with FP16, FP8, INT8, NF4, and MXFP4 quantizations
- **Both image and video**: supports single-image upscaling and video resolution enhancement

## Model Installation

Download the SeedVR2 checkpoint(s) from the [Comfy-Org SeedVR2 repository](https://huggingface.co/Comfy-Org/SeedVR2) and save them to the corresponding ComfyUI folder:

- **seedvr2_3b_fp16.safetensors**: 3B FP16 base model, lower VRAM requirement
- **seedvr2_3b_fp8_e4m3fn.safetensors**: 3B FP8 quantized, further reduced VRAM
- **seedvr2_7b_fp16.safetensors (recommended)**: 7B FP16 base model, higher quality
- **seedvr2_7b_fp8_e4m3fn.safetensors**: 7B FP8 quantized, balance of quality and VRAM
- **seedvr2_7b_sharp_fp16.safetensors**: 7B FP16 sharp variant for enhanced detail
- **seedvr2_ema_vae_fp16.safetensors**: VAE checkpoint (shared across all model variants)

Place the files in the following directories:

```
ComfyUI/
├── models/
│   ├── diffusion_models/
│   │   ├── seedvr2_3b_fp16.safetensors
│   │   ├── seedvr2_3b_fp8_e4m3fn.safetensors
│   │   ├── seedvr2_7b_fp16.safetensors
│   │   ├── seedvr2_7b_fp8_e4m3fn.safetensors
│   │   ├── seedvr2_7b_sharp_fp16.safetensors
│   └── vae/
│       └── seedvr2_ema_vae_fp16.safetensors
```

## 1. Image Upscale

**What it does:** Takes a single image and upscales it using the SeedVR2 model. The workflow includes a before/after comparison for easy visual verification of the upscaling result.

### 1.1 Steps to Run

1. Place your image in the `ComfyUI/input/` folder and select it in the `Load Image` node
2. Select your SeedVR2 model checkpoint in the SeedVR2 upscaler subgraph
3. Click `Queue` or use `Ctrl(cmd) + Enter` to run

The output image is saved to the `ComfyUI/output/` folder with the prefix `ComfyUI-upscaled`.

### Tips

For better results, downscale the input image to approximately 0.35 megapixels using the `ImageScaleToTotalPixels` node before feeding it to the SeedVR2 upscaler. This is especially helpful for higher-resolution source images.

## 2. Video Upscale

**What it does:** Upscales a video using the SeedVR2 model. The workflow supports configurable output resolution and model selection.

<Card title="Download Workflow" icon="download" href="https://github.com/Comfy-Org/workflow_templates/blob/main/templates/utility_seedvr2_video_upscale.json">
    Download JSON or search "SeedVR2 Video Upscale" in Template Library
  </Card>

<Card title="Download Sample Video" icon="download" href="https://raw.githubusercontent.com/Comfy-Org/workflow_templates/main/input/input_3d.mp4">
    Get the example input video for this workflow
  </Card>

### 2.1 Steps to Run

1. Place your video in the `ComfyUI/input/` folder and select it in the `Load Video` node
2. Set the **Resolution** parameter (based on output height): `1080` for 1080p, `1440` for 1440p, `2160` for 4K
3. Select your SeedVR2 model checkpoint. The default is `seedvr2_ema_7b_fp16.safetensors`
4. The `ImageFromBatch` node limits processing to 96 frames by default. Select it and press `Ctrl-B` to disable for full-length video processing
5. Click `Queue` or use `Ctrl(cmd) + Enter` to run

### How Resolution Works

The Resolution setting is based on the output height. The workflow automatically adjusts the width to maintain the source aspect ratio:

| Resolution Setting | Example Output (16:9) |
|:------------------:|:---------------------:|
| 1080 | 1920x1080 (1080p) |
| 1440 | 2560x1440 (1440p) |
| 2160 | 3840x2160 (4K) |

If your source is 1280x720 and you set Resolution to 1080, the output will be 1920x1080. Use the `Image Scale` node before the upscaler to downscale your input to the correct starting size. The upscaler center-crops the input to match the target aspect ratio.

### Performance

Higher target resolutions and larger models (7B) require significantly more processing time. Start at lower resolutions and use the frame batch limit when iterating on settings before committing to a full render.
