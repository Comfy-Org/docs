HiDream-O1-Image is a unified image generative foundation model open-sourced by HiDream-ai on May 8, 2026. Built on a Pixel-level Unified Transformer (UiT), it operates without external VAEs or disjoint text encoders, supporting **text-to-image**, **instruction-based image editing**, **subject-driven personalization**, and **storyboard generation** at up to 2,048 × 2,048 resolution.

HiDream-O1-Image is released under the [MIT License](https://github.com/HiDream-ai/HiDream-O1-Image?tab=MIT-1-ov-file), and is natively supported in ComfyUI (PR [#13817](https://github.com/Comfy-Org/ComfyUI/pull/13817)).

### 2. Manually Download Models

**Checkpoint** — Choose one based on your VRAM:

| File | VRAM Requirement |
|------|-----------------|
| [hidream_o1_image_fp8_scaled.safetensors](https://huggingface.co/Comfy-Org/HiDream-O1-Image/resolve/main/checkpoints/hidream_o1_image_fp8_scaled.safetensors) | ~12 GB+ |

**Text Encoder** — One file shared across all versions:

- [gemma4_e4b_it_fp8_scaled.safetensors](https://huggingface.co/Comfy-Org/gemma-4/resolve/main/text_encoders/gemma4_e4b_it_fp8_scaled.safetensors)

```
📂 ComfyUI/
├── 📂 models/
│   ├── 📂 checkpoints/
│   │   └── hidream_o1_image_fp8_scaled.safetensors
│   └── 📂 text_encoders/
│       └── gemma4_e4b_it_fp8_scaled.safetensors
```

### 3. Follow the Steps

1. In the `CheckpointLoaderSimple` node, ensure it loads the `hidream_o1_image_fp8_scaled.safetensors` (or the variant you downloaded)
2. In the `CLIPTextEncode` node, enter your text prompt in the positive text field
3. For **text-to-image**: leave the `Load Image` node disabled (Ctrl+B to bypass)
4. For **instruction editing**: enable the `Load Image` node and connect it to `HiDreamO1ReferenceImages`
5. In the **SamplerCustom** settings:
   - Set `steps` to **50**
   - Set `cfg` to **5.0**
   - Set `sampler` to `lcm`
   - Set `scheduler` to `normal`
6. In the `EmptyHiDreamO1LatentImage` node, you can adjust the output resolution. The model was trained at the following resolutions:

| Width | Height | Category |
|-------|--------|----------|
| 2048 | 2048 | Square |
| 2304 | 1728 | Landscape |
| 2304 | 1792 | Landscape |
| 2496 | 1664 | Landscape |
| 2560 | 1440 | Landscape |
| 3104 | 1312 | Landscape |
| 1728 | 2304 | Portrait |
| 1792 | 2304 | Portrait |
| 1664 | 2496 | Portrait |
| 1440 | 2560 | Portrait |
| 1312 | 3104 | Portrait |

7. Click `Run` or use `Ctrl(cmd) + Enter` to generate

> **Note:** HiDream-O1-Image outputs pixel-space latents directly — no `VAEDecode` node needed. Connect the output of SamplerCustom to a `PreviewImage` or `SaveImage` node.

## Using Reference Images

The `HiDreamO1ReferenceImages` node allows you to attach up to **10 reference images** to the conditioning for:

- **Single image + edit instruction:** Load one reference image and write an editing prompt
- **Multiple images (2-10):** Subject-driven personalization — the model learns the subject identity and generates new scenes

![Subject-Driven Personalization](https://raw.githubusercontent.com/HiDream-ai/HiDream-O1-Image/main/assets/IP_2.jpg)

## Additional Tips

### Long Text Rendering
HiDream-O1-Image excels at rendering text within images. Write clear instructions specifying text content, font style, color, and positioning for best results.

![Long-Text Rendering & Layout Control](https://raw.githubusercontent.com/HiDream-ai/HiDream-O1-Image/main/assets/text-layout.webp)

### Image Scaling
If your input reference image is too large, enable the `ImageScaleToTotalPixels` node (Ctrl+B to unbypass) to scale it down to 1 megapixel total pixels.

### Patch Seam Smoothing (Experimental)
The `HiDreamO1PatchSeamSmoothing` node can reduce visible patch seams during sampling. It's disabled by default and only recommended if you notice seam artifacts.
