HiDream-O1-Image is a unified image generative foundation model open-sourced by HiDream-ai on May 8, 2026. Built on a Pixel-level Unified Transformer (UiT), it operates without external VAEs or disjoint text encoders — encoding raw pixels, text, and task-specific conditions in a single shared token space.

The model is released under the [MIT License](https://github.com/HiDream-ai/HiDream-O1-Image?tab=MIT-1-ov-file), supporting personal projects, scientific research, and commercial use. It supports **text-to-image**, **instruction-based image editing**, **subject-driven personalization**, and **storyboard generation** at up to 2,048 × 2,048 resolution.

## Key Features

**Pixel-Level Unified Transformer** — One end-to-end model on raw pixels. No VAE encoding/decoding, no disjoint text encoders. Everything operates in the same unified token space.

**Multi-Task in One Architecture** — Text-to-image, long-text rendering, instruction editing, subject-driven personalization, and storyboard generation — all in a single 8B-parameter model.

**Reasoning-Driven Prompt Agent** — Built-in "thinking" agent (based on Gemma-4-31B-it) that resolves implicit knowledge, layout, and text rendering before generation, significantly improving complex prompt fidelity.

**Native High Resolution** — Direct synthesis at up to 2,048 × 2,048 with sharp, fine-grained detail.

**Exceptional Efficiency at 8B Scale** — With only 8B parameters, achieves performance parity with or surpasses much larger models on multiple benchmarks.

## ComfyUI Native Support

HiDream-O1-Image is natively supported in ComfyUI (PR [#13817](https://github.com/Comfy-Org/ComfyUI/pull/13817)). The built-in nodes are located in `comfy_extras/nodes_hidream_o1.py` and the model architecture is defined in `comfy/ldm/hidream_o1/`.

<Tip>
Before starting, make sure your ComfyUI is updated to include this PR (built-in from v0.21.0+ / nightly builds after May 12, 2026).
</Tip>

### How HiDream-O1-Image Differs from HiDream-I1

Unlike HiDream-I1, the HiDream-O1-Image model:

- **Does not require a VAE** — it operates directly in pixel space with 3-channel latents
- **Does not use separate text encoders** — the model uses a single bundled checkpoint file with an integrated Gemma-4 text encoder
- **Uses the `CheckpointLoaderSimple` node** instead of separate `Load Diffusion Model` + `QuadrupleCLIPLoader` + `Load VAE` nodes
- **Produces pixels directly** — the output from the KSampler/SamplerCustom can be previewed and saved without a `VAEDecode` node
- **Supports reference image conditioning** for editing and personalization via the `HiDreamO1ReferenceImages` node
- **Supports patch seam smoothing** via the experimental `HiDreamO1PatchSeamSmoothing` node

## Model Installation

HiDream-O1-Image uses a **checkpoint-style** model bundled with its own text encoder. Unlike diffusion-model+VAE architectures, you only need one checkpoint file and one text encoder file.

**Checkpoint models** — download one of the following depending on your VRAM:

### 1. Model Setup

Download the checkpoint model and text encoder as described in [Model Installation](#model-installation) above. For the Full version:

1. Download `hidream_o1_image_fp8_scaled.safetensors` (or mxfp8/bf16 equivalent) to `ComfyUI/models/checkpoints/`
2. Download `gemma4_e4b_it_fp8_scaled.safetensors` to `ComfyUI/models/text_encoders/`

### 2. Load the Workflow

Download the image below and drag it into ComfyUI to load the corresponding workflow:

![HiDream-O1-Image Full Workflow](https://raw.githubusercontent.com/Comfy-Org/workflow_templates/main/templates/image_hidream_o1-1.webp)

The workflow will prompt you to download any missing models automatically.

### 3. Complete the Workflow Step by Step

Follow these steps to run the workflow:

1. In the `CheckpointLoaderSimple` node, make sure it loads the `hidream_o1_image_fp8_scaled.safetensors` (or the version you downloaded)
2. The `CLIPTextEncode` node — enter your **text prompt** in the positive text field
3. For **basic text-to-image**: leave the `Load Image` node empty or disable it (Ctrl+B to bypass)
4. For **instruction editing**: load an image into the `Load Image` node, and connect it to the `HiDreamO1ReferenceImages` node
5. In the **KSampler** settings:
   - Set `steps` to **50**
   - Set `cfg` to **5.0**
   - Set `sampler` to `lcm` (recommended)
   - Set `scheduler` to `normal`
6. The `EmptyHiDreamO1LatentImage` node generates pixel-space latents at the target resolution. You can use the default 2048×2048 or choose one of the [trained resolutions](#trained-resolutions)
7. Click the `Run` button, or use the shortcut `Ctrl(cmd) + Enter` to generate

> **Note:** The output from the KSampler/SamplerCustom is already in pixel space — no `VAEDecode` node is needed. Connect the latent output directly to a `PreviewImage` or `SaveImage` node.

### 3. Complete the Workflow Step by Step

1. In the `CheckpointLoaderSimple` node, load `hidream_o1_image_dev_fp8_scaled.safetensors`
2. Enter your **text prompt** in the `CLIPTextEncode` positive text field
3. For **text-to-image**: leave the `Load Image` node disabled
4. For **editing / personalization**: enable the `Load Image` node and connect it to `HiDreamO1ReferenceImages`
5. In the **KSampler** settings:
   - Set `steps` to **28**
   - (Important) Set `cfg` to **1.0** — negative prompts are not needed for the Dev version
   - Set `sampler` to `lcm`
   - Set `scheduler` to `normal`
6. Click `Run` or `Ctrl(cmd) + Enter` to generate

> **Important for Dev version:** Set `cfg` to `1.0` and leave the negative CLIP text encoder empty — the distilled model does not use negative conditioning.

## Using Reference Images

The `HiDreamO1ReferenceImages` node enables attaching up to **10 reference images** to the conditioning, powering both editing and personalization:

- **Single image + edit instruction:** Load one reference image and write an editing prompt (e.g., "remove the earphones", "change the background to a beach")
- **Multiple images (2-10):** Subject-driven personalization — the model learns the subject identity from the reference images and generates the subject in new scenes, poses, or contexts

![Subject-Driven Personalization](https://raw.githubusercontent.com/HiDream-ai/HiDream-O1-Image/main/assets/IP_2.jpg)

### Reference Image Workflow Steps

1. Add one or more `Load Image` nodes for your reference images
2. Connect them to the `HiDreamO1ReferenceImages` node's image inputs (the node auto-grows inputs as needed)
3. Connect the `positive` and `negative` outputs from `HiDreamO1ReferenceImages` to the sampler
4. For the **positive** conditioning write your desired prompt
5. For **single image editing**, connect the reference images to both positive and negative paths
6. Click `Run`

## Additional Tips

### Image Scaling
If your input reference image is too large (exceeding the ~4MP training resolution), enable the `ImageScaleToTotalPixels` node (Ctrl+B to unbypass) to scale it down to 1 megapixel total pixels.

### Patch Seam Smoothing (Experimental)
For advanced users, the `HiDreamO1PatchSeamSmoothing` node can be used to reduce visible patch seams during sampling. It averages the model output across multiple shifted patch-grid positions during the late portion of sampling. This is an experimental node — it's disabled by default and only recommended if you notice seam artifacts.

### Long Text Rendering
HiDream-O1-Image excels at rendering text within images. For best results with multi-line or multi-region text, write clear instructions in your prompt specifying the text content, font style, color, and positioning.
