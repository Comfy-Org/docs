**Stable Audio 3** ([GitHub](https://github.com/Stability-AI/stable-audio-3)) is Stability AI's latest open-source audio generation model, capable of producing high-quality stereo audio — including music, sound effects, and instruments — from text descriptions. It is natively supported in ComfyUI as of version 0.22.0.

**Key features**:

- **Text-to-audio generation** — Turn short descriptions into stereo audio clips
- **Category-aware reprompting** — Optional Qwen-powered prompt expansion using presets: Music, Instrument, SFX, One-shot
- **Local inference** — Runs entirely on your machine with a single checkpoint
- **Stereo output** — Generates full stereo audio, not mono
- **Open-source** — Released for local use and customization

**Related links**:

The **Stable Audio 3 Medium** workflow is a full-featured text-to-audio generation pipeline. You provide a short text idea, optional duration, seed, and category — the workflow expands your prompt using Qwen with a **category-aware reprompt template**, then generates stereo audio via the Stable Audio 3 checkpoint.

**How to use**:
1. **Text idea** — Enter a short description of the sound, music, or effect you want (e.g. "upbeat electronic dance track with heavy bass")
2. **Duration** — Set the desired clip length in seconds (default varies)
3. **Seed** — Control reproducibility by adjusting the seed value
4. **Category** — Choose a reprompt preset: **Music**, **Instrument**, **SFX**, or **One-shot**
5. **Enable reprompt** — Toggle `use_reprompt` on to let Qwen expand your short idea into a detailed prompt before generation
6. Click **Run** (`Ctrl/Cmd + Enter`) to generate. The audio will be saved to `ComfyUI/output/audio/`

The **Stable Audio 3 Medium Base** workflow is a simplified version without Qwen reprompt expansion. It expects a complete text prompt and passes it directly to the Stable Audio 3 model. Use this when you already have a detailed prompt and want faster generation without the LLM expansion step.

**How to use**:
1. **Text prompt** — Enter a detailed description of the audio you want (the model handles the prompt directly, so provide enough detail)
2. **Duration** — Set the clip length in seconds
3. **Seed** — Control reproducibility
4. Click **Run** (`Ctrl/Cmd + Enter`) to generate

## Model download for local users

Stable Audio 3 requires the following files. ComfyUI will attempt to download them automatically when you load the workflow. You can also download them manually and place them in the correct folders.

**Checkpoints** (place in `ComfyUI/models/checkpoints/`):
- [`stable_audio_3_medium.safetensors`](https://huggingface.co/Comfy-Org/stable-audio-3/resolve/main/checkpoints/stable_audio_3_medium.safetensors)

**Text encoders** (place in `ComfyUI/models/text_encoders/`):
- [`t5gemma_b_b_ul2.safetensors`](https://huggingface.co/Comfy-Org/stable-audio-3/resolve/main/text_encoders/t5gemma_b_b_ul2.safetensors)
- [`qwen3.5_2b_bf16.safetensors`](https://huggingface.co/Comfy-Org/Qwen3.5/resolve/main/text_encoders/qwen3.5_2b_bf16.safetensors) (only needed for the Medium workflow with Qwen reprompt)

After placing the files, restart ComfyUI and load the workflow. The checkpoint will be available in the **subgraph's internal loader**.

> **Note**: The Medium Base workflow uses the `stable_audio_3_medium_base.safetensors` checkpoint variant. Make sure you have the correct checkpoint for the workflow you are using.
