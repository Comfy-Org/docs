Vidu Q1 is the foundational generation tier of Vidu AI video generation. It offers five different node types for various video generation scenarios: text-to-video, image-to-video, reference-based video, start-end frame animation, and video extension.

## Vidu Q1 Workflows

All Vidu Q1 nodes use the model parameter `viduq1` and support 720p and 1080p resolutions with up to 5 seconds of video output.

## Text to Video

The **ViduTextToVideoNode** generates video directly from a text prompt. Connect it to a SaveVideo node, write your prompt describing the scene, action, and cinematography, and run the workflow.

**Parameters:**
- **model**: Set to `viduq1` for Q1 generation
- **prompt**: Describe the video content in detail
- **duration**: Video length in seconds (up to 5)
- **seed**: Controls randomness (use "randomize" for varied results)
- **aspect_ratio**: Choose from `16:9`, `9:16`, `1:1`, etc.
- **resolution**: `720p` or `1080p`
- **style**: Optional style preset

## Image to Video

The **ViduImageToVideoNode** animates a still image into a video. Connect a LoadImage node to the `image` input, then provide a text prompt describing the motion and scene dynamics.

**Parameters:**
- Same as Text to Video, plus an **image** input slot
- The input image serves as the first frame of the generated video

## Reference to Video

The **ViduReferenceVideoNode** generates videos using multiple reference images. You can connect up to 7 reference images to maintain consistent characters, scenes, or elements across the generated video. Images are batched using ImageBatch nodes before connecting to the reference node.

**Key features:**
- Use multiple reference images for character consistency (e.g., different angles of a character)
- Reference scene elements, environments, and objects
- The prompt can refer to elements from the reference images for better cohesion

## Start-End to Video

The **ViduStartEndToVideoNode** creates smooth video transitions between a start frame and an end frame. Connect two LoadImage nodes — one for `first_frame` and one for `end_frame` — and the node generates a video that interpolates between them.

**Use cases:**
- Character animation (e.g., opening eyes, changing expressions)
- Scene transitions
- Object morphing or transformation sequences

## Video Extension

The **ViduExtendVideoNode** extends an existing video forward by generating new content. It takes a video input and an optional `end_frame` image to guide the extension. The node uses the `viduq2-pro` model internally for the extension.

**Workflow setup:**
1. Load a video with LoadVideo node
2. (Optional) Extract the last frame using GetVideoComponents to use as the end_frame guide
3. Connect to ViduExtendVideoNode
4. The extended video is output through SaveVideo

> **Note**: The extended video will not retain the original audio.
