Vidu Q2 is the second-generation fast video generation tier. It focuses on speed and efficiency while maintaining high quality. Q2 introduces a new node family (`Vidu2*`) with optimized models for faster generation.

## Vidu Q2 Workflows

Q2 nodes support a range of models: `viduq2`, `viduq2-pro`, `viduq2-pro-fast`, and more. They output at 720p resolution by default with up to 5 seconds of video. Some Q2 nodes support additional features like audio generation and subject-labeled reference images.

## Text to Video (Q2)

The **Vidu2TextToVideoNode** generates video from text prompts with optimized speed. It's a straightforward node with no image input — just prompt, model, seed, duration, and resolution parameters.

**Parameters:**
- **model**: Select from `viduq2`, `viduq2-pro`, `viduq2-pro-fast`, etc.
- **prompt**: Detailed text description of the desired video
- **duration**: Video length in seconds (up to 5)
- **seed**: Random seed for reproducibility
- **aspect_ratio**: `16:9`, `9:16`, or other supported ratios
- **resolution**: Defaults to `720p`

## Image to Video (Q2)

The **Vidu2ImageToVideoNode** animates a still image into video with fast generation. Connect a LoadImage node to the `image` input.

**Parameters:**
- Same as Q2 Text to Video, plus an **image** input
- The prompt should describe the motion and actions to animate from the input image

## First-Last-Frame to Video (Q2)

The **Vidu2StartEndToVideoNode** creates video transitions between a first frame and an end frame. Connect two LoadImage nodes for the start and end frames.

**Parameters:**
- **first_frame** input: The starting image
- **end_frame** input: The target ending image
- Uses same model/prompt/seed parameters as other Q2 nodes
- Supports an **advanced** section with additional settings

## Reference to Video (Q2)

The **Vidu2ReferenceVideoNode** extends Q1's reference capabilities with **named subject slots**. Instead of batching images, this node accepts individual image inputs with explicit subject labels (`subject1`, `subject2`, `subject3`), allowing for precise character and element references.

**Key features:**
- Up to 3 named subject slots (subject1, subject2, subject3)
- Use `@a`, `@b` notation in prompts to reference specific subjects
- Optional audio generation toggle for adding sound to the output

**Example prompt syntax with subject references:**
```
@a(red-haired woman in elegant attire) sits on a fallen log...
@b(aged robot with weathered white-grey body) stands before her...
```
