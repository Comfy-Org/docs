The Seedance 2.0 First-Last-Frame-to-Video node (`ByteDance2FirstLastFrameNode`) generates the full video between a starting image and an ending image. You provide the first and last frames, and Seedance 2.0 creates the motion and transitions that connect them.

<ReqHint/>
<UpdateReminder/>

## Workflow overview

The First-Last-Frame workflow uses four nodes:

1. **LoadImage (x2)** — Load the first frame and last frame images
2. **ByteDance2FirstLastFrameNode** — Takes the two images plus a text prompt, outputs a VIDEO
3. **SaveVideo** — Saves the generated video to disk

## Node parameters

| Parameter | Description |
| :-------- | :---------- |
| `model` | Model name. Default: `Seedance 2.0` |
| `prompt` | Text description of the video. Use to describe the transition, camera movement, and scene |
| `resolution` | Output resolution. Options: `480p`, `720p`, `1080p` |
| `aspect_ratio` | Aspect ratio. Options: `16:9`, `9:16`, `1:1`, `4:5` |
| `duration` | Video duration in seconds |
| `generate_audio` | Whether to generate audio synchronized with the video |
| `seed` | Random seed for reproducibility |
| `cfg` | Classifier-free guidance scale |

## Inputs

| Input | Type | Description |
| :---- | :--- | :---------- |
| `first_frame` | IMAGE | The starting image (direct image input) |
| `last_frame` | IMAGE | The ending image (direct image input) |
| `first_frame_asset_id` | STRING | Asset ID for a verified real-human first frame |
| `last_frame_asset_id` | STRING | Asset ID for a verified real-human last frame |

## Running the workflow

1. **Load images** — Use LoadImage nodes to load your first frame and last frame images
2. **Write a prompt** — Describe the transition and overall scene. The prompt helps guide motion, camera work, and style
3. **Set parameters** — Choose resolution, aspect ratio, duration, and audio settings
4. **Run the workflow** — Click the Queue button or use `Ctrl(Cmd) + Enter`
5. **Review the output** — The generated video appears in the SaveVideo node

## Tips

- **Choose distinct frames** — The more contrast between first and last frames, the more interesting the generated transition
- **Prompt as a guide** — The prompt describes the transition, not just the scene. Include camera movement direction (e.g., "drone flies forward, then rotates 180 degrees")
- **Asset IDs for real humans** — When using verified real-human portraits, connect `asset_id` strings instead of direct image inputs
