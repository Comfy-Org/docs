The **ElevenLabsSpeechToText** node transcribes audio into text using ElevenLabs Scribe models. It outputs the transcribed text, detected language code, and word-level JSON data with timing information.

<ReqHint/>
<UpdateReminder/>

## Supported models

| Model | Description |
| :---- | :---------- |
| `scribe_v2` | High-accuracy speech-to-text transcription |

### Speech-to-Text outputs

The node produces three outputs:

| Output | Type | Description |
| :----- | :--- | :---------- |
| **text** | STRING | Full transcribed text |
| **language_code** | STRING | Detected language code |
| **words_json** | STRING | Word-level transcription with timing (JSON) |

### Speech-to-Text parameters

| Parameter | Description |
| :-------- | :---------- |
| **model_id** | Transcription model selection (`scribe_v2`) |
| **tag_audio_events** | Tag audio events like laughter, applause |
| **num_speakers** | Number of speakers for diarization (0 = auto) |
| **timestamps_granularity** | Timestamp granularity: none, word, character |
| **diarize** | Enable/disable speaker diarization |
| **output_format** | Output format for result text |

If you use the template as a reference, a typical run looks like this:

1. Load an audio file through **LoadAudio** (e.g. `voice_demo.mp3`)
2. (Optional) Use **RecordAudio** to capture audio directly
3. Connect the audio to **ElevenLabsSpeechToText**
4. Select the model (`scribe_v2`)
5. Configure optional parameters:
   - Enable **diarize** for multi-speaker transcription
   - Set **timestamps_granularity** to "word" for word-level timing
   - Set **num_speakers** to specify speaker count (0 for auto-detect)
6. Click **Run** or press `Ctrl(cmd) + Enter`
7. View the transcribed text in the **PreviewAny** node
8. Inspect word-level JSON in the **words_json** PreviewAny node

### Additional notes

- The **RecordAudio** node is bypassed by default. Use `Ctrl-B` to enable it for live recording
- **words_json** output contains start/end timestamps for each word, useful for subtitle generation or alignment
- **language_code** is auto-detected; you can use it for downstream multilingual processing
- For pricing, see [ElevenLabs on the pricing page](/tutorials/partner-nodes/pricing#elevenlabs)
