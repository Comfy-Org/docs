The **ElevenLabsTextToDialogue** node generates multi-speaker dialogue audio with distinct voices assigned to each participant. Each speaker gets their own text input and voice selection.

<ReqHint/>
<UpdateReminder/>

## Supported models

| Model | Description |
| :---- | :---------- |
| `eleven_v3` | English-optimized high-quality dialogue generation |

### Text-to-Dialogue parameters

| Parameter | Description |
| :-------- | :---------- |
| **similarity** | Voice similarity (0.0–1.0), default 0.5 |
| **stability** | Voice stability (0.0–1.0), default auto |
| **model** | Voice model selection |
| **turn_count** | Number of dialogue turns (speaker switches) |
| **text_1 / text_2** | Text content for each speaker |
| **output_format** | Audio output format (e.g. mp3_44100_192) |
| **seed** | Random seed for reproducible output |
| **seed_mode** | Seed mode: randomize / fixed |

The workflow has two options:

1. **Use Preset Voice** — select distinct voices for each speaker from two **ElevenLabsVoiceSelector** nodes (e.g. "Roger (male, american)" and "Laura (female, american)")
2. **Create a Voice Clone** — load an audio sample and enable **ElevenLabsInstantVoiceClone** for custom voices

If you use the template as a reference, a typical run looks like this:

1. Select voices for each speaker using **ElevenLabsVoiceSelector** nodes
2. (Optional) Enable **ElevenLabsInstantVoiceClone** for cloned voices
3. Enter dialogue text for each speaker in the **ElevenLabsTextToDialogue** node
4. Set **turn_count** to control how many times speakers alternate
5. Click **Run** or press `Ctrl(cmd) + Enter`
6. The generated dialogue audio is saved via **SaveAudioMP3**

### Additional notes

- The voice clone section is bypassed by default. Use `Ctrl-B` to enable it
- For multi-turn dialogues, the node alternates between the two voices automatically
- Each speaker's text field supports separate content for natural conversation flow
- For pricing, see [ElevenLabs on the pricing page](/tutorials/partner-nodes/pricing#elevenlabs)
