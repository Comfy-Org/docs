# ComfyUI MoGe Introduction

[MoGe](https://github.com/microsoft/MoGe) (CVPR 2025, from Microsoft Research) is a monocular geometry model that can estimate dense 3D geometry from a single image. It supports:

- **Metric-scale point maps and depth maps** (MoGe-2)
- **Surface normal prediction** (MoGe-2)
- **Object mask estimation**
- **Mesh generation** from both perspective and panoramic images

MoGe comes in two checkpoints:

| File | Version | Use Case |
|------|---------|----------|
| `moge_1_vitl_fp16` | **MoGe-1** | Point map + depth + mask; relative scale. Lighter baseline. |
| `moge_2_vitl_normal_fp16` | **MoGe-2** | **Recommended.** Metric-scale depth/points, sharper detail, includes normal maps for better mesh edges. |

<Tip>
ComfyUI now natively supports MoGe nodes. Make sure you have updated to the latest version of [ComfyUI](https://github.com/comfyanonymous/ComfyUI) before starting.

The workflow example PNG images in this tutorial contain workflow JSON in their metadata:
- You can drag them directly into ComfyUI
- Or use the menu `Workflows` -> `Open (ctrl+o)`

This will load the corresponding workflow and prompt you to download the required models. Generated `.glb` format models will be output to the `ComfyUI/output/mesh` folder.
</Tip>

## Example Workflows

This section covers three workflows:

1. **Panorama to Mesh** — Convert equirectangular (360°) panorama photos to textured GLB
2. **Perspective to Mesh** — Convert a single perspective photo to textured GLB
3. **Depth Estimation** — Estimate depth from a single image

---

## 1. Panorama to Mesh

Convert an equirectangular (360°) panorama into a textured 3D GLB model. This workflow uses `MoGePanoramaInference` which splits the panorama into 12 perspective views, runs MoGe inference on each, then merges them into a complete mesh.

### 1.2 Manual Model Installation

Download the MoGe checkpoint and save it to the corresponding ComfyUI folder:

- **MoGe-2 (recommended)**: [moge_2_vitl_normal_fp16.safetensors](https://huggingface.co/Comfy-Org/MoGe/resolve/main/moge_2_vitl_normal_fp16.safetensors)
- **MoGe-1 (baseline)**: [moge_1_vitl_fp16.safetensors](https://huggingface.co/Comfy-Org/MoGe/resolve/main/moge_1_vitl_fp16.safetensors)

```
ComfyUI/
├── models/
│   ├── geometry_estimation/
│   │   ├── moge_2_vitl_normal_fp16.safetensors
│   │   └── moge_1_vitl_fp16.safetensors
```

### 1.3 Steps to Run

1. Ensure the `LoadMoGeModel` node has loaded one of the MoGe checkpoints
2. Load an equirectangular panorama image in the `Load Image` node
3. Click the `Queue` button, or use the shortcut `Ctrl(cmd) + Enter` to run the workflow

### 1.4 Controls

| Control | Meaning |
|---------|---------|
| `source_panorama` | Input equirectangular image |
| `inference_resolution` | MoGe detail level (0-9); higher is slower but retains more detail |
| `inference_batch_size` | Batch size for `MoGeInference`; use `1` for a single still image |
| `auto_resize_input` | Automatically downsamples the input if width > 2048 |

### 2.2 Steps to Run

1. Ensure the `LoadMoGeModel` node has loaded a MoGe checkpoint
2. Load a perspective photo in the `Load Image` node
3. (Optional) View the OpenGL and DirectX normal previews
4. Click `Queue` or use `Ctrl(cmd) + Enter` to run

### 2.3 Controls

| Control | Meaning |
|---------|---------|
| `source_image` | Input RGB image |
| `inference_resolution` | MoGe detail level (0-9); higher is slower but retains more detail |
| `inference_batch_size` | Batch size for `MoGeInference`; use `1` for a single still image |
| `mesh_frame_index` | Frame index used when converting geometry to mesh; use `0` for one image |
| `mesh_decimation` | Mesh decimation step; `1` keeps the densest mesh |
| `mesh_gap_threshold` | Removes faces across large depth discontinuities |

2. Load an image in the `Load Image` node
3. Click `Queue` or use `Ctrl(cmd) + Enter` to run
4. The workflow outputs colored depth preview, raw depth preview, and a mask

| Control | Meaning |

| `moge_model` | Model checkpoint loaded by `LoadMoGeModel` |
| `auto_resize_input` | Enables pre-inference resize for large images |
