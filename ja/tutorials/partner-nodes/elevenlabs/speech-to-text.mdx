**ElevenLabsSpeechToText** ノードは、ElevenLabs Scribe モデルを使用して音声をテキストに変換します。変換されたテキスト、検出された言語コード、およびタイミング情報を含む単語レベルの JSON データを出力します。

<ReqHint/>
<UpdateReminder/>

## 対応モデル

| モデル | 説明 |
| :---- | :---------- |
| `scribe_v2` | 高精度音声認識 |

### 音声認識の出力

ノードは3つの出力を生成します：

| 出力 | 型 | 説明 |
| :----- | :--- | :---------- |
| **text** | STRING | 完全な変換テキスト |
| **language_code** | STRING | 検出された言語コード |
| **words_json** | STRING | タイミング情報付きの単語レベルの変換（JSON） |

### 音声認識パラメーター

| パラメーター | 説明 |
| :-------- | :---------- |
| **model_id** | 変換モデル選択（`scribe_v2`） |
| **tag_audio_events** | 笑い声、拍手などの音声イベントをタグ付け |
| **num_speakers** | 話者分離の話者数（0 = 自動） |
| **timestamps_granularity** | タイムスタンプの粒度: none, word, character |
| **diarize** | 話者分離の有効/無効 |
| **output_format** | 結果テキストの出力フォーマット |

テンプレートを参考にする場合の一般的な実行手順：

1. **LoadAudio** で音声ファイルを読み込み（例: `voice_demo.mp3`）
2. （オプション）**RecordAudio** を使用して直接録音
3. 音声を **ElevenLabsSpeechToText** に接続
4. モデルを選択（`scribe_v2`）
5. オプションパラメーターを設定：
   - **diarize** を有効にしてマルチスピーカー変換
   - **timestamps_granularity** を "word" に設定して単語レベルのタイムスタンプ
   - **num_speakers** を設定して話者数を指定（0 で自動検出）
6. **実行** をクリックするか `Ctrl(cmd) + Enter` を押す
7. **PreviewAny** ノードで変換テキストを表示
8. **words_json** PreviewAny ノードで単語レベルの JSON データを確認

### 補足事項

- **RecordAudio** ノードはデフォルトでバイパスされています。`Ctrl-B` で有効化してライブ録音
- **words_json** 出力には各単語の開始/終了タイムスタンプが含まれ、字幕生成やアライメントに便利
- **language_code** は自動検出され、ダウンストリームの多言語処理に使用可能
