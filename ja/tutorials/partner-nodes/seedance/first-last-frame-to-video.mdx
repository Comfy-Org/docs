Seedance 2.0 先頭・最終フレームから動画ノード（`ByteDance2FirstLastFrameNode`）は、開始画像と終了画像の間の完全な動画を生成します。最初と最後のフレームを提供すると、Seedance 2.0 がそれらを繋ぐモーションとトランジションを作成します。

<ReqHint/>
<UpdateReminder/>

## ワークフロー概要

先頭・最終フレームワークフローは4つのノードを使用します：

1. **LoadImage（×2）** — 先頭フレームと最終フレームの画像を読み込み
2. **ByteDance2FirstLastFrameNode** — 2枚の画像とテキストプロンプトを受け取り VIDEO を出力
3. **SaveVideo** — 生成された動画をディスクに保存

## ノードパラメータ

| パラメータ | 説明 |
| :--- | :--- |
| `model` | モデル名。デフォルト：`Seedance 2.0` |
| `prompt` | 動画のテキスト説明。トランジション、カメラワーク、シーンを記述 |
| `resolution` | 出力解像度。オプション：`480p`、`720p`、`1080p` |
| `aspect_ratio` | アスペクト比。オプション：`16:9`、`9:16`、`1:1`、`4:5` |
| `duration` | 動画の長さ（秒） |
| `generate_audio` | 動画と同期したオーディオを生成するかどうか |
| `seed` | ランダムシード |
| `cfg` | Classifier-free guidance scale |

## 入力

| 入力 | タイプ | 説明 |
| :--- | :--- | :--- |
| `first_frame` | IMAGE | 開始フレーム画像（直接画像入力） |
| `last_frame` | IMAGE | 終了フレーム画像（直接画像入力） |
| `first_frame_asset_id` | STRING | 検証済み実人物の先頭フレーム Asset ID |
| `last_frame_asset_id` | STRING | 検証済み実人物の最終フレーム Asset ID |

## ワークフローの実行

1. **画像を読み込み** — LoadImage ノードで先頭フレームと最終フレームの画像を読み込み
2. **プロンプトを記述** — トランジションと全体のシーンを記述。プロンプトはモーション、カメラワーク、スタイルのガイドに役立ちます
3. **パラメータを設定** — 解像度、アスペクト比、長さ、オーディオ設定を選択
4. **ワークフローを実行** — 「Queue」ボタンをクリックするか `Ctrl(Cmd) + Enter` を使用
5. **出力を確認** — SaveVideo ノードに生成された動画が表示

## ヒント

- **異なるフレームを選ぶ** — 先頭と最終フレームの差が大きいほど、生成されるトランジションが面白くなります
- **プロンプトをガイドとして** — プロンプトはシーンだけでなくトランジションを記述。カメラの移動方向（例：「ドローンが前方に飛び、その後180度回転」）を含めましょう
- **実人物の Asset ID** — 検証済みの実人物肖像を使用する場合、直接画像入力の代わりに `asset_id` 文字列を接続
