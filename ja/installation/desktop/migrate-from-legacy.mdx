従来の **Desktop Legacy**（シングルインストール）を使用している場合、Comfy Desktop は自動的に検出して移行できます。移行により、既存のカスタムノード、ワークフロー、設定、ユーザーデータを含む新しい Standalone インストールが作成されます。

## 移行の仕組み

Desktop Legacy はシングルインストールのデスクトップアプリです。Comfy Desktop はマルチインストレーションマネージャーです。移行時：

1. **新しい Standalone インストールが作成されます** — 独自の Python 環境と ComfyUI バージョン
2. **カスタムノードがコピーされます** — 新しいインストールへ
3. **モデルはリンクされます**（コピーされません）— 元の場所に残ります
4. **ワークフローと設定が引き継がれます**

Desktop Legacy インストールは変更されません。

## 移行手順

### 手順 1 — Comfy Desktop をインストール

[公式ダウンロードページ](https://dl.todesktop.com/241130tqe9q3y)から Comfy Desktop をダウンロードしてインストールします。

### 手順 2 — Comfy Desktop を起動

初回起動時に、Comfy Desktop はシステム上の Desktop Legacy インストールを自動検出します。移行を促すプロンプトが表示されます。

### 手順 3 — 「今すぐ移行」をクリック

**Migrate Now** をクリックして移行を開始します。Comfy Desktop は：
1. Desktop Legacy インストールを検出
2. 新しい Standalone インストールを作成
3. カスタムノード、ワークフロー、設定をコピー
4. モデルディレクトリをリンク

移行が完了すると、新しい Standalone インストールカードが Chooser ビューに表示されます。

## 移行されるもの

| 項目 | 移行される？ | 方法 |
|------|:---------:|------|
| ワークフロー | ✅ | 新しいインストールにコピー |
| カスタムノード | ✅ | 新しい環境で再インストール |
| モデル | ✅ | 共有ディレクトリ経由でリンク（コピーなし） |
| 設定 | ✅ | Desktop Legacy 設定を引き継ぎ |
| Python 環境 | 🆕 | 新しいバンドル Python |
| ComfyUI バージョン | 🆕 | 最新バージョンをインストール |

## なぜ移行するのか？

Comfy Desktop の Desktop Legacy に対する利点：

- **マルチインスタンス管理** — 異なるバージョンを同時実行
- **スナップショット** — 更新前に自動バックアップ
- **ワンクリックアップデート** — 最新の ComfyUI に即時更新
- **クラウド統合** — Comfy Cloud のネイティブサポート
- **Git クローンサポート** — 開発者向け
- **リモート接続** — 他のマシンの ComfyUI インスタンスに接続
