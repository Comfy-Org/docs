左上の **☰ メニュー**（1）をクリックし、**Desktop Settings**（2）を選択するとグローバル設定が開きます。

![デスクトップ設定を開く](/images/desktop/usage/comfy_desktop_settings_00_menu.png)

設定パネルは 4 つのタブに分かれています。サイドバーのタブをクリックして切り替えます。

## 環境設定

![環境設定](/images/desktop/usage/comfy_desktop_settings_01_preferences.png)

- **言語** — 表示言語を選択：**English** または **中文**。
- **起動時に最後のインスタンスを自動再起動** — 起動時に最後に使用したインスタンスを自動起動。
- **最後のウィンドウを閉じたら Comfy Desktop を終了する（Dashboard に戻らない）** — 最後のウィンドウを閉じたらアプリを完全終了。
- **インスタンスピッカーからクラウドを非表示** — Chooser ビューからクラウドインスタンスを非表示。
- **匿名利用テレメトリーを共有** — 匿名使用データを送信して Comfy Desktop の改善に貢献。
- **Comfy Desktop** — ComfyDesktop GitHub リポジトリへのリンク。

## アップデート

![アップデート](/images/desktop/usage/comfy_desktop_settings_02_updates.png)

- **ステータス** — 最新バージョンかどうか、インストール済みバージョン、最終確認日時を表示。
- **アップデートを確認** — 手動で新バージョンを確認。更新がある場合ボタンが**ダウンロード** → **インストール**に変化。
- **デスクトップアップデートを自動インストール** — バックグラウンドで自動更新。

## ストレージ

ストレージタブでは、Comfy Desktop がファイルを検索・保存する場所を設定します。核となる概念は**共有**です。各インスタンスが個別にモデルや入出力を持つ代わりに、すべてのインスタンスがアクセスできる共有ディレクトリを設定します。

![ストレージ](/images/desktop/usage/comfy_desktop_settings_03_storage.png)

- **共有モデル** — ここにリストされた各ディレクトリは、すべてのインスタンスによってスキャンされます。**プライマリ**とマークされたディレクトリに、新しいモデルがデフォルトで保存されます。

  ![プライマリに設定](/images/desktop/usage/comfy_desktop_settings_03-2_storage.png)

  - フォルダアイコンでファイルマネージャーで開く。
  - ディレクトリの **...** メニュー（1）をクリックし **Make primary**（2）を選択するとプライマリを変更できます。
  - **+ ディレクトリを追加**でモデルディレクトリを追加。
- **入力ディレクトリ** — すべてのインスタンスが入力ファイル（画像、動画など）を検索する共有パス。
- **出力ディレクトリ** — すべてのインスタンスが生成結果を保存する共有パス。カスタマイズして出力を一元管理。
- フォルダアイコンでパスを選択。**?** ツールチップで詳細情報。

## 詳細設定

![詳細設定](/images/desktop/usage/comfy_desktop_settings_04_advanced.png)

- **デフォルトのインストール場所** — 新インスタンスの作成先。フォルダアイコンで変更可能。
- **PyPI ミラー URL** — カスタム pip ミラー（例：`https://mirrors.aliyun.com/pypi/simple/`）。
- **中国ミラーを使用（Git & PyPI）** — Git と PyPI に中国ミラーサーバーを使用。
- **キャッシュディレクトリ** — 一時データとキャッシュの保存先。フォルダアイコンで変更可能。

## データ保存場所

Comfy Desktop はプラットフォーム固有のディレクトリにデータを保存します：

### Windows
```
アプリデータ:  %APPDATA%\Comfy Desktop
ログ:          %APPDATA%\Comfy Desktop\logs
```

### macOS
```
アプリデータ:  ~/Library/Application Support/Comfy Desktop
ログ:          ~/Library/Application Support/Comfy Desktop/logs
```

### Linux（XDG ベースディレクトリ）
```
設定:  ~/.config/Comfy Desktop
データ:  ~/.local/share/Comfy Desktop
キャッシュ:  ~/.cache/Comfy Desktop
状態:  ~/.local/state/Comfy Desktop
```

## キーボードショートカット

| ショートカット | 操作 |
|-------------|------|
| `Ctrl+R` / `Cmd+R` | 現在のビューを更新 |
| `Ctrl+W` / `Cmd+W` | 現在のパネルを閉じる |
| `Escape` | モーダルまたはオーバーレイを閉じる |
| `Ctrl+,` / `Cmd+,` | グローバル設定を開く |

## Deep Links (comfy://)

Comfy Desktop は `comfy://` カスタムプロトコルをサポートしており、ブラウザやターミナルから直接ローカルのワークフローファイルを開くことができます。

### Deep Link でワークフローを開く

```
comfy://open?file=/path/to/workflow.json
```

`comfy://` リンクをクリックするか、コマンドラインから実行すると：
1. Comfy Desktop を起動（未起動の場合）
2. 指定されたワークフローファイルを ComfyUI エディタで開く
3. 既存のウィンドウを前面に表示（既に起動している場合）

> **注意：** Deep links はローカルファイルパスのみ対応。Comfy Desktop がプロトコルハンドラとして登録されている必要があります（インストール時に自動登録されます）。
