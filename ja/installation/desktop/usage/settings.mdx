左サイドバーの歯車アイコン（⚙️）からグローバル設定を開きます。

設定パネルは 4 つのタブに分かれています。サイドバーのタブをクリックして切り替えます。

## 環境設定

![環境設定](/images/desktop/usage/comfy_desktop_settings_01_preferences.png)

### 言語

ドロップダウンメニューから表示言語を選択できます：
- **English**
- **中文**

### アプリの動作

- **起動時に最後のインスタンスを自動再起動** — オンにすると、Comfy Desktop 起動時に最後に使用したインスタンスが自動的に起動します。
- **最後のウィンドウを閉じたら Comfy Desktop を終了する（Dashboard に戻らない）** — オンにすると、最後の ComfyUI ウィンドウを閉じた時に、Dashboard に戻らずアプリケーションを完全に終了します。
- **インスタンスピッカーからクラウドを非表示** — オンにすると、Chooser ビューからクラウドインスタンスが非表示になります（クラウドサービスを使用しない場合に便利）。

### プライバシー

- **匿名利用テレメトリーを共有** — オンにすると、Comfy Desktop は匿名の使用データを送信して製品改善に役立てます。基本的な機能使用統計とクラッシュレポートが含まれます。

### コミュニティ

- **Comfy Desktop** — ComfyDesktop GitHub リポジトリへのリンクボタン。クリックするとブラウザでリポジトリが開きます。

## アップデート

![アップデート](/images/desktop/usage/comfy_desktop_settings_02_updates.png)

アップデートタブでは、現在の Comfy Desktop バージョンとアプリケーションの更新管理を行えます。

ステータス領域には、Comfy Desktop が最新かどうか、インストールされているバージョン、最終チェック日時が表示されます。

- **アップデートを確認** — クリックして手動で新しいバージョンを確認します。利用可能なアップデートがある場合、ボタンは**ダウンロード**、次に**インストール**に変わります。
- **デスクトップアップデートを自動インストール** — オンにすると、Comfy Desktop はバックグラウンドで自動的に更新をダウンロードしてインストールします。

## ストレージ

![ストレージ](/images/desktop/usage/comfy_desktop_settings_03_storage.png)

ストレージタブでは、モデルの検索場所と入出力の保存先を管理します。

### 共有モデル

Comfy Desktop がモデルをスキャンするディレクトリを追加します。リストの最初のディレクトリは**プライマリ**としてマークされます。新しいモデルはデフォルトでこのディレクトリに保存されます。
- フォルダアイコンをクリックすると、ファイルマネージャーでディレクトリが開きます。
- ディレクトリの **...** メニューから順序の変更やプライマリ設定が行えます。
- **+ ディレクトリを追加**をクリックして、追加のモデルディレクトリを選択します。

### 共有ディレクトリ

- **入力ディレクトリ** — ComfyUI が入力ファイル（画像、動画など）を検索するディレクトリ。カスタムパスを指定して複数のインスタンス間で入力を共有できます。
- **出力ディレクトリ** — ComfyUI が生成結果（画像、動画など）を保存するディレクトリ。パスをカスタマイズしてすべての出力を 1 か所にまとめられます。

フォルダアイコンをクリックして新しいパスを選択します。各ラベルの **?** ツールチップで詳細を確認できます。

## 詳細設定

![詳細設定](/images/desktop/usage/comfy_desktop_settings_04_advanced.png)

### デフォルトのインストール場所

新しいスタンドアロン ComfyUI インスタンスが作成されるデフォルトディレクトリを設定します。フォルダアイコンをクリックして別の場所を選択します。

### 詳細

- **PyPI ミラー URL** — カスタム pip ミラー URL を入力します（例：`https://mirrors.aliyun.com/pypi/simple/`）。デフォルトの PyPI が遅い、またはブロックされている環境に便利です。
- **中国ミラーを使用（Git & PyPI）** — オンにすると、Comfy Desktop は Git リポジトリと PyPI パッケージに中国のミラーサーバーを自動的に使用します。中国本土のユーザーに便利です。

### キャッシュ

- **キャッシュディレクトリ** — 現在のキャッシュパスを表示します。フォルダアイコンをクリックして新しいディレクトリを選択できます。Comfy Desktop は一時データとキャッシュデータをここに保存します。

## データ保存場所

Comfy Desktop はプラットフォーム固有のディレクトリにデータを保存します：

### Windows
```
アプリデータ:  %APPDATA%\Comfy Desktop
ログ:          %APPDATA%\Comfy Desktop\logs
```

### macOS
```
アプリデータ:  ~/Library/Application Support/Comfy Desktop
ログ:          ~/Library/Application Support/Comfy Desktop/logs
```

### Linux（XDG ベースディレクトリ）
```
設定:  ~/.config/Comfy Desktop
データ:  ~/.local/share/Comfy Desktop
キャッシュ:  ~/.cache/Comfy Desktop
状態:  ~/.local/state/Comfy Desktop
```

## キーボードショートカット

| ショートカット | 操作 |
|-------------|------|
| `Ctrl+R` / `Cmd+R` | 現在のビューを更新 |
| `Ctrl+W` / `Cmd+W` | 現在のパネルを閉じる |
| `Escape` | モーダルまたはオーバーレイを閉じる |
| `Ctrl+,` / `Cmd+,` | グローバル設定を開く |

## Deep Links (comfy://)

Comfy Desktop は `comfy://` カスタムプロトコルをサポートしており、ブラウザやターミナルから直接ローカルのワークフローファイルを開くことができます。

### Deep Link でワークフローを開く

```
comfy://open?file=/path/to/workflow.json
```

`comfy://` リンクをクリックするか、コマンドラインから実行すると：
1. Comfy Desktop を起動（未起動の場合）
2. 指定されたワークフローファイルを ComfyUI エディタで開く
3. 既存のウィンドウを前面に表示（既に起動している場合）

> **注意：** Deep links はローカルファイルパスのみ対応。Comfy Desktop がプロトコルハンドラとして登録されている必要があります（インストール時に自動登録されます）。
