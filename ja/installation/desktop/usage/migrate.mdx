旧版の **Desktop Legacy**（シングルインストール版）を使用している場合、Comfy Desktop が自動的に検出して移行できます。既存のカスタムノード、ワークフロー、設定、ユーザーデータを引き継いだ新しい Standalone インストールが作成されます。

## 移行の仕組み

Desktop Legacy はシングルインストールのデスクトップアプリです。Comfy Desktop はマルチインストールマネージャーです。移行時：

1. **新しい Standalone インストールが作成**：独自の Python 環境と ComfyUI バージョンを持つ
2. **カスタムノードがコピー**：新しいインストールへ
3. **モデルがリンク**（コピーされない）— 元の場所に残る
4. **ワークフローと設定が引き継がれる**：そのまま続行可能

Desktop Legacy インストールは移行中に変更されません。

## 移行手順

<Steps>
<Step title="Comfy Desktop をインストール">
[公式ダウンロードページ](https://comfy.org/download) から Comfy Desktop をダウンロードしてインストールします。
</Step>

<Step title="Comfy Desktop を起動">
インストール後、初めて Comfy Desktop を開くと、システム上の Desktop Legacy インストールを自動的に検出します。移行バナーまたはプロンプトが表示されます。
</Step>

<Step title="「今すぐ移行」を選択">
**Migrate Now** をクリックして移行を開始します：

1. Desktop Legacy インストールを検出
2. 新しい Standalone インストールを作成
3. カスタムノード、ワークフロー、設定をコピー
4. モデルディレクトリをリンク

移行完了後、新しい Standalone インストールカードが Chooser ビューに表示されます。
</Step>

<Step title="確認">
新しいインストールを起動して動作を確認：
- すべてのワークフローが利用可能であること
- カスタムノードがインストールされていること
- モデルにアクセスできること
</Step>
</Steps>

## 移行後

移行が成功したら、新しい Standalone インストールをそのまま使い続けられます。Desktop Legacy インストールはバックアップとして手つかずのまま残ります。

## 引き継がれる内容

| 項目 | 引き継がれる？ | 方法 |
|------|:------------:|------|
| ワークフロー | ✅ | 新しいインストールにコピー |
| カスタムノード | ✅ | 新しい環境で再インストール |
| モデル | ✅ | 共有ディレクトリ経由でリンク（コピーなし） |
| 設定 | ✅ | Desktop Legacy の設定を引き継ぎ |
| Python 環境 | 🆕 | バンドル Python の新環境 |
| ComfyUI バージョン | 🆕 | 最新版をインストール |

## トラブルシューティング

### Desktop Legacy が自動検出されない

手動で追加する方法：

1. Chooser ビューで **+** カードをクリック
2. **Legacy Desktop** をソースタイプとして選択
3. Comfy Desktop が既存のインストールを検索
4. プロンプトに従って移行

### 移行に失敗する

エラーが発生した場合：

- **ログを確認**：プラットフォーム固有のログディレクトリで詳細を確認
- **Desktop Legacy インストールが完全であることを確認**：移行は既存のインストールを読み取る必要あり
- **再試行**：ネットワークタイムアウトなどの一時的なエラーの場合、再試行で成功する可能性あり

### 移行後、モデルが見つからない

モデルは移行時にリンク（コピーではなく）されます。モデルが表示されない場合：

1. 移行したインストールの **Manage** パネルを開く
2. **Settings → Model directories** に移動
3. 古いモデルディレクトリがリストにあるか確認
4. ない場合は手動で追加
