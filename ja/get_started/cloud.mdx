## Comfy Cloud とは？

ComfyUI Cloud は、ローカルバージョンと同じ機能を備えた ComfyUI のクラウドバージョンです。すべてプリインストールされており、すぐに使用できます。

### 主な機能

<CardGroup cols={2}>
  <Card title="設定不要" icon="bolt">
    インストールは不要です。すべてのモデルとカスタムノードがプリインストールされており、すぐに使用できます
  </Card>
  
  <Card title="強力な GPU" icon="microchip">
    自有ハードウェアがなくても、強力なサーバー GPU でワークフローを高速実行できます
  </Card>
  
  <Card title="常に最新" icon="arrows-rotate">
    最新の ComfyUI リリースと機能に自動的に更新されます
  </Card>
  
  <Card title="どこからでもアクセス" icon="globe">
    インターネット接続のある任意のデバイスから ComfyUI を使用できます - ローカルインストールは不要です
  </Card>
</CardGroup>

## クラウド版 vs ローカル版

ComfyUI には、公式クラウドバージョンである [Comfy Cloud](https://comfy.org/cloud) と、オープンソースのセルフホストバージョンの両方があります。強力な GPU をお持ちの場合、ComfyUI をローカルで実行するのは素晴らしい選択肢です。一方、クラウドバージョンはすぐに使用できるオンラインサービスです。URL を開くだけで、インストールや設定は不要です。


| 类别 | Comfy Cloud | 自托管（本地 ComfyUI） |
|----------|-------------|------------------------------|
| **コスト** | 月額サブスクリプション | 無料 |
| **GPU** | 強力な Blackwell RTX 6000 Pro | 自有 GPU を用意 |
| **技術知識** | 技術知識は不要です。 | デスクトップ版とポータブル版では簡単に始められますが、カスタムノードのインストールやローカルインストールの問題を解決する必要があります。 |
| **カスタムノード** | プリインストールされたカスタムノードを使用でき、互換性の問題を心配する必要はありません。 | 任意のカスタムノードをインストールできますが、自分で管理する必要があります。 |
| **モデル** | プリインストールされたモデルを使用します。Civitai から LoRA モデルをインポートできます。Hugging Face からモデルをインポートできます（近日予定）。 | 任意のモデルを使用できますが、まずダウンロードする必要があります。 |
| **主な違い** | チームのオンボーディングが容易 | オフライン動作、無限にカスタマイズ可能 |
| **始める** | [ComfyUI Cloud を実行](https://comfy.org/cloud) | [ComfyUI をローカルにインストール](/installation/system_requirements) |

## ComfyUI Cloud の使い方

ComfyUI Cloud の使用は、ローカルの ComfyUI を使用するのと基本的に同じです。ComfyUI を初めて使用する場合は、以下のクイックヒントを参考にしてください。

<Steps>
  <Step title="テンプレートを選択">
    左サイドバーのテンプレートアイコンをクリックして、利用可能なワークフローを閲覧してください。
    ![ワークフローテンプレートを開く](/images/cloud/open_template.webp)
  </Step>
  
  <Step title="実行したいワークフローを選択">
    実行したいテンプレートをクリックしてください。
    ![ワークフローテンプレートを選択](/images/cloud/template_library.webp)
  </Step>
  
  <Step title="テンプレートの入力を更新">
    1. 読み込まれたワークフローでは、クラウドバージョンですべてのモデルがプリインストールされているため、すべてのテンプレートは読み込み後すぐに実行準備が整います。
    2. 更新が必要な場合は、画像入力またはテキストプロンプトを提供するだけです。`Load Image` ノードまたは `CLIP Text Encode` ノードを確認してください。
    ![入力を確認](/images/cloud/workflow_input_or_loader.webp)
  </Step>
  
  <Step title="ワークフローを実行">
    すべて正しければ、「Run」アイコンをクリックするか、ショートカット「Ctrl + Enter」を使用してワークフローを実行してください。
    ![ワークフローを実行](/images/cloud/workflow_run_workflow.webp)
  </Step>

  <Step title="出力を確認">
    実行をクリックすると、サービスはワークフロー用にマシンの割り当てを開始します。キューパネルで該当するワークフローの実行ステータスを確認できます。
    ![キューを確認](/images/cloud/workflow_check_queue.webp)
  </Step>

  <Step title="コンテンツをローカルに保存">
    ワークフローの実行が完了した後、生成されたコンテンツをローカルに保存できます。アセットの種類に応じて、保存方法は以下の通りです。
    
    <Tabs>
      <Tab title="画像を保存">
        キューパネルまたは画像保存ノードで、生成された画像を右クリックし、「Save image」を選択して画像をローカルに保存してください。
        
        ![画像アセットを保存](/images/cloud/download_image_assets.jpg)
      </Tab>
      
      <Tab title="動画を保存">
        キューパネルまたは動画保存ノードで：
        1. ブラウザプレーヤーコンポーネントの 3 つの点をクリックしてメニューを開きます
        2. 「Download」を選択して動画をローカルに保存してください。
        
        ![動画アセットを保存](/images/cloud/download_video_assets.webp)
      </Tab>
      
      <Tab title="音声を保存">
        キューパネルまたは音声保存ノードで：
        1. ブラウザプレーヤーコンポーネントの 3 つの点をクリックしてメニューを開きます
        2. 「Download」を選択して音声ファイルをローカルに保存してください。
        
        ![音声アセットを保存](/images/cloud/download_audio_assets.webp)
      </Tab>
      
      <Tab title="3D アセットを保存">
        3D ブラウザノードメニューで、「Export」オプションを選択し、保存したいフォーマットを選択すると、3D ファイルがローカルに保存されます。
        
        ![3D アセットを保存](/images/cloud/download_3d_assets.webp)
      </Tab>
    </Tabs>
  </Step>
</Steps>
