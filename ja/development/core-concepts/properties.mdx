## ノードはプロパティのコンテナです

ノード通常は**_プロパティ_**を持っています。**_パラメータ_**または**_属性_**とも呼ばれるノードプロパティは、変更可能な変数です。一部のプロパティは、**_ウィジェット_**と呼ばれるデータ入力フィールドを使用して、ユーザーが手動で調整できます。他のプロパティは、プロパティの**_入力スロット_**またはポートに接続された他のノードによって自動的に駆動されます。通常、プロパティはウィジェットから入力へ、またその逆へ変換でき、ユーザーがプロパティ値を手動または自動的に制御できるようになります。

プロパティは多くの形式を取り、多くの異なる種類の情報を保持できます。例えば、**Load Checkpoint** ノードには単一のプロパティがあります：生成モデルチェックポイントファイルへのファイルパスです。**KSampler** ノードには、サンプリング**ステップ数**、**CFG** スケール、**サンプラー名** などの複数のプロパティがあります。

## データ型

情報は**_データ型_**と呼ばれる多くの異なる形式で存在できます。例えば、英数字テキストは**_文字列_**として知られ、整数は**_整数_**、小数点を含む数字は**_浮動小数点数_**または**_浮動小数_**として知られています。新しいデータ型は常に ComfyUI に追加されています。

ComfyUI は Python スクリプト言語で書かれており、データ型について非常に寛容です。対照的に、ComfyUI 環境は非常に**_強い型付け_**です。これは、異なるデータ型を混在させることができないことを意味します。例えば、画像出力を整数入力に接続することはできません。これはユーザーにとって大きな利点であり、適切なワークフロー構築をガイドし、プログラムエラーを防ぎます。
