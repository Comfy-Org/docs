[comfy-cli](https://github.com/Comfy-Org/comfy-cli)はComfyのコマンドラインツールです。[MCPサーバー](/ja/agent-tools)と並んで、エージェントや開発者にUI外からComfyを操作するもう一つの方法を提供します。

**ターミナルやスクリプトからアクセスしたい**場合に使用します。バッチジョブ、CIパイプライン、手軽な単発生成、Claude Codeスキルとの連携などに最適です。`comfy generate`サブコマンド（ベータ版）は、ワークフローに組み込むのと同じホスト型[パートナーノード](/ja/tutorials/partner-nodes/overview)を、ワンショットのCLI呼び出しとして実行します。

<Info>
  **`comfy generate`はベータ版です。** フラグ名、モデルエイリアス、出力形式は変更される可能性があります。基盤となるパートナーエンドポイントは安定しています。フィードバックは[comfy-cli GitHubリポジトリ](https://github.com/Comfy-Org/comfy-cli/issues)にお寄せください。
</Info>

## MCPとの連携

| | Comfy Cloud MCP | Comfy CLI (`comfy generate`) |
|---|---|---|
| **インターフェース** | MCPクライアント（Claude Code、Claude Desktopなど）とのチャット | ターミナルコマンドとスクリプト |
| **最適な用途** | 対話的なエージェントワークフロー、テンプレート検索、フルCloudワークフロー | 自動化、バッチ実行、CI、シェルパイプライン |
| **認証** | OAuth（クライアント）またはAPIキー（ヘッドレス） | [Comfy APIキー](/ja/development/api-development/getting-an-api-key) |
| **出力** | チャットセッション内に返される | ディスクに保存（`--download`）またはJSON（`--json`） |

MCPのセットアップ、インストール、ツール一覧、認証方法については、[Comfy Cloud MCP](/ja/agent-tools/cloud)を参照してください。

エージェントにターミナルからのComfy操作方法を教えるには、[Comfy Skills](https://github.com/Comfy-Org/comfy-skills/)リポジトリでスキルを参照またはコントリビュートしてください。

## インストール

初回のガイド付きセットアップ（ローカル vs Cloudルーティング、認証、スキルインストール）を実行するには:

```bash
comfy setup
```

CIやスクリプトで非対話的にインストールする場合は`-y`を渡します。

## `comfy generate`の前提条件

1. [Comfy APIキーを作成](/ja/development/api-development/getting-an-api-key)する
2. アカウントに[クレジットを追加](/ja/interface/credits)する
3. オプション: [パートナーノードと呼び出しごとの料金を参照](/ja/tutorials/partner-nodes/pricing)する

キーを一度設定します:

```bash
export COMFY_API_KEY=comfyui-...   # または各呼び出しで--api-keyを渡す
```

## 初めての生成

```bash
comfy generate flux-pro \
    --prompt "a cat on the moon, cinematic lighting" \
    --width 1024 --height 1024 \
    --download cat.png
```

CLIはローカルファイル入力をアップロードし、ジョブを送信し、準備ができるまでポーリングし、結果を`cat.png`に保存します。

## 利用可能なモデル

`comfy generate`は画像と動画のパートナーモデルをカバーします。CLIバージョンにインストールされているものを確認するには:

```bash
comfy generate list                            # すべてのモデル
comfy generate list --category text-to-video   # カテゴリでフィルター
comfy generate list --partner kling            # パートナーでフィルター
comfy generate schema flux-kontext             # 1つのモデルのパラメータ
```

よく使われるエイリアス:

| カテゴリ | 例 |
|---|---|
| **画像** | `flux-pro`, `flux-kontext`, `nano-banana` (Gemini Flash Image), `grok`, `grok-edit`, `ideogram-edit`, `dalle`, `recraft`, `stability` |
| **動画** | `seedance`, `kling`, `luma`, `runway`, `pika`, `vidu`, `hailuo`, `moonvalley`, `grok-video` |

環境の全セットは`comfy generate list`を実行してください。ベータ期間中はエイリアスやフラグが変更される可能性があります。

## 使用例

### テキストから画像へ

```bash
comfy generate nano-banana \
    --prompt "a watercolor of a sleeping fox" \
    --download fox.png
```

### 画像編集（参照画像）

ローカルパスを直接渡します。CLIはComfyストレージを通じてアップロードするか（またはパートナーによってはインラインでBase64エンコードします）:

```bash
comfy generate nano-banana \
    --prompt "add a top hat" \
    --image ./cat.png \
    --download edited.png

comfy generate flux-kontext \
    --prompt "add a top hat and a monocle" \
    --input_image ./photo.jpg \
    --download out.png
```

一度アップロードして署名付きURLを再利用する:

```bash
comfy generate upload ./photo.jpg
# --input_image用の署名付きURLを出力
```

<Note>
  アップロードされた参照アセットは24時間後に自動削除されます。長期実行パイプラインでは、ジョブごとに再アップロードしてください。詳細は[CLIリファレンス](/ja/comfy-cli/reference#upload)を参照。
</Note>

### テキストから動画へ

動画ジョブは非同期です。CLIはデフォルトでブロックし、準備ができるまでポーリングします:

```bash
comfy generate seedance \
    --prompt "a hummingbird hovering over a flower" \
    --resolution 1080p --duration 5 \
    --download hummingbird.mp4

comfy generate kling \
    --prompt "a paper boat drifting on a river at dusk" \
    --duration 5 \
    --download boat.mp4
```

`--async`を渡すと即座にジョブIDが返り、後で再開できます:

```bash
comfy generate luma --prompt "neon koi swimming through clouds" --aspect_ratio 16:9 --async
comfy generate resume luma <job_id> --download out.mp4
```

### JSONでのスクリプト処理

パイプラインでは、`--json`が生のAPIレスポンスを出力します:

```bash
comfy generate dalle --prompt "a watercolor whale" --json | jq '.data[0].url'
```
