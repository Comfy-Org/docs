Access Global Settings from the gear icon (⚙️) in the left sidebar.

The settings panel is organized into four tabs. Click a tab in the sidebar to switch views.

## Preferences

![Preferences tab](/images/desktop/usage/comfy_desktop_settings_01_preferences.png)

### Language

Choose the display language from the dropdown menu. The following languages are available:
- **English**
- **中文**

### App Behavior

- **Auto-restart last instance on launch** — When enabled, Comfy Desktop automatically starts the last used instance on boot.
- **Close Comfy Desktop on last window (don't return to Dashboard)** — When enabled, closing the last ComfyUI window will fully exit the application instead of returning to the Dashboard.
- **Hide Cloud from the Instance Picker** — When enabled, cloud instances are hidden from the Chooser view (useful if you don't use cloud services).

### Privacy

- **Share anonymous usage telemetry** — When enabled, Comfy Desktop sends anonymous usage data to help improve the product. This includes basic feature usage statistics and crash reports.

### Community

- **Comfy Desktop** — A link button to the ComfyDesktop GitHub repository. Click to open the repo in your browser.

## Updates

![Updates tab](/images/desktop/usage/comfy_desktop_settings_02_updates.png)

The Updates tab shows your current Comfy Desktop version and lets you manage application updates.

The status area displays whether Comfy Desktop is up to date, the currently installed version, and the last update check time.

- **Check for updates** — Click to manually check for a new version. If an update is available, the button changes to **Download** and then **Install**.
- **Automatically install Desktop updates** — When enabled, Comfy Desktop downloads and installs updates in the background without requiring manual intervention.

## Storage

![Storage tab](/images/desktop/usage/comfy_desktop_settings_03_storage.png)

The Storage tab manages where Comfy Desktop finds your models and stores inputs/outputs.

### Shared Models

Add directories that Comfy Desktop scans for models. The first directory in the list is marked as **Primary** — this is where new models are saved by default.
- Click the folder icon to open a directory in your file manager.
- Click the **...** menu on a directory to change its order or make it the Primary directory.
- Click **+ Add directory** to browse for and include an additional model directory.

### Shared Directories

- **Input Directory** — The directory where ComfyUI looks for input files (images, videos, etc.). You can specify a custom path to share inputs across multiple instances.
- **Output Directory** — The directory where ComfyUI saves generated outputs (images, videos, etc.). Customizing this path lets you keep all outputs in one place.

Click the folder icon to browse and select a new path for either directory. Use the **?** tooltip next to each label for more details.

## Advanced

![Advanced tab](/images/desktop/usage/comfy_desktop_settings_04_advanced.png)

### Default Install Location

Set the default directory where new standalone ComfyUI instances are created. Click the folder icon to browse for a different location.

### Advanced

- **PyPI Mirror URL** — Enter a custom pip mirror URL (e.g., `https://mirrors.aliyun.com/pypi/simple/`) for environments where the default PyPI is slow or blocked.
- **Use Chinese Mirrors (Git & PyPI)** — When enabled, Comfy Desktop automatically uses Chinese mirror servers for Git repositories and PyPI packages. This is helpful for users in mainland China where direct access is slow.

### Cache

- **Cache Directory** — Shows the current cache path. You can change it by clicking the folder icon and selecting a new directory. This is where Comfy Desktop stores temporary and cached data.

## Data Locations

Comfy Desktop stores data in platform-specific directories:

### Windows
```
App data:  %APPDATA%\Comfy Desktop
Logs:      %APPDATA%\Comfy Desktop\logs
```

### macOS
```
App data:  ~/Library/Application Support/Comfy Desktop
Logs:      ~/Library/Application Support/Comfy Desktop/logs
```

### Linux (XDG Base Directory)
```
Config:  ~/.config/Comfy Desktop
Data:    ~/.local/share/Comfy Desktop
Cache:   ~/.cache/Comfy Desktop
State:   ~/.local/state/Comfy Desktop
```

## Deep Links (comfy://)

Comfy Desktop supports the `comfy://` custom protocol for opening local workflow files directly from your browser or terminal.

### Opening a Workflow via Deep Link

```
comfy://open?file=/path/to/workflow.json
```

Clicking a `comfy://` link or running it from the command line will:
1. Launch Comfy Desktop (if not already running)
2. Open the specified workflow file in the ComfyUI editor
3. Focus the existing window (if already running)

This works on all platforms:
- **macOS**: Click a `comfy://` link in any app
- **Windows/Linux**: Click the link in a browser or run it from the terminal

> **Note:** Deep links only work with local file paths. The feature requires Comfy Desktop to be installed as the protocol handler — it registers itself automatically during installation.
