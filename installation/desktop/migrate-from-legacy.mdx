If you're using the original **Desktop Legacy** (single-install), ComfyUI Desktop can automatically detect and migrate your installation. Migration creates a new Standalone installation with your existing custom nodes, workflows, settings, and user data.

## How Migration Works

Desktop Legacy was a single-install desktop app. ComfyUI Desktop is a multi-installation manager. When you migrate:

1. **A new Standalone installation is created** — with its own Python environment and ComfyUI version
2. **Your custom nodes are copied** — to the new installation
3. **Your models are linked** (not copied) — they stay where they are
4. **Your workflows and settings are carried over** — so you can pick up right where you left off

Your Desktop Legacy installation is not modified during migration.

## Step-by-Step Migration

### Step 1 — Install ComfyUI Desktop

Download and install ComfyUI Desktop from the [official download page](https://comfy.org/download).

### Step 2 — Start ComfyUI Desktop

When you first open ComfyUI Desktop after installing, it will automatically detect any Desktop Legacy installation on your system. You'll see a migration banner or be prompted to migrate.

### Step 3 — Choose "Migrate Now"

Click **Migrate Now** to start the migration process. ComfyUI Desktop will:

1. Detect your Desktop Legacy installation
2. Create a new Standalone installation
3. Copy custom nodes, workflows, and settings
4. Link your model directories

After migration completes, your new Standalone installation card will appear in the Chooser view.

### Step 4 — Verify

Launch your new installation to confirm everything works:
- All your workflows should be available
- Custom nodes should be installed
- Models should be accessible

## Post-Migration

After migrating successfully, you can continue using your new Standalone installation. Your Desktop Legacy installation remains untouched as a backup.

## What Gets Carried Over

| Item | Carried Over? | How |
|------|:------------:|-----|
| Workflows | ✅ | Copied to the new installation |
| Custom Nodes | ✅ | Re-installed in the new environment |
| Models | ✅ | Linked via shared directories (not copied) |
| Settings | ✅ | Desktop Legacy settings are carried over |
| Python Environment | 🆕 | Fresh environment with bundled Python |
| ComfyUI Version | 🆕 | Latest version installed |

## Troubleshooting

### Desktop Legacy not detected automatically

If ComfyUI Desktop doesn't detect your Desktop Legacy installation, you can manually add it:

1. In the Chooser view, click the **+** card
2. Select **Legacy Desktop** as the source type
3. ComfyUI Desktop will search for any existing Desktop Legacy installations
4. Follow the migration prompts

### Migration fails

If the migration process encounters an error:

- **Check the logs** at the platform-specific log directory for details
- **Ensure your Desktop Legacy installation is intact** — the migration process reads from your existing install
- **Try again** — if the issue was a network timeout or temporary error, a second attempt may succeed

### After migration, models are missing

Models are linked (not copied) during migration. If models don't appear in the new installation:

1. Open the **Manage** panel for the migrated installation
2. Go to **Settings → Model directories**
3. Verify that your old model directory is listed
4. If not, add it manually

## Why Migrate?

ComfyUI Desktop offers several advantages over Desktop Legacy:

- **Multi-installation management** — Run different versions side by side
- **Snapshots** — Automatic backups before every update
- **One-click updates** — Latest ComfyUI with a single click
- **Cloud integration** — Native Comfy Cloud support
- **Git clone support** — For developers who want to track branches
- **Remote connections** — Connect to ComfyUI instances on other machines
