## 개요

텍스트 프롬프트를 기반으로 기존 이미지를 수정합니다. 이 노드는 이미지와 텍스트 설명을 Grok API로 전송하며, Grok API는 사용자의 지시에 따라 이미지를 편집한 후 결과를 반환합니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|----------|-------|-------------|
| `prompt` | 문자열 | 예 | 없음 | 이미지를 생성하는 데 사용되는 텍스트 프롬프트입니다. 공백을 제거한 후 최소 1자 이상이어야 합니다. |
| `model` | 모델 | 예 | 설명 참조 | 사용할 Grok 이미지 모델입니다. 이 매개변수에는 모델 선택 후 나타나는 여러 하위 옵션이 있습니다. 사용 가능한 모델: `grok-imagine-image-quality`<br />`grok-imagine-image-pro`<br />`grok-imagine-image`. 각 모델은 서로 다른 기능을 제공합니다(아래 참고 참조). |
| `seed` | 정수 | 예 | 0부터 2147483647까지 | 노드가 다시 실행될지 여부를 결정하는 시드입니다. 시드와 상관없이 실제 결과는 비결정적입니다. (기본값: 0) |

**`model` 매개변수 제약 사항에 대한 참고:**
- `model` 매개변수는 동적 콤보로, `해상도`, `이미지 수`, `이미지`, `종횡비`에 대한 하위 옵션을 포함합니다.
- **`grok-imagine-image-quality`**: 최대 3개의 입력 이미지를 지원하며, 사용자 정의 종횡비를 허용합니다.
- **`grok-imagine-image-pro`**: 단일 입력 이미지만 지원하며, 사용자 정의 종횡비를 허용하지 않습니다.
- **`grok-imagine-image`**: 최대 3개의 입력 이미지를 지원하며, 사용자 정의 종횡비를 허용합니다.
- **편집을 위해 적어도 하나의 입력 이미지가 필요합니다**. 이미지를 제공하지 않으면 노드에서 오류가 발생합니다.
- **사용자 정의 종횡비**(`aspect_ratio` 하위 옵션)는 이미지 입력에 여러 이미지를 연결한 경우에만 허용됩니다. 단일 이미지만 제공되는 경우 종횡비는 "자동"으로 설정해야 합니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `IMAGE` | 이미지 | Grok API에서 반환된 편집된 이미지입니다. 단일 이미지가 생성된 경우 바로 반환됩니다. 여러 이미지가 생성된 경우, 이를 하나의 배치 텐서로 연결하여 반환합니다. |
