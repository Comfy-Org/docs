지정된 디렉터리에서 이미지와 해당 텍스트 캡션의 배치를 로드하여 학습에 사용합니다. 이 노드는 자동으로 이미지 파일과 관련 캡션 텍스트 파일을 검색하고, 지정된 크기 조정 설정에 따라 이미지를 처리하며, 제공된 CLIP 모델을 사용해 캡션을 인코딩합니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|----------|-------|-------------|
| `folder` | 문자열 | 예 | - | 이미지를 로드할 폴더입니다. |
| `clip` | CLIP | 예 | - | 텍스트를 인코딩하는 데 사용되는 CLIP 모델입니다. |
| `resize_method` | 콤보 | 아니요 | "없음"<br />"늘림"<br />"자르기"<br />"패딩" | 이미지를 리사이즈하는 데 사용되는 방법(기본값: "없음"). |
| `width` | 정수 | 아니요 | -1부터 10000까지 | 이미지를 리사이즈할 너비입니다. -1은 원래 너비를 사용함을 의미합니다(기본값: -1). |
| `height` | 정수 | 아니요 | -1부터 10000까지 | 이미지를 리사이즈할 높이입니다. -1은 원래 높이를 사용함을 의미합니다(기본값: -1). |

**참고:** CLIP 입력은 유효해야 하며, None일 수 없습니다. CLIP 모델이 체크포인트 로더 노드에서 가져온 경우, 체크포인트에 유효한 CLIP 또는 텍스트 인코더 모델이 포함되어 있는지 확인하세요.

**폴더 구조 참고:** 이 노드는 kohya-ss/sd-scripts 폴더 구조를 지원합니다. 하위 폴더 이름이 숫자 뒤에 언더스코어(_)로 시작하면(예: `5_myclass`), 그 숫자가 반복 횟수로 사용되며, 해당 하위 폴더 내의 이미지는 그만큼 여러 번 로드됩니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-------------|-----------|-------------|
| `IMAGE` | 이미지 | 로드되고 처리된 이미지의 배치입니다. |
| `CONDITIONING` | CONDITIONING | 텍스트 캡션에서 인코딩된 컨디셔닝 데이터입니다. |
