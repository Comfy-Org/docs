이 노드는 비디오 생성을 위한 Alibaba Wan Fun Control 모델을 지원하기 위해 추가되었으며, [이 커밋](https://github.com/comfyanonymous/ComfyUI/commit/3661c833bcc41b788a7c9f0e7bc48524f8ee5f82) 이후 추가되었습니다.

- **용도:** Wan 2.1 Fun Control 모델을 사용하여 비디오 생성에 필요한 조건 정보를 준비합니다.

WanFunControlToVideo 노드는 비디오 생성을 위한 Wan Fun Control 모델을 지원하도록 설계된 ComfyUI 추가 기능으로, WanFun 제어를 활용해 비디오를 생성하는 것을 목표로 합니다.

이 노드는 필수적인 조건 정보를 준비하는 지점이며, 잠재 공간의 중심점을 초기화하여 Wan 2.1 Fun 모델을 사용한 후속 비디오 생성 과정을 안내합니다. 노드의 이름은 그 기능을 명확히 나타내며, 다양한 입력을 받아 WanFun 프레임워크 내에서 비디오 생성을 제어할 수 있는 형식으로 변환합니다.

ComfyUI 노드 계층 구조에서 이 노드의 위치는 비디오 생성 파이프라인의 초기 단계에서 작동하며, 실제 비디오 프레임의 샘플링이나 디코딩 전에 조건 신호를 조작하는 데 중점을 둡니다.

## 입력

| 매개변수 이름      | 필수 | 데이터 유형           | 설명                                                  | 기본값 |
|:-------------------|:---------|:-------------------|:-------------------------------------------------------------|:-------------|
| positive           | 예      | CONDITIONING       | 표준 ComfyUI 긍정적 조건 정보, 일반적으로 "CLIP 텍스트 인코더" 노드에서 가져옵니다. 긍정적 프롬프트는 사용자가 생성할 비디오의 내용, 주제 및 예술적 스타일을 설명합니다. | N/A  |
| negative           | 예      | CONDITIONING       | 표준 ComfyUI 부정적 조건 정보, 일반적으로 "CLIP 텍스트 인코더" 노드에서 생성됩니다. 부정적 프롬프트는 사용자가 생성된 비디오에서 피하고자 하는 요소, 스타일 또는 아티팩트를 지정합니다. | N/A  |
| vae                | 예      | VAE                | Wan 2.1 Fun 모델 계열과 호환되는 VAE(변분 오토인코더) 모델이 필요하며, 이미지/비디오 데이터의 인코딩 및 디코딩에 사용됩니다. | N/A  |
| width              | 예      | INT                | 출력 비디오 프레임의 원하는 너비(픽셀), 기본값은 832, 최소값은 16, 최대값은 nodes.MAX_RESOLUTION에 따라 결정되며, 증분 크기는 16입니다. | 832  |
| height             | 예      | INT                | 출력 비디오 프레임의 원하는 높이(픽셀), 기본값은 480, 최소값은 16, 최대값은 nodes.MAX_RESOLUTION에 따라 결정되며, 증분 크기는 16입니다. | 480  |
| length             | 예      | INT                | 생성된 비디오의 총 프레임 수, 기본값은 81, 최소값은 1, 최대값은 nodes.MAX_RESOLUTION에 따라 결정되며, 증분 크기는 4입니다. | 81   |
| batch_size         | 예      | INT                | 한 번에 생성되는 비디오의 수, 기본값은 1, 최소값은 1, 최대값은 4096입니다. | 1    |
| clip_vision_output | 아니요     | CLIP_VISION_OUTPUT | (선택사항) CLIP 비전 모델이 추출한 시각적 특징으로, 시각적 스타일과 내용을 안내할 수 있습니다. | 없음 |
| start_image        | 아니요     | IMAGE              | (선택사항) 생성된 비디오의 시작 부분에 영향을 미치는 초기 이미지입니다. | 없음 |
| control_video      | 아니요     | IMAGE              | (선택사항) 사용자가 사전 처리된 ControlNet 참조 비디오를 제공하여 생성된 비디오의 움직임과 잠재적 구조를 안내할 수 있도록 합니다. | 없음 |

## 출력

| 매개변수 이름      | 데이터 유형           | 설명                                                  |
|:-------------------|:-------------------|:-------------------------------------------------------------|
| positive           | CONDITIONING       | 강화된 긍정적 조건 정보를 제공하며, 여기에는 인코딩된 start_image와 control_video가 포함됩니다. |
| negative           | CONDITIONING       | 또한 강화된 부정적 조건 정보를 제공하며, 동일한 concat_latent_image를 포함합니다. |
| latent             | LATENT             | "samples"라는 키를 가진 빈 잠재 텐서를 포함한 딕셔너리입니다. |
