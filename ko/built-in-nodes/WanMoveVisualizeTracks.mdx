> 이 문서는 AI로 생성되었습니다. 오류를 발견하거나 개선할 점이 있다면 언제든지 기여해 주세요! [GitHub에서 편집하기](https://github.com/Comfy-Org/embedded-docs/blob/main/comfyui_embedded_docs/docs/WanMoveVisualizeTracks/en.md)

WanMoveVisualizeTracks 노드는 모션 트래킹 데이터를 이미지 또는 비디오 프레임 시퀀스 위에 오버레이합니다. 트래킹된 포인트들의 움직임 경로와 현재 위치를 포함한 시각적 표현을 그려 모션 데이터를 눈에 보이게 하고 분석을 쉽게 만듭니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|----------|-------|-------------|
| `images` | IMAGE | 예 | - | 트랙을 시각화할 입력 이미지 또는 비디오 프레임의 시퀀스입니다. |
| `tracks` | TRACKS | 아니요 | - | 포인트 경로와 가시성 정보를 포함한 모션 트래킹 데이터입니다. 제공되지 않으면 입력 이미지는 변경 없이 전달됩니다. |
| `line_resolution` | INT | 예 | 1 - 1024 | 각 트랙의 추적 경로 선을 그리는 데 사용할 이전 프레임의 수 (기본값: 24) |
| `circle_size` | INT | 예 | 1 - 128 | 각 트랙의 현재 위치에 그려지는 원의 크기 (기본값: 12) |
| `opacity` | FLOAT | 예 | 0.0 - 1.0 | 그려진 트랙 오버레이의 불투명도 (기본값: 0.75) |
| `line_width` | INT | 예 | 1 - 128 | 트랙 경로를 그리는 데 사용되는 선의 폭 (기본값: 16) |

**참고:** 입력 이미지의 수가 제공된 `tracks` 데이터의 프레임 수와 맞지 않을 경우, 이미지 시퀀스가 트랙 길이에 맞도록 반복됩니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-------------|-----------|-------------|
| `IMAGE` | IMAGE | 모션 트래킹 데이터가 오버레이로 시각화된 이미지 시퀀스입니다. `tracks`가 제공되지 않았다면 원본 입력 이미지를 그대로 반환합니다. |
