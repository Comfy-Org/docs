스위치 노드는 불리언 조건에 따라 두 가지 가능한 입력 중 하나를 선택합니다. 스위치가 활성화되면 `on_true` 입력을 출력하고, 비활성화되면 `on_false` 입력을 출력합니다. 이를 통해 조건부 로직을 만들고 워크플로우에서 서로 다른 데이터 경로를 선택할 수 있습니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|----------|-------|-------------|
| `switch` | BOOLEAN | 예 | | 어떤 입력을 통과시킬지 결정하는 불리언 조건입니다. 활성화된 경우(참) `on_true` 입력이 선택되고, 비활성화된 경우(거짓) `on_false` 입력이 선택됩니다. |
| `on_false` | MATCH_TYPE | 아니요 | | 스위치가 비활성화된 경우(거짓) 출력으로 전달될 데이터입니다. 이 입력은 스위치가 거짓일 때만 필요합니다. |
| `on_true` | MATCH_TYPE | 아니요 | | 스위치가 활성화된 경우(참) 출력으로 전달될 데이터입니다. 이 입력은 스위치가 참일 때만 필요합니다. |

**입력 요구사항 참고:** `on_false`와 `on_true` 입력은 조건부로 필요합니다. 노드는 스위치가 참일 때만 `on_true` 입력을 요청하고, 스위치가 거짓일 때만 `on_false` 입력을 요청합니다. 두 입력은 반드시 동일한 데이터 유형이어야 합니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-------------|-----------|-------------|
| `output` | MATCH_TYPE | 선택된 데이터입니다. 스위치가 참이면 `on_true` 입력의 값이 되고, 스위치가 거짓이면 `on_false` 입력의 값이 됩니다. |
