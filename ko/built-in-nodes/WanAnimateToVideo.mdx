> 이 문서는 AI로 생성되었습니다. 오류를 발견하거나 개선할 점이 있다면 언제든지 기여해 주세요! [GitHub에서 편집하기](https://github.com/Comfy-Org/embedded-docs/blob/main/comfyui_embedded_docs/docs/WanAnimateToVideo/en.md)

WanAnimateToVideo 노드는 포즈 참조, 얼굴 표정, 배경 요소 등 여러 조건부 입력을 결합하여 비디오 콘텐츠를 생성합니다. 다양한 비디오 입력을 처리해 프레임 간 시간적 일관성을 유지하면서 일관성 있는 애니메이션 시퀀스를 만들어냅니다. 이 노드는 잠재 공간 연산을 처리하며, 기존 비디오를 확장해 모션 패턴을 이어갈 수 있습니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `positive` | CONDITIONING | 예 | - | 원하는 콘텐츠로의 생성을 안내하는 긍정적 조건부 입력 |
| `negative` | CONDITIONING | 예 | - | 원치 않는 콘텐츠로부터 생성을 멀리하도록 유도하는 부정적 조건부 입력 |
| `vae` | VAE | 예 | - | 이미지 데이터 인코딩 및 디코딩에 사용되는 VAE 모델 |
| `width` | INT | 예 | 16부터 MAX_RESOLUTION까지 | 출력 비디오의 가로 픽셀 수 (기본값: 832, 단계: 16) |
| `height` | INT | 예 | 16부터 MAX_RESOLUTION까지 | 출력 비디오의 세로 픽셀 수 (기본값: 480, 단계: 16) |
| `length` | INT | 예 | 1부터 MAX_RESOLUTION까지 | 생성할 프레임 수 (기본값: 77, 단계: 4) |
| `batch_size` | INT | 예 | 1부터 4096까지 | 동시에 생성할 비디오 수 (기본값: 1) |
| `clip_vision_output` | CLIP_VISION_OUTPUT | 아니요 | - | 추가적인 조건부 입력을 위한 옵션 CLIP 비전 모델 출력 |
| `reference_image` | IMAGE | 아니요 | - | 생성의 시작점으로 사용되는 참조 이미지 |
| `face_video` | IMAGE | 아니요 | - | 얼굴 표정 안내를 제공하는 비디오 입력 |
| `pose_video` | IMAGE | 아니요 | - | 포즈와 모션 안내를 제공하는 비디오 입력 |
| `continue_motion_max_frames` | INT | 예 | 1부터 MAX_RESOLUTION까지 | 이전 모션에서 이어갈 최대 프레임 수 (기본값: 5, 단계: 4) |
| `background_video` | IMAGE | 아니요 | - | 생성된 콘텐츠와 합성할 배경 비디오 |
| `character_mask` | MASK | 아니요 | - | 캐릭터 영역을 정의해 선택적 처리를 가능하게 하는 마스크 |
| `continue_motion` | IMAGE | 아니요 | - | 시간적 일관성을 위해 이어받을 이전 모션 시퀀스 |
| `video_frame_offset` | INT | 예 | 0부터 MAX_RESOLUTION까지 | 모든 입력 비디오에서 탐색할 프레임 수. 비디오를 더 길게 생성하려면 이전 노드의 video_frame_offset 출력을 연결하세요. (기본값: 0, 단계: 1) |

**매개변수 제약사항:**

- `pose_video`가 제공되면, `trim_to_pose_video` 로직이 활성화된 경우 출력 길이는 포즈 비디오의 지속시간에 맞춰 조정됩니다. (현재 소스 코드에서는 `False`로 설정됨)
- `face_video`는 처리 시 자동으로 512x512 해상도로 리사이즈되고 -1.0에서 1.0 범위로 정규화됩니다.
- `continue_motion` 프레임은 `continue_motion_max_frames` 매개변수에 의해 제한되며, 입력에서 마지막 `continue_motion_max_frames` 프레임만 사용됩니다.
- 입력 비디오들(`face_video`, `pose_video`, `background_video`, `character_mask`)는 처리 전에 `video_frame_offset`만큼 오프셋됩니다. 오프셋이 비디오 길이를 초과하면 입력은 무시됩니다.
- `character_mask`가 한 프레임만 포함된 경우, 모든 프레임에 반복 적용됩니다.
- `clip_vision_output`가 제공되면, 긍정적 및 부정적 조건부 모두에 적용됩니다.
- `reference_image`가 제공되지 않으면 기본값으로 검은색 이미지(모두 0)가 사용됩니다.
- `continue_motion`이 제공되지 않으면 초기 프레임은 회색(밝기 0.5) 노이즈로 채워집니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-----------|-----------|-------------|
| `positive` | CONDITIONING | CLIP 비전 출력, 포즈 비디오 잠재, 얼굴 비디오 픽셀, 연결된 잠재 이미지, 연결된 마스크를 포함한 추가 비디오 컨텍스트와 함께 수정된 긍정적 조건부 입력 |
| `negative` | CONDITIONING | CLIP 비전 출력, 포즈 비디오 잠재, 얼굴 비디오 픽셀(반전), 연결된 잠재 이미지, 연결된 마스크를 포함한 추가 비디오 컨텍스트와 함께 수정된 부정적 조건부 입력 |
| `latent` | LATENT | [batch_size, 16, latent_length + trim_latent, latent_height, latent_width] 형태의 잠재 공간 형식으로 생성된 비디오 콘텐츠 |
| `trim_latent` | INT | 잠재 공간 트리밍 정보로, 처음부터 트리밍할 잠재 프레임 수를 나타냅니다. (참조 이미지 잠재 프레임에 해당) |
| `trim_image` | INT | 참조 모션 프레임의 이미지 공간 트리밍 정보로, 처음부터 트리밍할 이미지 프레임 수를 나타냅니다. |
| `video_frame_offset` | INT | 청크별 비디오 생성을 이어나가기 위한 업데이트된 프레임 오프셋으로, 이전 오프셋에 생성된 길이를 더한 값입니다. |
