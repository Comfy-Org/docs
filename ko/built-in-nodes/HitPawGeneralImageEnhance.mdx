> 이 문서는 AI로 생성되었습니다. 오류를 발견하거나 개선할 점이 있다면 언제든지 기여해 주세요! [GitHub에서 편집하기](https://github.com/Comfy-Org/embedded-docs/blob/main/comfyui_embedded_docs/docs/HitPawGeneralImageEnhance/en.md)

이 노드는 저해상도 이미지를 초고해상도로 업스케일링하여 화질을 향상시키고, 아티팩트와 노이즈를 제거합니다. 이미지를 처리하기 위해 외부 API를 사용하며, 입력 크기를 자동으로 조정하여 처리 한계 내에 유지할 수 있습니다. 최대 허용 출력 크기는 32메가픽셀입니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `model` | 문자열 | 예 | `"generative_portrait"`<br />`"generative"` | 사용할 향상 모델입니다. `generative_portrait` 모델은 인물 사진에 최적화되어 있으며, `generative` 모델은 범용 모델입니다. |
| `image` | 이미지 | 예 | - | 향상시킬 입력 이미지입니다. |
| `upscale_factor` | 정수 | 예 | `1`<br />`2`<br />`4` | 이미지의 크기를 확대하는 배율입니다. 배율 1은 업스케일링을 하지 않는 것을 의미하고, 2는 크기를 두 배로 늘리며, 4는 네 배로 늘립니다. |
| `auto_downscale` | 불리언 | 아니요 | - | 출력 크기가 한계를 초과할 경우 입력 이미지를 자동으로 축소합니다. 활성화되면 노드는 입력 이미지 크기를 줄이거나 업스케일링 배율을 낮춰 32메가픽셀 출력 한계 내에 맞추려고 시도합니다. (기본값: `False`) |

**참고:** 계산된 출력 크기(입력 높이 × 업스케일링 배율 × 입력 너비 × 업스케일링 배율)가 32,000,000픽셀(32MP)을 초과하고 `auto_downscale`가 비활성화된 경우 노드는 오류를 발생시킵니다. `auto_downscale`가 활성화된 경우, 노드는 요청된 업스케일링 배율을 적용하기 전에 입력 이미지를 한계 내에 맞도록 축소하려고 시도합니다. 2배 이상 축소해야 하는 경우, 대신 업스케일링 배율을 낮춥니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `image` | 이미지 | 향상되고 업스케일링된 출력 이미지입니다. |
