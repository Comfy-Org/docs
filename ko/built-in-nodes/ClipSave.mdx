`CLIPSave` 노드는 CLIP 텍스트 인코더 모델을 SafeTensors 형식으로 디스크에 저장합니다. 이 노드는 고급 모델 병합 워크플로우를 위해 설계되었으며, 모델의 내부 구조에 따라 CLIP 모델을 구성 요소별로 자동으로 분리하여 각 구성 요소를 별도의 파일로 저장합니다. (예: CLIP-L, CLIP-G 또는 T5XXL 등)

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `clip` | CLIP | 예 | - | 저장할 CLIP 모델입니다. |
| `filename_prefix` | 문자열 | 예 | - | 저장된 파일의 경로와 파일명 접두어입니다. 노드는 구성 요소 접미사(예: `_clip_l`, `_clip_g`)와 카운터를 추가해 고유한 파일명을 생성합니다(기본값: `clip/ComfyUI`). |
| `prompt` | 프롬프트 | 아니요 | - | 워크플로우 프롬프트 정보로, 출력 파일에 메타데이터로 저장됩니다. 이 매개변수는 UI에서 숨겨져 있습니다. |
| `extra_pnginfo` | EXTRA_PNGINFO | 아니요 | - | 추가 메타데이터로, 출력 파일에 키-값 쌍으로 저장됩니다. 이 매개변수는 UI에서 숨겨져 있습니다. |

## 출력

이 노드에는 출력 연결이 없습니다. 처리된 파일은 바로 `ComfyUI/output/` 디렉토리에 저장됩니다.

### 저장된 파일 세부정보

노드는 CLIP 모델의 상태 사전을 분석해 감지된 각 구성 요소에 대해 별도의 SafeTensors 파일을 저장합니다. 구성 요소는 매개변수 키의 접두어로 식별됩니다. 다음 접두어가 확인됩니다:

- `clip_l.` (CLIP-L 텍스트 인코더)
- `clip_g.` (CLIP-G 텍스트 인코더)
- `clip_h.` (CLIP-H 텍스트 인코더)
- `t5xxl.` (T5-XXL 텍스트 인코더)
- `pile_t5xl.` (Pile-T5-XL 텍스트 인코더)
- `mt5xl.` (mT5-XL 텍스트 인코더)
- `umt5xxl.` (UMT5-XXL 텍스트 인코더)
- `t5base.` (T5-Base 텍스트 인코더)
- `gemma2_2b.` (Gemma 2 2B 텍스트 인코더)
- `llama.` (LLaMA 텍스트 인코더)
- `hydit_clip.` (Hydit CLIP 텍스트 인코더)
- 빈 접두어 (기타 CLIP 구성 요소)

감지된 각 구성 요소에 대해 노드는 `{filename_prefix}_{counter:05}_.safetensors`라는 이름의 파일을 생성하며, 여기서 구성 요소 접두어가 파일명 접두어에 추가됩니다(예: `clip/ComfyUI_clip_l_00001_.safetensors`). 저장 시 매개변수 키에서 `transformer.` 접두어는 제거됩니다.
