SDPoseKeypointExtractor 노드는 SDPose 모델을 사용하여 입력 이미지에서 사람의 자세 키포인트를 감지합니다. 전체 이미지 또는 바운딩 박스로 정의된 특정 영역을 처리할 수 있으며, 감지된 키포인트를 OpenPose 형식으로 출력합니다. 이 형식에는 각 사람의 좌표와 각 키포인트에 대한 신뢰도 점수가 포함됩니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `model` | MODEL | 예 | - | 키포인트 감지를 위해 사용하는 SDPose 모델입니다. 반드시 `heatmap_head` 속성을 가진 모델로, 특히 SDPose 저장소의 모델이어야 합니다. |
| `vae` | VAE | 예 | - | 입력 이미지를 잠재 공간으로 인코딩하여 처리하는 데 사용되는 VAE 모델입니다. |
| `image` | IMAGE | 예 | - | 자세 키포인트를 추출할 입력 이미지 또는 이미지 배치입니다. |
| `batch_size` | INT | 아니요 | 1~10000 | 전체 이미지 모드(즉, `bboxes`가 제공되지 않은 경우)에서 한 번에 처리할 이미지 수입니다. 이 옵션은 처리 속도를 높일 수 있습니다. (기본값: 16) |
| `bboxes` | BOUNDINGBOX | 아니요 | - | 보다 정확한 감지를 위한 선택적 바운딩 박스입니다. 다인종 감지를 위해서는 필수입니다. 제공되면 노드는 지정된 각 영역에서 키포인트를 추출합니다. |

**매개변수 제약사항:**
*   `model` 입력은 특정 SDPose 모델이어야 합니다. 제공된 모델에 `heatmap_head` 속성이 없으면 노드는 오류를 발생시킵니다.
*   노드는 `bboxes` 입력에 따라 두 가지 다른 모드로 작동합니다:
    1.  **바운딩 박스 모드:** `bboxes`가 제공되면 각 지정된 영역을 개별적으로 처리합니다. 이는 하나의 이미지에서 여러 사람을 감지하는 데 필요합니다.
    2.  **전체 이미지 모드:** `bboxes`가 제공되지 않으면 전체 이미지를 배치로 처리합니다. `batch_size` 매개변수는 이 모드에서만 적용됩니다.
*   `bboxes`가 제공될 경우, 반드시 바운딩 박스 목록이어야 합니다. 이미지 수보다 바운딩 박스 목록이 적다면, 마지막 목록을 나머지 이미지에 재사용합니다.
*   너비나 높이가 0인 바운딩 박스(예: `x2 <= x1` 또는 `y2 <= y1`)는 건너뜁니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `keypoints` | POSE_KEYPOINT | OpenPose 프레임 형식의 키포인트(캔버스 너비, 캔버스 높이, 사람 수). 출력에는 감지된 사람마다 키포인트 좌표(x, y)와 해당 신뢰도 점수가 배열로 포함됩니다. |
