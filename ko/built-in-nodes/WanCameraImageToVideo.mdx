> 이 문서는 AI로 생성되었습니다. 오류를 발견하거나 개선할 점이 있다면 언제든지 기여해 주세요! [GitHub에서 편집하기](https://github.com/Comfy-Org/embedded-docs/blob/main/comfyui_embedded_docs/docs/WanCameraImageToVideo/en.md)

WanCameraImageToVideo 노드는 이미지로부터 비디오 생성을 위한 조건부 데이터와 잠재적 데이터를 준비합니다. 긍정적 및 부정적 조건부 프롬프트와 선택적 시작 이미지 및 카메라 제어를 입력받아 수정된 조건부 데이터와 비디오 모델이 채워 넣을 수 있는 빈 잠재적 텐서를 출력합니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|----------|-------|-------------|
| `positive` | CONDITIONING | 예 | - | 비디오 생성을 위한 긍정적 조건부 프롬프트 |
| `negative` | CONDITIONING | 예 | - | 비디오 생성 시 피해야 할 부정적 조건부 프롬프트 |
| `vae` | VAE | 예 | - | 이미지를 잠재 공간으로 인코딩하는 VAE 모델 |
| `width` | INT | 예 | 16부터 MAX_RESOLUTION까지 | 출력 비디오 너비(픽셀 단위, 기본값: 832, 증분: 16) |
| `height` | INT | 예 | 16부터 MAX_RESOLUTION까지 | 출력 비디오 높이(픽셀 단위, 기본값: 480, 증분: 16) |
| `length` | INT | 예 | 1부터 MAX_RESOLUTION까지 | 비디오 시퀀스의 프레임 수(기본값: 81, 증분: 4) |
| `batch_size` | INT | 예 | 1부터 4096까지 | 동시에 생성할 비디오 수(기본값: 1) |
| `clip_vision_output` | CLIP_VISION_OUTPUT | 아니요 | - | 추가적인 조건부 데이터를 위한 옵션 CLIP 비전 출력 |
| `start_image` | IMAGE | 아니요 | - | 비디오 시퀀스를 초기화할 선택적 시작 이미지입니다. 제공될 경우, 비디오의 첫 프레임은 이 이미지를 기반으로 하며, 마스크를 적용해 시작 프레임과 생성된 내용을 혼합합니다. 이미지는 지정된 너비와 높이에 맞게 크기가 조정됩니다. |
| `camera_conditions` | WAN_CAMERA_EMBEDDING | 아니요 | - | 비디오 생성을 위한 선택적 카메라 임베딩 조건입니다. 제공될 경우, 이 조건들은 긍정적 및 부정적 조건부 모두에 적용됩니다. |

**참고:** `start_image`가 제공되면 노드는 이를 사용해 비디오 시퀀스를 초기화하고 마스크를 적용해 시작 프레임과 생성된 내용을 혼합합니다. `camera_conditions`와 `clip_vision_output` 매개변수는 선택적이지만, 제공될 경우 긍정적 및 부정적 프롬프트의 조건부를 수정합니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-------------|-----------|-------------|
| `positive` | CONDITIONING | 카메라 조건, CLIP 비전 출력 및/또는 시작 이미지 데이터가 적용된 수정된 긍정적 조건부 |
| `negative` | CONDITIONING | 카메라 조건, CLIP 비전 출력 및/또는 시작 이미지 데이터가 적용된 수정된 부정적 조건부 |
| `latent` | LATENT | 비디오 모델과 함께 사용할 빈 비디오 잠재적 표현입니다. 잠재적 텐서의 차원은 [batch_size, 16, frames, height/8, width/8]이며, 여기서 frames는 ((length - 1) // 4) + 1로 계산됩니다. |
