# SaveImageAdvanced

**SaveImageAdvanced** 노드는 이미지를 ComfyUI 출력 디렉터리에 저장하며, 파일 형식, 비트 깊이 및 색 공간에 대한 고급 제어 기능을 제공합니다. PNG 또는 EXR 파일로 저장을 지원하며, 저장된 파일에 워크플로우 메타데이터를 포함시킬 수 있습니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `images` | IMAGE | 예 | - | 저장할 이미지들입니다. |
| `filename_prefix` | STRING | 예 | - | 저장할 파일의 접두어입니다. `%date:yyyy-MM-dd%`나 `%Empty Latent Image.width%`와 같은 서식 토큰을 포함할 수 있습니다. (기본값: "ComfyUI") |
| `format` | COMBO | 예 | `"png"`<br />`"exr"` | 이미지를 저장할 파일 형식입니다. 형식을 선택하면 해당 형식에 대한 추가 옵션이 표시됩니다. |
| `bit_depth` | COMBO | 조건부로 예 | PNG의 경우: `"8-bit"`<br />`"16-bit"`<br />EXR의 경우: `"32-bit float"` | 선택한 형식의 비트 깊이입니다. 형식을 선택하면 이 매개변수가 나타납니다. (기본값: PNG의 경우 "8-bit", EXR의 경우 "32-bit float") |
| `input_color_space` | COMBO | 조건부로 예 | PNG의 경우: `"sRGB"`<br />EXR의 경우: `"sRGB"`<br />`"HDR"`<br />`"linear"` | 입력 텐서의 색공간입니다. PNG의 경우 sRGB만 사용 가능합니다. EXR의 경우 이미지는 항상 일치하는 감마에서 장면 선형으로 작성됩니다. (기본값: "sRGB") |

**매개변수 의존성 참고사항:**
- `bit_depth`와 `input_color_space` 매개변수는 특정 `format`이 선택되었을 때만 사용 가능합니다.
- PNG 형식의 경우, "8-bit"와 "16-bit" 비트 깊이만 사용 가능하며, 색공간은 "sRGB"만 가능합니다.
- EXR 형식의 경우, "32-bit float" 비트 깊이만 사용 가능하며, 색공간은 "sRGB", "HDR" 또는 "linear" 중 하나입니다.
- EXR의 `input_color_space` 매개변수는 입력 텐서를 어떻게 해석할지 결정합니다:
  - `"sRGB"` — 입력은 sRGB 인코딩 Rec.709; 역 sRGB EOTF가 적용됩니다.
  - `"HDR"` — 입력은 HLG 인코딩 Rec.2020 (BT.2100); 역 HLG OETF가 적용되어 장면 선형 빛을 얻습니다.
  - `"linear"` — 입력은 이미 장면 선형(Rec.709 원색) 상태이며, 변경 없이 그대로 쓰입니다. 렌더러/합성기 출력용으로 사용하세요.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `images` | IMAGE | 저장된 이미지 결과의 리스트로, 각 이미지에는 파일명, 하위 폴더 및 유형("output")이 포함됩니다. 이 출력은 UI 표시 용도로 사용됩니다. |
