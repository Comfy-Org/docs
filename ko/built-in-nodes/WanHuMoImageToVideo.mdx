> 이 문서는 AI로 생성되었습니다. 오류를 발견하거나 개선할 점이 있으면 언제든지 기여해 주세요! [GitHub에서 편집하기](https://github.com/Comfy-Org/embedded-docs/blob/main/comfyui_embedded_docs/docs/WanHuMoImageToVideo/en.md)

WanHuMoImageToVideo 노드는 이미지를 비디오 프레임의 잠재적 표현을 생성하여 비디오 시퀀스로 변환합니다. 이 노드는 조건부 입력을 처리하며, 참조 이미지와 오디오 임베딩을 포함시켜 비디오 생성에 영향을 줄 수 있습니다. 노드는 수정된 조건부 데이터와 비디오 합성에 적합한 잠재적 표현을 출력합니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `positive` | CONDITIONING | 예 | - | 비디오 생성을 원하는 콘텐츠로 유도하는 긍정적 조건부 입력 |
| `negative` | CONDITIONING | 예 | - | 비디오 생성을 원치 않는 콘텐츠로부터 멀어지게 하는 부정적 조건부 입력 |
| `vae` | VAE | 예 | - | 참조 이미지를 잠재 공간으로 인코딩하는 데 사용되는 VAE 모델 |
| `width` | INT | 예 | 16부터 MAX_RESOLUTION까지 | 출력 비디오 프레임의 너비(픽셀 단위) (기본값: 832, 16의 배수여야 함) |
| `height` | INT | 예 | 16부터 MAX_RESOLUTION까지 | 출력 비디오 프레임의 높이(픽셀 단위) (기본값: 480, 16의 배수여야 함) |
| `length` | INT | 예 | 1부터 MAX_RESOLUTION까지 | 생성된 비디오 시퀀스의 프레임 수 (기본값: 97, (length - 1)이 4로 나누어져야 함) |
| `batch_size` | INT | 예 | 1부터 4096까지 | 동시에 생성할 비디오 시퀀스의 수 (기본값: 1) |
| `audio_encoder_output` | AUDIOENCODEROUTPUT | 선택 | - | 오디오 콘텐츠에 따라 비디오 생성에 영향을 줄 수 있는 옵션 오디오 인코딩 데이터 |
| `ref_image` | IMAGE | 선택 | - | 비디오 생성 스타일과 콘텐츠를 유도하는 데 사용되는 옵션 참조 이미지 |

**참고:** 참조 이미지를 제공하면 해당 이미지가 인코딩되어 긍정적 및 부정적 조건부 모두에 추가됩니다. 오디오 인코더 출력을 제공하면 이를 처리하여 조건부 데이터에 통합합니다. 둘 중 하나도 제공되지 않으면 참조 잠재와 오디오 임베딩 모두에 0으로 채워진 자리표시 텐서가 사용됩니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-------------|-----------|-------------|
| `positive` | CONDITIONING | 참조 이미지 및/또는 오디오 임베딩이 통합된 수정된 긍정적 조건부 |
| `negative` | CONDITIONING | 참조 이미지 및/또는 오디오 임베딩이 통합된 수정된 부정적 조건부 |
| `latent` | LATENT | 비디오 시퀀스 데이터를 포함하는 생성된 잠재적 표현 |
