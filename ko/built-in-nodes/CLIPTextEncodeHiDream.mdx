CLIPTextEncodeHiDream 노드는 네 가지 별도의 텍스트 입력을 서로 다른 언어 모델(CLIP-L, CLIP-G, T5-XXL, LLaMA)을 사용해 처리한 후 이를 하나의 조건부 출력으로 결합합니다. 각 텍스트 입력을 해당 모델로 토큰화하고, 스케줄된 인코딩 방식을 통해 함께 인코딩하여 여러 언어 모델을 동시에 활용함으로써 보다 정교한 텍스트 조건부 처리가 가능합니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `clip` | CLIP | 예 | - | 토큰화와 인코딩에 사용되는 CLIP 모델 |
| `clip_l` | STRING | 예 | - | CLIP-L 모델 처리용 텍스트 입력. 다중 줄 텍스트 및 동적 프롬프트 지원 |
| `clip_g` | STRING | 예 | - | CLIP-G 모델 처리용 텍스트 입력. 다중 줄 텍스트 및 동적 프롬프트 지원 |
| `t5xxl` | STRING | 예 | - | T5-XXL 모델 처리용 텍스트 입력. 다중 줄 텍스트 및 동적 프롬프트 지원 |
| `llama` | STRING | 예 | - | LLaMA 모델 처리용 텍스트 입력. 다중 줄 텍스트 및 동적 프롬프트 지원 |

**참고:** 네 가지 텍스트 입력(`clip_l`, `clip_g`, `t5xxl`, `llama`)은 모두 올바른 작동을 위해 필수이며, 각 입력은 스케줄된 인코딩 과정을 통해 최종 조건부 출력에 기여합니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-----------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | 처리된 모든 텍스트 입력을 결합한 조건부 출력으로, 스케줄된 인코딩 방식을 사용해 인코딩됨 |
