**GLSL Shader** 노드를 사용하면 **GLSL ES 3.00**(WebGL 2.0 호환)로 맞춤형 프래그먼트 셰이더를 작성하여 GPU에서 직접 이미지를 처리할 수 있습니다. 블러, 색상 그레이딩, 필름 그레인, 글로우 등 다양한 이미지 효과를 GPU 속도로 구현할 수 있습니다.

<Note>
GLSL Shader 노드는 현재 **실험적**으로 표시되어 있으며, 향후 릴리스에서 노드가 업데이트되고 확장될 수 있습니다.
</Note>

## 최소한의 셰이더

가장 간단한 셰이더 - 입력 이미지를 그대로 출력하는 패스스루:

```glsl
#version 300 es
precision highp float;

uniform sampler2D u_image0;

in vec2 v_texCoord;
layout(location = 0) out vec4 fragColor0;

void main() {
    fragColor0 = texture(u_image0, v_texCoord);
}
```

<Note>
**왜 GLSL ES 3.00인가요?** 셰이더는 두 환경에서 실행되어야 합니다: ComfyUI 프론트엔드에서 실시간 미리보기를 위해 **브라우저**(WebGL 2.0을 통해 GLSL ES 3.00만 지원)와 워크플로우가 실행될 때 **파이썬 백엔드**(데스크톱 OpenGL을 통해)입니다. GLSL ES 3.00은 두 곳에서 모두 작동하는 공통된 규격입니다.
</Note>

## 사용 가능한 유니폼

이 유니폼들은 ComfyUI에서 자동으로 설정됩니다. 모든 유니폼을 선언할 필요는 없으며, 사용하는 것만 선언하면 됩니다.

### 이미지

| 유니폼 | 타입 | 설명 |
|---------|------|-------------|
| `u_image0` – `u_image4` | `sampler2D` | 입력 이미지(최대 5개). `texture(u_image0, v_texCoord)`로 샘플링합니다. 이미지는 RGBA 부동소수점 텍스처이며, 선형 필터링과 가장자리 클램프 방식으로 반복됩니다. |

### 부동소수점

| 유니폼 | 타입 | 설명 |
|---------|------|-------------|
| `u_float0` – `u_float19` | `float` | 최대 20개의 사용자가 제어하는 부동소수점 값. 노드의 **floats** 입력 그룹에서 매핑됩니다. |

### 정수

| 유니폼 | 타입 | 설명 |
|---------|------|-------------|
| `u_int0` – `u_int19` | `int` | 최대 20개의 사용자가 제어하는 정수 값. 노드의 **ints** 입력 그룹에서 매핑됩니다. |

<Note>
**정수 유니폼을 드롭다운으로 사용하기:** 정수 유니폼은 **Custom Combo** 노드의 인덱스 출력과 잘 어울립니다. 사용자가 드롭다운에서 옵션을 선택하면 셰이더가 선택된 항목의 인덱스를 받습니다.

```glsl
const int BLEND_SCREEN  = 0;
const int BLEND_OVERLAY = 1;
const int BLEND_MULTIPLY = 2;

// ...

if (u_int0 == BLEND_SCREEN) {
    // ...
}
```
</Note>

### 불린

| 유니폼 | 타입 | 설명 |
|---------|------|-------------|
| `u_bool0` – `u_bool9` | `bool` | 최대 10개의 사용자가 제어하는 불린 값. 노드의 **bools** 입력 그룹에서 매핑됩니다. |

### 커브(1D LUT)

| 유니폼 | 타입 | 설명 |
|---------|------|-------------|
| `u_curve0` – `u_curve3` | `sampler2D` | 최대 4개의 사용자가 편집 가능한 커브 LUT. 각 커브는 1차원 조회 테이블로 저장된 단일 행 텍스처입니다. |

<Note>
**커브 유니폼 사용하기:** 커브를 사용하면 UI에서 임의의 톤 맵핑 그래프를 그릴 수 있습니다(예: 대비, 감마, 채널별 그레이딩 또는 사용자가 직접 정의한 `입력 → 출력` 재매핑). 입력값을 X 좌표로 사용해 커브를 샘플링하세요. 먼저 `[0, 1]` 범위로 클램프하는 것을 잊지 마세요:

```glsl
float applyCurve(sampler2D curve, float value) {
    return texture(curve, vec2(clamp(value, 0.0, 1.0), 0.5)).r;
}

// 사용 예: 각 RGB 채널을 마스터 커브를 통해 재매핑
color.r = applyCurve(u_curve0, color.r);
color.g = applyCurve(u_curve0, color.g);
color.b = applyCurve(u_curve0, color.b);
```

일반적인 사용례: 마스터 RGB 커브, 채널별 R/G/B 커브, 휘도 기반 재매핑, 맞춤 감마, 그리고 사용자가 시각적으로 응답 곡선을 조정하고 싶은 모든 효과.
</Note>

### 해상도

| 유니폼 | 타입 | 설명 |
|---------|------|-------------|
| `u_resolution` | `vec2` | **출력** 프레임버퍼의 픽셀 단위 크기(`width, height`). 이 크기는 작성되는 크기로, `size_mode`가 `"custom"`일 경우 입력 이미지의 크기와 다를 수 있습니다. |

<Note>
**샘플링을 위한 텍셀 크기 계산하기:** 입력 텍스처에서 한 픽셀을 이동하려면 `1.0 / u_resolution`을 사용하지 마세요. `u_resolution`은 *출력* 크기로, 입력 크기와 일치하지 않을 수 있습니다. 대신 실제로 샘플링하는 텍스처에서 `textureSize()`를 사용하세요:

```glsl
vec2 texel = 1.0 / vec2(textureSize(u_image0, 0));
```

`u_resolution`은 출력 프레임버퍼의 크기 자체를 필요로 할 때만 사용하세요(예: `gl_FragCoord.xy / u_resolution`을 사용해 스크린 공간 UV를 계산하는 경우).
</Note>

### 멀티패스

| 유니폼 | 타입 | 설명 |
|---------|------|-------------|
| `u_pass` | `int` | 현재 패스 인덱스(0부터 시작). `#pragma passes`를 사용할 때만 의미가 있습니다. 자세한 내용은 [멀티패스 핑퐁 렌더링](#멀티패스-핑퐁-렌더링) 참조. |

### 버텍스 셰이더 출력

| 바이어링 | 타입 | 설명 |
|---------|------|-------------|
| `v_texCoord` | `vec2` | 왼쪽 아래가 (0,0)이고 오른쪽 위가 (1,1)인 텍스처 좌표. |

## 여러 출력(MRT)

노드는 최대 **4개의 동시 출력**을 지원하며, 여러 렌더 타겟을 사용합니다. 추가 출력을 명시적인 위치로 선언하세요:

```glsl
#version 300 es
precision highp float;

uniform sampler2D u_image0;

in vec2 v_texCoord;
layout(location = 0) out vec4 fragColor0;
layout(location = 1) out vec4 fragColor1;
layout(location = 2) out vec4 fragColor2;
layout(location = 3) out vec4 fragColor3;

void main() {
    vec4 color = texture(u_image0, v_texCoord);
    fragColor0 = vec4(vec3(color.r), 1.0);  // 빨간색 채널
    fragColor1 = vec4(vec3(color.g), 1.0);  // 초록색 채널
    fragColor2 = vec4(vec3(color.b), 1.0);  // 파란색 채널
    fragColor3 = vec4(vec3(color.a), 1.0);  // 알파 채널
}
```

각 `fragColor`는 노드의 해당 `IMAGE` 출력에 매핑됩니다. ComfyUI는 사용자가 어떤 출력을 사용하는지 자동으로 감지하며, 사용되지 않는 출력은 검은색으로 채워집니다.

## 멀티패스 핑퐁 렌더링

분리 가능한 블러와 같은 일부 효과는 이미지를 여러 번 통과해야 합니다. `#pragma passes N` 지시문을 사용해 이를 활성화하세요:

```glsl
#version 300 es
#pragma passes 2
precision highp float;

uniform sampler2D u_image0;
uniform float u_float0;  // 블러 반경
uniform int u_pass;

in vec2 v_texCoord;
layout(location = 0) out vec4 fragColor0;

void main() {
    vec2 texel = 1.0 / vec2(textureSize(u_image0, 0));
    int radius = int(ceil(u_float0));

    // 패스 0 = 수평 블러, 패스 1 = 수직 블러
    vec2 dir = (u_pass == 0) ? vec2(1.0, 0.0) : vec2(0.0, 1.0);

    vec4 color = vec4(0.0);
    float total = 0.0;

    for (int i = -radius; i <= radius; i++) {
        vec2 offset = dir * float(i) * texel;
        float w = 1.0;  // 박스 블러 가중치
        color += texture(u_image0, v_texCoord + offset) * w;
        total += w;
    }

    fragColor0 = color / total;
}
```

### 핑퐁 작동 방식

1. **패스 0**: 원본 `u_image0` 입력을 읽고 내부 핑퐁 텍스처에 씁니다.
2. **패스 1–N**: `u_image0`를 통해 *이전 패스 출력*을 읽고 다른 핑퐁 텍스처에 씁니다.
3. **최종 패스**: 실제 출력 프레임버퍼(`fragColor0`)에 씁니다.

<Note>
MRT(멀티 출력)를 사용하는 멀티패스에서는 첫 번째 출력(`fragColor0`)만 핑퐁에 참여합니다. 최종 패스에서는 모든 출력을 씁니다.
</Note>

## 예제

### 회색조 변환

```glsl
#version 300 es
precision highp float;

uniform sampler2D u_image0;
in vec2 v_texCoord;
layout(location = 0) out vec4 fragColor0;

void main() {
    vec4 color = texture(u_image0, v_texCoord);
    float gray = dot(color.rgb, vec3(0.2126, 0.7152, 0.0722));
    fragColor0 = vec4(vec3(gray), color.a);
}
```

### 이미지 블렌딩

두 입력 이미지를 부동소수점 매개변수를 사용해 블렌딩합니다:

```glsl
#version 300 es
precision highp float;

uniform sampler2D u_image0;
uniform sampler2D u_image1;
uniform float u_float0;  // 블렌딩 요소 [0.0 – 1.0]

in vec2 v_texCoord;
layout(location = 0) out vec4 fragColor0;

void main() {
    vec4 a = texture(u_image0, v_texCoord);
    vec4 b = texture(u_image1, v_texCoord);
    fragColor0 = mix(a, b, clamp(u_float0, 0.0, 1.0));
}
```

## LLM을 이용한 셰이더 생성

어떤 LLM(Claude, ChatGPT 등)을 사용해도 GLSL 셰이더를 작성할 수 있습니다. 다음 프롬프트를 복사해 원하는 효과를 입력하세요:

````markdown
ComfyUI의 GLSLShader 노드용 GLSL ES 3.00 프래그먼트 셰이더를 작성하세요.

**원하는 효과:** [원하는 효과를 여기에 설명하세요]

**요구사항:**
- `#version 300 es`로 시작해야 합니다.
- `precision highp float;`를 사용하세요.
- 버텍스 셰이더는 `in vec2 v_texCoord`를 제공합니다(0~1 UV 좌표, 왼쪽 아래 원점).
- 출력은 `layout(location = 0) out vec4 fragColor0`(RGBA)로 합니다.
- 추가 출력 가능: `fragColor1`, `fragColor2`, `fragColor3` 위치 1~3

**따르는 템플릿(최소 패스스루 셰이더 - 이 구조를 그대로 사용하세요):**

```glsl
#version 300 es
precision highp float;

uniform sampler2D u_image0;

in vec2 v_texCoord;
layout(location = 0) out vec4 fragColor0;

void main() {
    fragColor0 = texture(u_image0, v_texCoord);
}
```

**사용 가능한 유니폼(사용하는 것만 선언하세요):**
- `uniform sampler2D u_image0`부터 `u_image4`까지 - 최대 5개의 입력 이미지(RGBA 부동소수점, 선형 필터링, 가장자리 클램프)
- `uniform vec2 u_resolution` - **출력 프레임버퍼**의 너비와 높이(픽셀 단위). 이는 입력 텍스처 크기가 아닙니다(사용자가 출력 크기를 맞춤 설정할 경우 다를 수 있음). **입력 텍스처에서 한 픽셀을 이동하려면 `vec2 texel = 1.0 / vec2(textureSize(u_image0, 0));`를 사용하세요 - 이때 `1.0 / u_resolution`을 사용하지 마세요.**
- `uniform float u_float0`부터 `u_float19`까지 - 최대 20개의 사용자가 제어하는 부동소수점 매개변수
- `uniform int u_int0`부터 `u_int19`까지 - 최대 20개의 사용자가 제어하는 정수 매개변수. 팁: 정수 유니폼은 **Custom Combo** 노드의 인덱스 출력에서 연결할 수 있어 사용자가 드롭다운에서 선택하면 셰이더가 인덱스를 받습니다. 모드 선택(예: 블렌드 모드, 블러 유형)에 사용하세요. 각 옵션에 이름 있는 상수를 정의하고 분기문을 사용하세요 - 원시 정수 리터럴과 비교하지 마세요. 예: `const int BLUR_GAUSSIAN = 0; const int BLUR_BOX = 1; ... if (u_int0 == BLUR_GAUSSIAN) { ... }`.
- `uniform bool u_bool0`부터 `u_bool9`까지 - 최대 10개의 사용자가 제어하는 불린 매개변수(기능 토글에 사용)
- `uniform sampler2D u_curve0`부터 `u_curve3`까지 - 최대 4개의 사용자가 편집 가능한 1D LUT 커브. `texture(u_curve0, vec2(clamp(x, 0.0, 1.0), 0.5)).r`로 샘플링하며, `x`는 0~1입니다. 톤 커브, 재매핑 등에 사용하세요.
- `uniform int u_pass` - 현재 패스 인덱스(멀티패스 사용 시)

**멀티패스 렌더링:**
- 두 번째 줄에 `#pragma passes N`을 추가해 N개의 패스를 활성화하세요.
- 패스 0에서는 `u_image0`가 원본 입력이고, 이후 패스에서는 이전 패스 출력을 포함합니다.
- `u_pass`를 사용해 패스마다 동작을 다르게 하세요(예: 수평 vs 수직 블러).

**중요한 제약사항:**
- GLSL ES 3.00만 사용 - GLSL 1.x `varying`/`attribute`, `gl_FragColor` 사용 불가
- `#include` 불가, 외부 텍스처 불가, 맞춤형 버텍스 셰이더 불가
- 각 유니폼에 목적과 예상 범위를 주석으로 설명하세요
````

### 예제 프롬프트 채우기
