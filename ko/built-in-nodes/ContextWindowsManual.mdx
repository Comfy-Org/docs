Context Windows (Manual) 노드를 사용하면 샘플링 중에 모델의 컨텍스트 윈도우를 수동으로 구성할 수 있습니다. 이 노드는 지정된 길이, 겹침 및 스케줄링 패턴을 가진 오버랩되는 컨텍스트 세그먼트를 만들어 데이터를 관리 가능한 조각으로 처리하면서 세그먼트 간 연속성을 유지합니다. 또한 이 노드는 노이즈 섞기, 조건부 유지, 인과적 윈도우 수정 등 컨텍스트 윈도우 적용 방식을 제어하는 고급 옵션을 제공합니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|----------|-------|-------------|
| `model` | MODEL | 예 | - | 샘플링 중에 컨텍스트 윈도우를 적용할 모델입니다. |
| `context_length` | INT | 아니요 | 1 이상 | 컨텍스트 윈도우의 길이(기본값: 16). |
| `context_overlap` | INT | 아니요 | 0 이상 | 컨텍스트 윈도우의 겹침(기본값: 4). |
| `context_schedule` | COMBO | 아니요 | `STATIC_STANDARD`<br />`UNIFORM_STANDARD`<br />`UNIFORM_LOOPED`<br />`BATCHED` | 컨텍스트 윈도우의 스트라이드입니다. |
| `context_stride` | INT | 아니요 | 1 이상 | 컨텍스트 윈도우의 스트라이드; 균일한 스케줄에만 적용됩니다(기본값: 1). |
| `closed_loop` | BOOLEAN | 아니요 | - | 컨텍스트 윈도우 루프를 닫을지 여부; 루프 스케줄에만 적용됩니다(기본값: False). |
| `fuse_method` | COMBO | 아니요 | `PYRAMID`<br />`LIST_STATIC` | 컨텍스트 윈도우를 퓨전하는 데 사용할 방법(기본값: PYRAMID). |
| `dim` | INT | 아니요 | 0~5 | 컨텍스트 윈도우를 적용할 차원(기본값: 0). |
| `freenoise` | BOOLEAN | 아니요 | - | FreeNoise 노이즈 섞기를 적용할지 여부, 윈도우 블렌딩을 개선합니다(기본값: False). |
| `cond_retain_index_list` | STRING | 아니요 | - | 각 윈도우의 조건부 텐서에서 유지할 잠재적 인덱스 목록입니다. 예를 들어 '0'으로 설정하면 각 윈도우마다 초기 시작 이미지를 사용합니다(기본값: ""). |
| `split_conds_to_windows` | BOOLEAN | 아니요 | - | ConditionCombine로 생성된 여러 조건부를 각 윈도우의 지역 인덱스에 따라 분할할지 여부(기본값: False). |
| `causal_window_fix` | BOOLEAN | 아니요 | - | 0이 아닌 인덱스를 가진 컨텍스트 윈도우에 인과적 수정 프레임을 추가할지 여부(기본값: True). |

**매개변수 제약사항:**

- `context_stride`는 균일한 스케줄이 선택된 경우에만 사용됩니다.
- `closed_loop`는 루프 스케줄에만 적용됩니다.
- `dim`은 0부터 5까지 포함된 값이어야 합니다.
- `cond_retain_index_list`는 쉼표로 구분된 정수 인덱스 목록을 문자열로 받아들입니다(예: "0,1,2").

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-------------|-----------|-------------|
| `model` | MODEL | 샘플링 중에 컨텍스트 윈도우가 적용된 모델입니다. |
