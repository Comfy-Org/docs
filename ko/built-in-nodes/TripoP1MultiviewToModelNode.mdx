## 개요

이 노드는 객체나 캐릭터의 2~4개의 참조 이미지로부터 3D 모델을 생성합니다. 서로 다른 각도(전면, 좌측, 후면, 우측)의 이미지를 제공하면 노드는 GLB 형식의 3D 메시를 만들어냅니다. 전면 보기(0°)는 반드시 필요하며, 더 나은 결과를 위해 선택적으로 나머지 세 가지 보기 중 어떤 조합을 추가할 수 있습니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `image` | IMAGE | 예 | - | 전면 보기(0°). 필수. |
| `image_left` | IMAGE | 아니요 | - | 좌측 보기(90°), 즉 피사체의 왼쪽 면. |
| `image_back` | IMAGE | 아니요 | - | 후면 보기(180°). |
| `image_right` | IMAGE | 아니요 | - | 우측 보기(270°), 즉 피사체의 오른쪽 면. |
| `output_mode` | COMBO | 예 | `"geometry"`<br />`"textured"`<br />`"detailed"` | 생성된 모델의 출력 모드. `"geometry"`는 원시 메시를 생성하고, `"textured"`는 표준 텍스처를 추가하며, `"detailed"`는 고해상도 텍스처 모델을 생성합니다(기본값: `"textured"`). |
| `face_limit` | INT | 아니요 | -1부터 100000까지 | 출력 메시의 최대 면 수. 한계 없음(-1)으로 설정 가능(기본값: -1). |
| `model_seed` | INT | 아니요 | 0부터 2147483647까지 | 재현 가능한 모델 생성을 위한 시드값. 무작위로 설정하려면 0으로 설정하세요(기본값: 0). |
| `auto_size` | BOOLEAN | 아니요 | True / False | 모델을 표준 경계 박스에 맞게 자동으로 크기를 조정합니다(기본값: False). |
| `export_uv` | BOOLEAN | 아니요 | True / False | 모델과 함께 UV 좌표를 내보내기(기본값: True). |
| `compress_geometry` | BOOLEAN | 아니요 | True / False | 파일 크기를 줄이기 위해 기하학적 데이터를 압축합니다(기본값: False). |

**참고:** 최소 2개의 이미지를 제공해야 합니다: 전면 보기(`image`)와 다른 보기 중 적어도 하나(`image_left`, `image_back`, 또는 `image_right`). 2개 미만의 이미지를 제공하면 노드는 오류를 발생시킵니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `model_file` | STRING | 생성된 GLB 모델의 파일명(후방 호환성을 위해만 사용됨). |
| `model_task_id` | MODEL_TASK_ID | 이 모델 생성 요청의 고유한 작업 ID. |
| `GLB` | FILE3DGLB | GLB 형식으로 생성된 3D 모델. |
