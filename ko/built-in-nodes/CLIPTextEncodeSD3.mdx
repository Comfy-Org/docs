CLIPTextEncodeSD3 노드는 여러 텍스트 프롬프트를 서로 다른 CLIP 모델을 사용해 인코딩함으로써 Stable Diffusion 3 모델용 텍스트 입력을 처리합니다. 이 노드는 세 가지 별도의 텍스트 입력(clip_g, clip_l, t5xxl)을 처리하며, 빈 텍스트 패딩을 관리하는 옵션을 제공합니다. 노드는 서로 다른 텍스트 입력 간의 적절한 토큰 정렬을 보장하고 SD3 생성 파이프라인에 적합한 컨디셔닝 데이터를 반환합니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `clip` | CLIP | 예 | - | 텍스트 인코딩에 사용되는 CLIP 모델 |
| `clip_l` | STRING | 예 | - | 로컬 CLIP 모델용 텍스트 입력. 다중 줄 텍스트와 동적 프롬프트를 지원합니다. |
| `clip_g` | STRING | 예 | - | 글로벌 CLIP 모델용 텍스트 입력. 다중 줄 텍스트와 동적 프롬프트를 지원합니다. |
| `t5xxl` | STRING | 예 | - | T5-XXL 모델용 텍스트 입력. 다중 줄 텍스트와 동적 프롬프트를 지원합니다. |
| `empty_padding` | COMBO | 예 | `"none"`<br />`"empty_prompt"` | 빈 텍스트 입력을 어떻게 처리할지 제어합니다. "none"으로 설정하면, `clip_g`, `clip_l`, 또는 `t5xxl`의 빈 텍스트 입력은 패딩 대신 빈 토큰 리스트를 반환합니다. 이는 고급 매개변수입니다(기본값: "none"). |

**매개변수 제약사항:**

- `empty_padding`을 "none"으로 설정하면, `clip_g`, `clip_l`, 또는 `t5xxl`의 빈 텍스트 입력은 패딩 대신 빈 토큰 리스트를 반환합니다.
- 노드는 `clip_l`과 `clip_g` 입력 간의 토큰 길이를 자동으로 맞추기 위해, 길이가 다른 경우 짧은 쪽에 빈 토큰을 패딩하여 균형을 맞춥니다.
- 모든 텍스트 입력은 동적 프롬프트와 다중 줄 텍스트 입력을 지원합니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `CONDITIONING` | CONDITIONING | SD3 생성 파이프라인에서 바로 사용 가능한 인코딩된 텍스트 컨디셔닝 데이터 |
