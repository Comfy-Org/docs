이 노드는 SeedVR2 비디오 잠재 변수를 더 작은 시간적 청크로 분할하여, 사용 가능한 VRAM 내에서 한 번에 하나씩 처리할 수 있도록 합니다. GPU 메모리를 기반으로 최적의 청크 크기를 자동으로 계산하거나, 청크 크기를 수동으로 지정할 수 있으며, 처리 순서대로 청크를 출력합니다.

## 입력

| 매개변수 | 설명 | 데이터 타입 | 필수 | 범위 |
|-----------|-------------|-----------|----------|-------|
| `latent` | 분할할 VAE 인코딩된 SeedVR2 잠재 변수입니다. | LATENT | 예 | - |
| `temporal_overlap` | 인접 청크 간에 공유되고 병합 시 크로스페이드되는 잠재 프레임입니다. 0은 중첩이 없음을 의미합니다(기본값: 0). | INT | 아니요 | 0 ~ 16384 |
| `chunking_mode` | 수동 모드는 frames_per_chunk를 정확히 사용합니다. 자동 모드는 사용 가능한 VRAM에 맞는 가장 큰 청크를 예측합니다. | COMBO | 예 | "auto"<br />"manual" |

`chunking_mode`가 "manual"로 설정되면 다음 추가 매개변수를 사용할 수 있습니다:

| 매개변수 | 설명 | 데이터 타입 | 필수 | 범위 |
|-----------|-------------|-----------|----------|-------|
| `frames_per_chunk` | 시간적 청크당 픽셀 프레임입니다. 4n+1 값(1, 5, 9, 13, 17, 21, ...)이어야 합니다(기본값: 21). | INT | 예 | 1 ~ 16384 |

참고: `frames_per_chunk` 매개변수는 `chunking_mode`가 "manual"로 설정된 경우에만 나타납니다. 값은 `(frames_per_chunk - 1) % 4 == 0` 공식을 충족해야 하며, 즉 1, 5, 9, 13, 17, 21 등 중 하나여야 합니다.

## 출력

| 출력 이름 | 설명 | 데이터 타입 |
|-------------|-------------|-----------|
| `latents` | 순서대로 정렬된 시간적 청크입니다. | LATENT |
| `temporal_overlap` | 인접 청크 간의 유효한 잠재 프레임 중첩으로, Merge SeedVR2 Latents에 사용됩니다. | INT |
