이 노드는 SeedVR2 비디오 잠재 변수를 더 작은 시간적 청크로 분할하여, 사용 가능한 VRAM 내에서 한 번에 하나씩 처리할 수 있도록 합니다. 청크는 Apply SeedVR2 Conditioning 노드와 샘플러 잠재 입력에 모두 전달된 후, Merge SeedVR2 Latents 노드를 사용하여 다시 결합되도록 설계되었습니다.

## 입력

| 매개변수 | 설명 | 데이터 타입 | 필수 | 범위 |
|-----------|-------------|-----------|----------|-------|
| `latent` | 분할할 VAE 인코딩된 SeedVR2 잠재 변수 | LATENT | 예 | - |
| `temporal_overlap` | 인접 청크 간에 공유되고 병합 시 크로스페이드되는 잠재 프레임 수입니다. 0은 중첩이 없음을 의미합니다(기본값: 0) | INT | 아니요 | 0 ~ 16384 |
| `chunking_mode` | manual = 정확히 frames_per_chunk를 사용합니다. auto = 사용 가능한 여유 VRAM에 맞는 가장 큰 청크를 예측합니다 | COMBO | 예 | "auto"<br />"manual" |

`chunking_mode`가 "manual"로 설정된 경우 다음 매개변수를 사용할 수 있습니다:

| 매개변수 | 설명 | 데이터 타입 | 필수 | 범위 |
|-----------|-------------|-----------|----------|-------|
| `frames_per_chunk` | 시간적 청크당 픽셀 프레임 수입니다(4n+1이어야 함: 1, 5, 9, 13, ...)(기본값: 21) | INT | 예 | 1 ~ 16384, 4단계 |

**매개변수 제약 조건 참고 사항:**
- 입력 `latent`는 (B, C, T, H, W) 형태의 5차원 비디오 잠재 변수여야 하며 정확히 4개의 잠재 채널을 가져야 합니다.
- "manual" 모드를 사용하는 경우 `frames_per_chunk`는 4n+1 값(1, 5, 9, 13, 17, 21, ...)이어야 합니다.
- `temporal_overlap`은 자동으로 청크 크기보다 작게 제한됩니다.
- "auto" 모드에서 노드는 사용 가능한 여유 VRAM을 기반으로 최적의 청크 크기를 계산합니다.

## 출력

| 출력 이름 | 설명 | 데이터 타입 |
|-------------|-------------|-----------|
| `latents` | 순서대로 정렬된 시간적 청크 | LATENT |
| `temporal_overlap` | Merge SeedVR2 Latents를 위한 인접 청크 간의 유효한 잠재 프레임 중첩 | INT |
