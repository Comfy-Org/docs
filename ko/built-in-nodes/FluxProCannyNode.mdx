컨트롤 이미지(Canny)를 사용하여 이미지를 생성합니다. 이 노드는 컨트롤 이미지를 입력받아 제공된 프롬프트를 기반으로 새로운 이미지를 생성하며, 컨트롤 이미지에서 감지된 에지 구조를 따릅니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `control_image` | IMAGE | 예 | - | Canny 에지 검출에 사용되는 입력 이미지 |
| `prompt` | STRING | 아니요 | - | 이미지 생성을 위한 프롬프트 (기본값: 빈 문자열) |
| `prompt_upsampling` | BOOLEAN | 아니요 | - | 프롬프트에 대해 업샘플링을 수행할지 여부. 활성화 시 더 창의적인 생성을 위해 자동으로 프롬프트를 수정하지만 결과는 비결정적입니다(동일한 시드로도 정확히 동일한 결과가 나오지 않음). (기본값: False) |
| `canny_low_threshold` | FLOAT | 아니요 | 0.01 - 0.99 | Canny 에지 검출의 낮은 임계값; `skip_preprocessing`가 True인 경우 무시됨 (기본값: 0.1) |
| `canny_high_threshold` | FLOAT | 아니요 | 0.01 - 0.99 | Canny 에지 검출의 높은 임계값; `skip_preprocessing`가 True인 경우 무시됨 (기본값: 0.4) |
| `skip_preprocessing` | BOOLEAN | 아니요 | - | 전처리를 건너뛸지 여부; `control_image`가 이미 Canny 처리된 이미지인 경우 True로 설정하고, 원시 이미지인 경우 False로 설정하세요. (기본값: False) |
| `guidance` | FLOAT | 아니요 | 1 - 100 | 이미지 생성 과정의 가이던스 강도 (기본값: 30) |
| `steps` | INT | 아니요 | 15 - 50 | 이미지 생성 과정의 단계 수 (기본값: 50) |
| `seed` | INT | 아니요 | 0 - 18446744073709551615 | 노이즈 생성에 사용되는 난수 시드. (기본값: 0) |

**참고:** `skip_preprocessing`를 True로 설정하면 `canny_low_threshold`와 `canny_high_threshold` 매개변수는 무시됩니다. 컨트롤 이미지는 이미 Canny 에지 이미지로 처리된 것으로 간주되기 때문입니다. 그러면 `control_image`가 바로 전처리된 이미지로 사용됩니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-----------|-----------|-------------|
| `output_image` | IMAGE | 컨트롤 이미지와 프롬프트를 기반으로 생성된 이미지 |
