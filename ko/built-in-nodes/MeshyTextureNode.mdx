Meshy: Texture Node는 AI로 생성된 텍스처를 3D 모델에 적용합니다. 이 노드는 이전 Meshy 3D 생성 또는 변환 노드에서 가져온 작업 ID를 받아 텍스트 설명이나 참조 이미지를 사용해 모델에 새로운 텍스처를 생성합니다. 노드는 GLB 및 FBX 파일 형식으로 텍스처가 적용된 모델을 출력합니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `model` | COMBO | 예 | `"latest"` | 텍스처링에 사용할 AI 모델 버전입니다. 현재는 "latest" 버전만 사용 가능합니다. |
| `meshy_task_id` | MESHY_TASK_ID | 예 | - | 이전 Meshy 3D 생성 또는 변환 작업에서 가져온 고유 식별자(작업 ID)입니다. 이는 텍스처링할 기본 3D 모델을 제공합니다. |
| `enable_original_uv` | BOOLEAN | 아니요 | - | 새 UV를 생성하는 대신 모델의 원본 UV를 사용합니다. 활성화 시(기본값: `True`), Meshy는 업로드된 모델의 기존 텍스처를 유지합니다. 모델에 원본 UV가 없는 경우 출력 품질이 떨어질 수 있습니다. |
| `pbr` | BOOLEAN | 아니요 | - | 텍스처링된 모델에 물리 기반 렌더링(PBR) 재질 출력을 활성화합니다(기본값: `False`). |
| `text_style_prompt` | STRING | 아니요 | - | 텍스트를 사용해 원하는 객체의 텍스처 스타일을 설명하세요. 최대 600자입니다. `image_style`와 동시에 사용할 수 없습니다. |
| `image_style` | IMAGE | 아니요 | - | 텍스처링 과정을 안내할 2D 이미지입니다. `text_style_prompt`과 동시에 사용할 수 없습니다. |

**매개변수 제약사항:**

* `text_style_prompt`과 `image_style` 중 하나만 제공해야 하며, 두 가지를 동시에 제공할 수 없습니다.
* `text_style_prompt`은 최대 600자로 제한됩니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `model_file` | STRING | 생성된 GLB 모델의 파일명입니다. 이 출력은 하위 호환성을 위해 제공됩니다. |
| `meshy_task_id` | MODEL_TASK_ID | 이 텍스처링 작업의 고유 작업 식별자로, 결과를 참조하는 데 사용할 수 있습니다. |
| `GLB` | FILE3DGLB | GLB 파일 형식으로 저장된 텍스처가 적용된 3D 모델입니다. |
| `FBX` | FILE3DFBX | FBX 파일 형식으로 저장된 텍스처가 적용된 3D 모델입니다. |
