ColorTransfer 노드는 대상 이미지의 색상 팔레트를 참조 이미지의 색상과 일치하도록 조정합니다. 이 노드는 밝기, 대비, 색조 분포와 같은 색상 특성을 분석하고 전송하는 데 서로 다른 수학적 알고리즘을 사용합니다. 이는 여러 이미지 간의 시각적 일관성을 유지하거나 특정 색감을 적용하는 데 유용합니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|----------|-------|-------------|
| `image_target` | IMAGE | 예 | - | 색상 변환을 적용할 이미지(들). |
| `image_ref` | IMAGE | 예 | - | 색상을 맞출 참조 이미지(들). |
| `method` | COMBO | 예 | `"reinhard_lab"`<br />`"mkl_lab"`<br />`"histogram"` | 사용할 색상 전송 알고리즘. |
| `source_stats` | DYNAMICCOMBO | 예 | `"per_frame"`<br />`"uniform"`<br />`"target_frame"` | 소스(대상) 이미지로부터 색상 통계를 계산하는 방식을 결정합니다. |
| `strength` | FLOAT | 예 | 0.0부터 10.0까지 | 색상 전송 효과의 강도. 값이 1.0이면 완전한 변환이 적용되고, 0.0이면 원본 이미지를 그대로 반환합니다. 기본값: 1.0 |

**매개변수 상세 정보:**
*   **`source_stats` 옵션:**
    *   **`per_frame`**: 배치 내 각 프레임을 개별적으로 `image_ref`와 일치시킵니다.
    *   **`uniform`**: 모든 소스 프레임의 색상 통계를 취합해 단일 기준선을 만들고, 이를 `image_ref`와 일치시킵니다.
    *   **`target_frame`**: 대상 배치에서 선택된 한 프레임을 기준선으로 사용해 `image_ref`로의 변환을 계산합니다. 이후 이 변환은 모든 프레임에 균일하게 적용되며, 프레임 간의 상대적인 색상 차이가 유지됩니다. 이 옵션이 선택되면 추가로 `target_index` 매개변수가 나타납니다.
*   **`target_index`** (`source_stats`가 `"target_frame"`일 때 표시됨): 변환 계산에 사용할 소스 기준선으로 삼는 프레임 인덱스(0부터 시작). 기본값: 0. 0부터 10000 사이여야 합니다.

**제약사항:**
*   `strength`를 0.0으로 설정하거나 `image_ref`가 `None`인 경우, 노드는 처리 없이 원래의 `image_target`를 반환합니다.
*   `source_stats`를 `"target_frame"`로 설정한 경우, `target_index`는 `image_target` 배치 내 유효한 인덱스여야 합니다. 만약 인덱스가 프레임 수를 초과하면 마지막 프레임이 사용됩니다.
*   `histogram` 방법에서 `source_stats`를 `"per_frame"`로 설정한 경우, `image_ref`의 배치 크기가 1보다 클 경우 각 대상 프레임은 인덱스에 따라 해당 참조 프레임과 일치합니다. 참조 배치에 프레임이 하나뿐인 경우, 모든 대상 프레임에 동일하게 사용됩니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `image` | IMAGE | 색상 변환이 적용된 결과 이미지(들). |
