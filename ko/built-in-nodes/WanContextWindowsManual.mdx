WAN 컨텍스트 윈도우(수동) 노드를 사용하면 2차원 처리 방식을 가진 WAN 유사 모델의 컨텍스트 윈도우를 수동으로 구성할 수 있습니다. 샘플링 시 커스텀 컨텍스트 윈도우 설정을 적용하며, 윈도우 길이, 겹침, 스케줄링 방법 및 퓨전 기법을 지정할 수 있습니다. 이를 통해 모델이 서로 다른 컨텍스트 영역에서 정보를 처리하는 방식을 정밀하게 제어할 수 있습니다.

## 입력

| 매개변수 | 데이터 타입 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `model` | MODEL | 예 | - | 샘플링 중 컨텍스트 윈도우를 적용할 모델입니다. |
| `context_length` | INT | 예 | 1부터 1048576까지 | 컨텍스트 윈도우의 길이 (기본값: 81). |
| `context_overlap` | INT | 예 | 0부터 1048576까지 | 컨텍스트 윈도우의 겹침 크기 (기본값: 30). |
| `context_schedule` | COMBO | 예 | `"static_standard"`<br />`"uniform_standard"`<br />`"uniform_looped"`<br />`"batched"` | 컨텍스트 윈도우의 스텝 크기입니다. |
| `context_stride` | INT | 예 | 1부터 1048576까지 | 컨텍스트 윈도우의 스텝 크기; 균일한 스케줄에만 적용됩니다 (기본값: 1). |
| `closed_loop` | BOOLEAN | 예 | - | 컨텍스트 윈도우 루프를 닫을지 여부; 루프 스케줄에만 적용됩니다 (기본값: False). |
| `fuse_method` | COMBO | 예 | `"pyramid"`<br />`"gaussian"`<br />`"average"`<br />`"overlap"` | 컨텍스트 윈도우를 퓨전하는 데 사용할 방법입니다 (기본값: "pyramid"). |
| `freenoise` | BOOLEAN | 예 | - | FreeNoise 노이즈 섞기를 적용할지 여부, 윈도우 블렌딩을 개선합니다 (기본값: False). |

**참고:** `context_stride` 매개변수는 균일한 스케줄에만 영향을 미치며, `closed_loop`는 루프 스케줄에만 적용됩니다. 컨텍스트 길이와 겹침 값은 처리 과정에서 최소 유효 값을 보장하도록 자동으로 조정됩니다. 또한 `fuse_method` 매개변수에는 이제 "pyramid" 외에도 추가 옵션이 포함되었습니다.

## 출력

| 출력 이름 | 데이터 타입 | 설명 |
|-------------|-----------|-------------|
| `model` | MODEL | 컨텍스트 윈도우 설정이 적용된 모델입니다. |
