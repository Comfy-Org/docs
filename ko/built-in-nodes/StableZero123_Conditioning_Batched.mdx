StableZero123_Conditioning_Batched 노드는 입력 이미지를 처리하여 3D 모델 생성을 위한 컨디셔닝 데이터를 생성합니다. 이 노드는 CLIP 비전과 VAE 모델을 사용해 이미지를 인코딩한 뒤, 고도와 방위각을 기반으로 카메라 임베딩을 만들어 양성 및 음성 컨디셔닝과 배치 처리를 위한 잠재 표현을 생성합니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `clip_vision` | CLIP_VISION | 예 | - | 입력 이미지를 인코딩하는 데 사용되는 CLIP 비전 모델 |
| `init_image` | IMAGE | 예 | - | 처리하고 인코딩할 초기 입력 이미지 |
| `vae` | VAE | 예 | - | 이미지 픽셀을 잠재 공간으로 인코딩하는 데 사용되는 VAE 모델 |
| `width` | INT | 아니요 | 16부터 MAX_RESOLUTION까지 | 처리된 이미지의 출력 너비 (기본값: 256, 8로 나누어져야 함) |
| `height` | INT | 아니요 | 16부터 MAX_RESOLUTION까지 | 처리된 이미지의 출력 높이 (기본값: 256, 8로 나누어져야 함) |
| `batch_size` | INT | 아니요 | 1부터 4096까지 | 배치에서 생성할 컨디셔닝 샘플 수 (기본값: 1) |
| `elevation` | FLOAT | 아니요 | -180.0부터 180.0까지 | 초기 카메라 고도 각도 (도 단위, 기본값: 0.0) |
| `azimuth` | FLOAT | 아니요 | -180.0부터 180.0까지 | 초기 카메라 방위각도 (도 단위, 기본값: 0.0) |
| `elevation_batch_increment` | FLOAT | 아니요 | -180.0부터 180.0까지 | 각 배치 항목마다 고도를 증가시키는 양 (기본값: 0.0) |
| `azimuth_batch_increment` | FLOAT | 아니요 | -180.0부터 180.0까지 | 각 배치 항목마다 방위각을 증가시키는 양 (기본값: 0.0) |

**참고:** `width`와 `height` 매개변수는 노드 내부적으로 잠재 공간 생성을 위해 이들 차원을 8로 나누어야 하므로 반드시 8의 배수가 되어야 합니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-------------|-----------|-------------|
| `positive` | CONDITIONING | 이미지 임베딩과 카메라 파라미터를 포함한 양성 컨디셔닝 데이터 |
| `negative` | CONDITIONING | 임베딩이 0으로 초기화된 음성 컨디셔닝 데이터 |
| `latent` | LATENT | 처리된 이미지의 잠재 표현으로, 배치 인덱싱 정보 포함 |
