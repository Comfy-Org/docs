HyperTile 노드는 확산 모델의 어텐션 메커니즘에 타일링 기법을 적용하여 이미지 생성 중 메모리 사용량을 최적화합니다. 잠재 공간을 작은 타일로 나누어 각각 별도로 처리한 후 결과를 다시 조합합니다. 이를 통해 더 큰 크기의 이미지를 메모리 부족 없이 작업할 수 있습니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `model` | MODEL | 예 | - | HyperTile 최적화를 적용할 확산 모델 |
| `tile_size` | INT | 아니요 | 1 - 2048 | 처리할 타일의 목표 크기 (기본값: 256). 유효한 타일 크기는 8의 배수로 반올림되며, 최소 크기는 32입니다. |
| `swap_size` | INT | 아니요 | 1 - 128 | 처리 과정에서 타일을 재배치하는 방식을 제어하여 효율성을 높입니다 (기본값: 2) |
| `max_depth` | INT | 아니요 | 0 - 10 | 타일링을 적용할 최대 깊이 수준(해상도 스케일). 값이 0이면 가장 높은 해상도에서만 타일링을 적용합니다 (기본값: 0) |
| `scale_depth` | BOOLEAN | 아니요 | True / False | 활성화 시, 더 깊은 깊이 수준에서 타일 크기가 비례적으로 조정됩니다. 이는 낮은 해상도에서도 품질을 유지하는 데 도움이 될 수 있습니다 (기본값: False) |

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-----------|-----------|-------------|
| `model` | MODEL | HyperTile 최적화가 적용된 수정된 모델 |
