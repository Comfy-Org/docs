VAEDecodeTiled 노드는 타일 방식을 사용해 잠재 표현을 이미지로 디코딩하여 대형 이미지를 효율적으로 처리합니다. 이 노드는 입력을 작은 타일 단위로 나누어 메모리 사용량을 관리하면서도 이미지 품질을 유지합니다. 또한 동영상 VAE를 지원하며, 시간적 프레임을 겹치는 조각으로 나누어 처리해 부드러운 전환을 구현합니다.

## 입력

| 매개변수 | 데이터 유형 | 필수 | 범위 | 설명 |
|-----------|-----------|------|-------|-------------|
| `samples` | LATENT | 예 | - | 이미지로 디코딩할 잠재 표현 |
| `vae` | VAE | 예 | - | 잠재 샘플을 디코딩하는 데 사용되는 VAE 모델 |
| `tile_size` | INT | 예 | 64-4096 (단계: 32) | 처리할 각 타일의 크기 (기본값: 512) |
| `overlap` | INT | 예 | 0-4096 (단계: 32) | 인접한 타일 간의 겹침 크기 (기본값: 64) |
| `temporal_size` | INT | 예 | 8-4096 (단계: 4) | 동영상 VAE에만 사용됨: 한 번에 디코딩할 프레임 수 (기본값: 64) |
| `temporal_overlap` | INT | 예 | 4-4096 (단계: 4) | 동영상 VAE에만 사용됨: 겹치는 프레임 수 (기본값: 8) |

**참고:** 노드는 겹침 값이 실용적인 한계를 초과하면 자동으로 조정합니다. 만약 `tile_size`가 `overlap`의 4배보다 작다면, 겹침은 타일 크기의 4분의 1로 줄어듭니다. 마찬가지로, `temporal_size`가 `temporal_overlap`의 2배보다 작다면, 시간적 겹침은 절반으로 줄어듭니다. 또한 노드는 공간적 및 시간적 차원의 타일 및 겹침 크기를 계산할 때 VAE의 내부 압축 비율까지 고려합니다.

## 출력

| 출력 이름 | 데이터 유형 | 설명 |
|-----------|-----------|-------------|
| `IMAGE` | IMAGE | 잠재 표현에서 생성된 디코딩된 이미지 또는 이미지들 |
