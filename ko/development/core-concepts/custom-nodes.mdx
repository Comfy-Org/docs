## 맞춤형 노드 소개

ComfyUI를 설치한 후에는 많은 기본 제공 노드가 포함되어 있음을 알게 될 것입니다. 이러한 기본 노드는 **Comfy Core** 노드라고 하며, ComfyUI에서 공식적으로 유지 관리합니다.

또한 ComfyUI 커뮤니티의 다양한 작성자가 만든 수많은 [**맞춤형 노드**](https://registry.comfy.org)가 있습니다. 이 맞춤형 노드들은 ComfyUI에 광범위한 기능을 추가하며, ComfyUI의 기능과 한계를 크게 확장해줍니다.

이 가이드에서는 맞춤형 노드와 관련된 다양한 작업, 즉 설치, 업데이트, 비활성화, 제거 및 종속성 설치에 대해 다룹니다.

누구나 ComfyUI용 맞춤형 확장을 개발하고 다른 사람들과 공유할 수 있습니다. 많은 커뮤니티 맞춤형 노드를 [여기](https://registry.comfy.org)에서 확인할 수 있습니다. 만약 자신만의 맞춤형 노드를 개발하고 싶다면 아래 섹션을 방문하여 시작해 보세요:

<Card title="맞춤형 노드 개발 시작하기" icon="link" href="/ko/custom-nodes/overview">
  맞춤형 노드 개발을 시작하는 방법을 알아보세요
</Card>

## 맞춤형 노드 관리

이 섹션에서는 다음 내용을 다룹니다:
- 맞춤형 노드 설치
- 노드 종속성 설치
- 맞춤형 노드 버전 관리
- 맞춤형 노드 제거
- 맞춤형 노드 일시적 비활성화
- 맞춤형 노드 종속성 충돌 처리


### 1. 맞춤형 노드 설치

현재 ComfyUI는 여러 가지 방법으로 맞춤형 노드를 설치할 수 있도록 지원합니다. 여기에는 다음이 포함됩니다:

- [ComfyUI Manager를 통한 설치(권장)](#install-via-comfyui-manager)
- Git를 통한 설치
- 수동 설치

우리는 **ComfyUI Manager**를 통해 맞춤형 노드를 설치할 것을 권장합니다. 이 도구는 ComfyUI 맞춤형 노드 생태계에서 매우 중요한 역할을 합니다. 맞춤형 노드 관리(검색, 설치, 업데이트, 비활성화 및 제거 등)를 간편하게 해주며, ComfyUI Manager에서 설치하려는 노드를 검색한 뒤 설치 버튼을 클릭하기만 하면 됩니다.

그러나 현재 모든 맞춤형 노드는 GitHub에 저장되어 있으므로, GitHub에 정상적으로 접근할 수 없는 지역에서는 이 가이드에서 다양한 맞춤형 노드 설치 방법에 대한 자세한 안내를 제공합니다.

또한 플러그인 관리를 위해 **ComfyUI Manager** 사용을 권장하므로, 이 도구를 이용해 플러그인 관리를 하는 것이 좋습니다. 해당 도구의 소스 코드는 [여기](https://github.com/Comfy-Org/ComfyUI-Manager)에서 확인할 수 있습니다.

따라서 이 문서에서는 ComfyUI Manager 설치를 맞춤형 노드 설치 예제로 사용하며, 관련 소개 섹션에서 노드 관리에 어떻게 사용하는지 보완해 드립니다.


### 2. 노드 종속성 설치

맞춤형 노드는 모두 관련 종속성 설치가 필요합니다. 예를 들어 ComfyUI-Manager의 경우, [requirements.txt](https://github.com/Comfy-Org/ComfyUI-Manager/blob/main/requirements.txt) 파일을 방문해 종속성 패키지 요구사항을 확인할 수 있습니다.

앞선 단계에서는 맞춤형 노드 코드만 로컬에 복제했을 뿐, 해당 종속성은 설치하지 않았으므로, 다음으로는 해당 종속성을 설치해야 합니다.

<Note>
실제로 ComfyUI Manager를 사용해 플러그인을 설치하면 ComfyUI Manager가 자동으로 종속성 설치를 도와줍니다. 플러그인 설치 후 ComfyUI를 다시 시작하기만 하면 됩니다. 이것이 바로 우리가 맞춤형 노드 설치에 ComfyUI Manager 사용을 강력히 권장하는 이유입니다.

하지만 어떤 상황에서는 ComfyUI Manager를 이용해 맞춤형 노드를 원활하게 설치할 수 없을 수도 있으므로, 이보다 더 자세한 종속성 설치 안내를 제공합니다.
</Note>

[종속성](/development/core-concepts/dependencies) 장에서 ComfyUI의 종속성 관련 내용을 소개했습니다. ComfyUI는 **Python** 기반 프로젝트이며, 우리는 ComfyUI 실행을 위해 독립적인 **Python** 런타임 환경을 구축했습니다. 모든 관련 종속성은 이 독립적인 **Python** 런타임 환경에 설치해야 합니다.

시스템 레벨 터미널에서 직접 `pip install -r requirements.txt`를 실행하면, 해당 종속성이 시스템 레벨 **Python** 환경에 설치될 수 있으며, 이는 ComfyUI 환경에서 여전히 종속성이 누락된 상태로 남아 있어 해당 맞춤형 노드가 정상적으로 작동하지 못하게 할 수 있습니다.

따라서 다음으로는 ComfyUI의 독립적인 Python 런타임 환경을 사용해 종속성을 설치해야 합니다.

ComfyUI 버전에 따라 각기 다른 방법으로 해당 종속성을 설치합니다:

<Tabs>
<Tab title="ComfyUI Portable">
ComfyUI Portable 버전에서는 `\ComfyUI_windows_portable\python_embeded` 디렉토리에 내장된 Python을 사용합니다. 이 Python을 사용해 종속성을 설치해야 합니다.

먼저 포터블 버전 디렉토리에서 터미널을 시작하거나, 터미널을 시작한 뒤 `cd` 명령어를 사용해 `\ComfyUI_windows_portable\` 디렉토리로 이동하세요.

![터미널 시작](/images/concepts/custom_nodes/open_terminal.jpg)

터미널 디렉토리가 `\ComfyUI_windows_portable\`인지 확인하세요. 아래는 `D:\ComfyUI_windows_portable\`의 예시입니다.

![터미널](/images/concepts/custom_nodes/terminal.jpg)

그런 다음 `python_embeded\python.exe`를 사용해 종속성을 설치하세요:

```bash
python_embeded\python.exe -m pip install -r ComfyUI\custom_nodes\ComfyUI-Manager\requirements.txt
```
물론 ComfyUI-Manager 대신 실제로 설치한 맞춤형 노드 이름으로 바꿀 수 있지만, 해당 노드 디렉토리에 `requirements.txt` 파일이 존재하는지 확인하세요.
</Tab>

<Tab title="ComfyUI Desktop">

<Tip>
ComfyUI Desktop 버전은 이미 설치 과정에서 ComfyUI-Manager와 그 종속성을 설치했으며, 이 가이드에서도 ComfyUI Manager를 맞춤형 노드 설치 예제로 사용하므로, 사실 Desktop 버전에서는 ComfyUI Manager 종속성 설치를 따로 수행할 필요가 없습니다.
예상치 못한 문제가 없다면, 맞춤형 노드 설치에 ComfyUI Manager를 사용하는 것을 권장하므로, 수동으로 종속성을 설치할 필요가 없습니다.
</Tip>

![ComfyUI Desktop 터미널](/images/concepts/custom_nodes/desktop_terminal.png)

그런 다음 다음 명령어를 사용해 해당 플러그인의 종속성을 설치하세요:

```bash
pip install -r .\custom_nodes\<맞춤형 노드 이름>\requirements.txt
```
아래는 ComfyUI-Hunyuan3Dwrapper의 종속성 설치 예시입니다:

![ComfyUI Desktop 종속성 설치](/images/concepts/custom_nodes/install_dependencies.png)

</Tab>

<Tab title="맞춤형 Python 환경 사용자">
맞춤형 Python 환경을 사용하는 사용자에게는 `pip install -r requirements.txt`를 사용해 종속성을 설치하는 것을 권장합니다.
</Tab>

</Tabs>

### 맞춤형 노드 버전 관리

맞춤형 노드 버전 관리는 사실 Git 버전 관리에 기반합니다. Git를 통해 노드 버전을 관리할 수 있지만, ComfyUI Manager는 이미 이 버전 관리 기능을 매우 잘 통합했습니다. 이렇게 편리한 도구를 만들어주신 [@Dr.Lt.Data](https://github.com/ltdrdata)님께 깊은 감사를 드립니다.

이 섹션에서는 두 가지 서로 다른 플러그인 버전 관리 방법을 설명드리지만, ZIP 패키지를 사용해 수동 설치한 경우에는 해당 git 버전 기록 정보가 사라져 버전 관리가 불가능합니다.

<Tabs>

<Tab title="ComfyUI Manager를 통한 버전 관리">

<Tip>ComfyUI Manager는 계속해서 개발되고 있으므로, 실제 최신 인터페이스와 단계는 크게 달라질 수 있습니다</Tip>

<Steps>
<Step title="노드 관리 인터페이스 진입">
다음과 같이 해당 작업을 수행해 ComfyUI Manager 인터페이스로 진입하세요
</Step>
<Step title="해당 맞춤형 노드 패키지 찾기">
필터를 사용해 설치된 노드 패키지를 필터링한 뒤, 해당 노드 관리를 수행할 수 있습니다
</Step>
<Step title="버전 전환 수행">
해당 버전으로 전환하세요. Manager가 해당 종속성 업데이트와 설치를 도와줍니다. 일반적으로 버전 전환 후 ComfyUI를 다시 시작해야 변경 사항이 적용됩니다.
</Step>
</Steps>

</Tab>

## ComfyUI Manager

![ComfyUI Manager 인터페이스](/images/concepts/core-concepts_nodes_manager.png)


이 도구는 현재 [Desktop 버전](/installation/desktop/windows)에 기본적으로 포함되어 있으며, [Portable 버전](/installation/comfyui_portable_windows)에서는 이 문서의 [Manager 설치](#installing-custom-nodes) 섹션의 설치 지침을 참고해야 합니다.

<Note>
ComfyUI가 지속적으로 발전하면서 ComfyUI Manager는 점점 더 중요한 역할을 하고 있습니다. 현재 ComfyUI-Manager는 공식적으로 Comfy Org 조직에 합류해 ComfyUI의 핵심 종속성의 일부가 되었으며, 원작자인 [Dr.Lt.Data](https://github.com/ltdrdata)가 계속해서 유지 관리하고 있습니다. 더 자세한 내용은 [이 블로그 게시글](https://blog.comfy.org/p/comfyui-manager-joins-comfy-org)을 참조하세요.
향후 버전에서는 ComfyUI Manager의 사용을 크게 최적화할 예정이므로, 이 문서에 표시된 인터페이스가 최신 ComfyUI Manager와 다를 수 있습니다.
</Note>

### Manager 설치

ComfyUI 서버 애플리케이션을 실행 중이라면, manager를 설치해야 합니다. ComfyUI가 실행 중이라면, 계속하기 전에 이를 종료하세요.

첫 번째 단계는 Git를 설치하는 것입니다. Git는 소프트웨어 버전 관리를 위한 명령줄 애플리케이션입니다. Git는 [github.com](https://github.com)에서 ComfyUI manager를 다운로드합니다. [git-scm.com](https://git-scm.com/)에서 Git를 다운로드하고 설치하세요.

Git 설치 후, ComfyUI 서버 프로그램 디렉토리로 이동해 **custom_nodes**라는 폴더로 진입하세요. 명령어 창이나 터미널을 열어주세요. 명령어 줄에 현재 디렉토리 경로가 **custom_nodes**로 표시되었는지 확인하세요. 다음 명령어를 입력하세요. 이는 manager를 다운로드하는 것입니다. 기술적으로 이것은 _Git 저장소 복제_라고 불립니다.
