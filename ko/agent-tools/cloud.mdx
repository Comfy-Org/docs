## 개요

**Comfy Cloud MCP 서버**는 [Model Context Protocol](https://modelcontextprotocol.io)을 통해 AI 에이전트를 [Comfy Cloud](https://cloud.comfy.org)에 연결합니다. 연결 후에는 에이전트와의 채팅에서 이미지, 비디오, 오디오 및 3D를 생성하고, 모델, 노드 및 템플릿을 검색하며, ComfyUI 워크플로를 실행할 수 있습니다.

서버는 `cloud.comfy.org/mcp`에서 원격으로 실행됩니다. 워크플로는 Comfy Cloud GPU에서 실행되므로 로컬 GPU가 필요하지 않습니다. 에이전트는 HTTPS를 통해 서버와 통신하며, 서버는 도구 호출을 대신 인증된 요청으로 변환하여 Comfy Cloud에 전송합니다.

현재 지원 범위는 **Claude Code**와 **Claude Desktop**에 국한되며, 이들은 **OAuth**(일회성 브라우저 로그인)를 통해 로그인합니다. 더 많은 클라이언트 지원이 곧 제공될 예정입니다.

## 에이전트가 할 수 있는 작업

직접 MCP 도구를 호출하지 않습니다. 에이전트가 사용자의 요청에 따라 적합한 도구를 선택합니다. 슬래시 명령어와 프롬프트(아래 참조)는 에이전트를 일반적인 작업으로 안내하는 단축키이지만, 일반 언어로도 작동합니다("고양이 우주비행사 이미지 생성", "이 사진 업스케일", "Wan 2.2 비디오 템플릿 찾기").

일반적인 흐름:

1. **탐색**: 사용 가능한 항목 확인(`search_templates`, `search_models`, `search_nodes`, 또는 그래프 스타일 질문의 경우 `cql`).
2. **실행**: 생성 실행. 일치하는 사전 구축된 템플릿에는 `run_template`, 사용자 정의 워크플로에는 `submit_workflow`(입력 이미지가 필요한 경우 `upload_file` 사용), Flux, Grok, Gemini, OpenAI, Ideogram, Seedance와 같은 파트너 모델에는 `partner_generate`를 사용.
3. **대기 및 출력 가져오기**: (`wait_for_job` 실행 후 `get_output`이 에이전트가 셸에서 실행하는 다운로드 명령을 반환).

서버는 처음부터 워크플로를 구축하기보다 [사전 구축된 템플릿](https://comfy.org/workflows)을 일치시키는 것을 선호하며, 이는 더 빠르고 나은 결과를 생성하는 경향이 있습니다.

## MCP 도구

연결되면 에이전트가 액세스할 수 있는 도구입니다. 이름은 MCP 클라이언트 로그 및 디버깅에 표시되는 것과 일치합니다.

### Discovery

| 도구 | 설명 |
| --- | --- |
| `search_templates` | 텍스트, 태그, 미디어 유형 또는 모델로 [comfy.org](https://comfy.org/workflows)의 사전 제작된 워크플로 템플릿 검색 |
| `get_template` | 검사 또는 직접 편집을 위해 템플릿의 전체 워크플로 JSON 가져오기 |
| `get_template_schema` | 실행 시 재정의할 수 있는 템플릿 매개변수 확인 |
| `search_models` | 텍스트, 유형, 베이스 모델 또는 소스로 모델 카탈로그 검색 |
| `search_nodes` | 텍스트, 카테고리 또는 입력/출력 유형으로 사용 가능한 노드 검색. 또한 **서브그래프 블루프린트**(예: 텍스트 → 이미지 또는 배경 제거와 같은 사전 연결된 노드 그룹)를 표시합니다. |
| `get_node` | 전체 옵션 목록을 포함하여 특정 노드의 완전한 입력 사양 가져오기 |
| `cql` | 구조적 질문(예: 어떤 노드가 이미지를 생성하는지, 또는 노드 유형 간의 경로)을 위해 [CQL](https://github.com/Comfy-Org/cql) 그래프 쿼리 실행. 특정 노드에 대한 전체 입력 사양이 필요한 경우 `search_nodes` 사용 |
| `get_prompting_guide` | 모델 계열별 프롬프트 스타일 및 권장 설정(steps, cfg, sampler, 해상도) 확인 |

### Generation

| 도구 | 설명 |
| --- | --- |
| `run_template` | 사전 구축된 템플릿을 이름으로 실행(선택적 매개변수 재정의 가능) — 일치하는 템플릿이 있을 때 선호되는 경로 |
| `submit_workflow` | Comfy Cloud에서 실행할 ComfyUI API 형식 워크플로 제출 |
| `partner_generate` | 파트너 API 모델(Flux/BFL, Grok, Gemini, OpenAI, Ideogram, Seedream/Seedance 등)로 생성 |
| `upload_file` | 워크플로에서 사용할 입력 이미지 또는 파일 업로드(예: LoadImage 사용) |
| `apply_slots` | 워크플로 내부 값에 매개변수 재정의를 적용하고 수정된 워크플로 반환 |

### Jobs and batches

| 도구 | 설명 |
| --- | --- |
| `get_job_status` | 제출된 워크플로의 실행 상태 폴링 |
| `wait_for_job` | 반복 폴링 대신 작업이 완료될 때까지 대기 |
| `get_output` | 완료된 워크플로에서 출력 이미지, 비디오 또는 오디오 검색 |
| `use_previous_output` | 한 실행의 출력을 다른 실행의 입력으로 재사용하여 워크플로 체이닝 |
| `cancel_job` | 대기 중 또는 실행 중인 작업 취소 |
| `get_queue` | 실행 중 및 대기 중인 작업 수 확인 |
| `submit_batch`, `get_batch_status`, `get_batch_output`, `wait_for_batch` | 한 번의 호출로 여러 생성을 제출하고 나중에 함께 수집 — 배치 ID는 세션 간에도 유효하게 유지됩니다 |

### Saved workflows

| 도구 | 설명 |
| --- | --- |
| `list_saved_workflows` | Comfy Cloud에서 저장된 워크플로 찾아보기 |
| `get_saved_workflow` | 저장된 워크플로의 노드, 입력 및 사용자 정의 가능한 설정 검사 |
| `save_workflow` | 워크플로를 Comfy Cloud 계정에 저장 |
| `update_workflow` | 기존 워크플로를 새 버전으로 그 자리에서 업데이트(이전 버전은 복구 가능하게 유지됨) |
| `run_saved_workflow` | 저장된 워크플로를 파일 이름으로 실행 — 서버가 에디터 형식에서 실행 가능한 형식으로 자동 변환 |

### Sharing workflows

| 도구 | 설명 |
| --- | --- |
| `share_workflow` | 저장된 워크플로를 게시하고 누구나 열 수 있는 `?share=<id>` URL 반환 |
| `import_shared_workflow` | 공유 URL 또는 bare 공유 ID를 워크플로 JSON으로 확인하고 선택적으로 계정에 저장 |

**허브 URL 공유 ID:** `comfy.org/workflows/<slug>-<hex>` 허브 URL에서 마지막 하이픈으로 구분된 16진수 토큰이 공유 ID입니다. 예를 들어, `comfy.org/workflows/topaz-starlight-upscale-1c77e82713b7`의 공유 ID는 `1c77e82713b7`입니다. 해당 토큰을 `share_id`로 `import_shared_workflow`에 전달하세요. `share_url` 매개변수는 `https://cloud.comfy.org/?share=...`와 같은 `?share=<id>` 쿼리 URL만 허용하며 허브 페이지 URL은 허용하지 않습니다.

### Apps and links

| 도구 | 설명 |
| --- | --- |
| `create_app` | 저장된 워크플로를 App Mode 앱으로 전환 — 선택한 입력과 출력으로 구성된 단순화된 "이 워크플로 실행" 보기 |
| `get_app_mode_url` | 워크플로를 실행 가능한 앱으로 여는 안정적인 링크 가져오기 |
| `get_workflow_canvas_url` | 워크플로를 Comfy Cloud 캔버스에서 바로 열어 보고, 편집하고, 실행할 수 있는 링크 가져오기 |

### Account and session

| 도구 | 설명 |
| --- | --- |
| `get_billing_status` | 크레딧 잔액, 구독 등급 및 결제 링크 확인 |
| `get_server_info` | 에이전트가 연결된 서버 확인: 환경, 호스트, 버전 및 인증 상태 |
| `submit_feedback` | 베타 피드백 설문 링크 가져오기 |
| `report_session_summary` | 익명화된 세션 요약을 Comfy 팀과 공유 — **사용자의 명시적 동의가 있을 때만** 실행되며, 에이전트가 먼저 물어봐야 하고 프롬프트, 파일 경로 또는 개인 정보는 포함되지 않습니다 |

## 크레딧 및 지출

탐색은 무료입니다: `search_templates`, `search_models`, `search_nodes`는 [Comfy 계정](https://cloud.comfy.org)만으로 작동합니다. 생성을 실행하면 Comfy Cloud 크레딧이 소모되며 구독 또는 크레딧 잔액이 필요합니다.

실제 비용이 발생하는 도구에는 **지출 게이트(spend-gate)**가 적용됩니다. 생성에 유료 파트너 모델(`partner_generate`)이나 유료 API 노드가 포함된 워크플로/템플릿(`run_template`, `run_saved_workflow`, `submit_batch`)이 사용되는 경우, 서버는 사용자가 크레딧 지출에 동의했음을 에이전트가 확인할 때까지 실행을 차단합니다. 따라서 해당 실행 전에 에이전트가 사용자에게 확인을 요청합니다. 무료 로컬 전용 워크플로는 확인을 요청하지 않습니다.

언제든지 에이전트에게 잔액을 물어볼 수 있습니다("크레딧이 얼마나 남았나요?"). 에이전트는 `get_billing_status`를 통해 답변합니다.

### <span style={{ marginRight: "0.6rem" }}><Icon icon="terminal" size={22} /></span> Claude Code

**comfy-cloud** 플러그인을 설치합니다. MCP 연결과 슬래시 명령이 한 번에 추가됩니다. 이 플러그인은 Comfy용 Claude Code 플러그인과 커뮤니티 에이전트 스킬을 호스팅하는 [Comfy Skills](https://github.com/Comfy-Org/comfy-skills/) 저장소를 통해 게시됩니다.

<Steps>
  <Step title="마켓플레이스 추가">
    Comfy Skills 마켓플레이스를 등록합니다(한 번만 실행):

    ```
    /plugin marketplace add Comfy-Org/comfy-skills
    ```
  </Step>
  <Step title="플러그인 설치">
    ```
    /plugin install comfy-cloud@comfy-skills
    ```
  </Step>
  <Step title="로그인">
    `/mcp`를 실행하고 **comfy-cloud** → **Authenticate**를 선택합니다. 브라우저가 열려 로그인되며, 토큰은 자동으로 새로 고쳐집니다.
  </Step>
</Steps>

플러그인은 다음 슬래시 명령을 추가합니다:

| 명령 | 기능 |
| --- | --- |
| `/comfy-cloud:generate-image` | 이미지 생성, 편집 또는 수정 |
| `/comfy-cloud:generate-video` | 비디오 생성, 편집 또는 확장 |
| `/comfy-cloud:generate-audio` | 오디오, 음악 또는 음향 효과 생성 |
| `/comfy-cloud:generate-3d` | 텍스트 또는 이미지로 3D 모델 생성 |
| `/comfy-cloud:remove-background` | 이미지에서 배경 제거 |
| `/comfy-cloud:upscale-image` | 이미지를 더 높은 해상도로 업스케일 |
| `/comfy-cloud:search-templates` | [comfy.org](https://comfy.org)에서 미리 빌드된 워크플로 템플릿 검색 |
| `/comfy-cloud:search-models` | 사용 가능한 모델 검색 |
| `/comfy-cloud:search-nodes` | 노드 검색 및 연결 제안 받기 |
| `/comfy-cloud:help` | ComfyUI Cloud로 할 수 있는 작업 확인 |

<Accordion title="플러그인 없이 연결만 선호하시나요?">
  서버를 직접 추가합니다:

  ```bash
  claude mcp add --transport http comfy-cloud https://cloud.comfy.org/mcp
  ```

  그런 다음 `/mcp`를 실행하고 **comfy-cloud** → **Authenticate**를 선택합니다. `-s user`를 추가하면 모든 프로젝트에서 사용할 수 있습니다.

  이 방법도 동일한 워크플로를 MCP 프롬프트로 노출합니다: `/mcp__comfy-cloud__generate-image`, `/mcp__comfy-cloud__search-models` 등 (`/mcp__<name>__` 접두사는 `claude mcp add`에 전달한 이름을 사용합니다). 위의 플러그인이 더 친숙한 `/comfy-cloud:*` 명령으로 감싸 주므로 권장됩니다.
</Accordion>

### <span style={{ marginRight: "0.6rem" }}><Icon icon="desktop" size={22} /></span> Claude Desktop

Claude Desktop은 UI를 통해 Comfy Cloud를 **사용자 정의 커넥터**로 추가한 후 OAuth 로그인을 실행합니다.

    ![Claude Desktop 커넥터](/images/cloud/mcp/desktop-connectors.png)
  </Step>
  <Step title="사용자 정의 커넥터 추가">

    ![사용자 정의 커넥터 추가](/images/cloud/mcp/desktop-add-custom-connector.png)
  </Step>
  <Step title="서버 URL 입력">
    원하는 이름을 지정하고(예: *Comfy Cloud*) **원격 MCP 서버 URL**을 다음으로 설정합니다:

    ```
    https://cloud.comfy.org/mcp
    ```

    ![커넥터 세부 정보](/images/cloud/mcp/desktop-connector-url.png)
  </Step>
  <Step title="로그인">

    ![Claude Desktop을 통해 로그인](/images/cloud/mcp/desktop-signin.png)
  </Step>
</Steps>

Claude Desktop은 Claude Code 슬래시 명령을 지원하지 않습니다. 대신 동일한 워크플로를 위해 **프롬프트 선택기**를 엽니다:

| 프롬프트 | 설명 |
| --- | --- |
| `generate-image` | 텍스트 설명에서 이미지 생성 |
| `generate-video` | 텍스트 또는 이미지에서 비디오 생성 |
| `generate-audio` | 오디오, 음악 또는 음향 효과 생성 |
| `generate-3d` | 텍스트 또는 이미지에서 3D 모델 생성 |
| `upscale-image` | 이미지를 더 높은 해상도로 업스케일 |
| `remove-background` | 이미지에서 배경 제거 |
| `search-templates` | 미리 빌드된 워크플로 템플릿 검색 |
| `search-models` | 모델 검색 (체크포인트, LoRA, VAE) |
| `search-nodes` | 노드 검색 및 연결 제안 받기 |
| `help` | ComfyUI Cloud로 할 수 있는 작업 확인 |

프롬프트를 건너뛰고 일반 언어로 요청할 수도 있습니다. MCP 도구는 동일하게 작동합니다.

### 헤드리스 / CI (API 키)

Claude Code와 Claude Desktop은 OAuth를 사용하므로 API 키가 필요하지 않습니다. 브라우저를 사용할 수 없는 **헤드리스 또는 CI** 설정의 경우 대신 API 키로 인증합니다:

<Steps>
  <Step title="키 생성">
    [platform.comfy.org/profile/api-keys](https://platform.comfy.org/profile/api-keys)로 이동하여 **New API Key**를 클릭하고 복사합니다(`comfyui-`로 시작).
  </Step>
  <Step title="키 헤더와 함께 서버 추가">
    ```bash
    claude mcp add --transport http comfy-cloud https://cloud.comfy.org/mcp -H "X-API-Key: comfyui-…"
    ```
  </Step>
</Steps>

## 출력 및 다운로드

MCP 서버는 클라우드에서 실행되며 **사용자 기기에 파일을 절대 쓰지 않습니다**. 생성이 완료되면 에이전트가 `get_output`을 호출하며, 다음을 반환합니다:

1. **임시 서명된 다운로드 URL**(짧은 기간 동안 유효).
2. 바로 실행 가능한 **셸 명령**(macOS 및 Linux에서는 `curl`, Windows에서는 `curl.exe`).

에이전트는 해당 명령을 셸에서 실행해야 합니다. 명령에는 대상 경로와 파일 이름이 포함됩니다.

<Warning>
  반환된 명령을 **그대로** 실행하세요. 서명된 URL을 다시 인코딩하거나 편집하지 마세요. 서명은 쿼리 문자열에 있으며 URL이 수정되면 깨집니다.
</Warning>

MCP 클라이언트가 셸 명령을 실행할 수 없는 경우(일부 GUI 전용 설정), 명령을 복사하여 터미널에서 직접 실행하세요.

## 알려진 제한 사항

Comfy Cloud MCP는 초기 릴리스입니다. 다음 제한 사항이 알려져 있으며 현재 작업 중입니다:

**워크플로**

- **에디터 형식에서 실행 가능한 형식으로의 변환에는 아직 미흡한 부분이 있습니다.** 저장된 워크플로는 `run_saved_workflow`를 통해 파일 이름으로 실행되지만(서버가 자동으로 변환), 일부 노드 구성은 여전히 불완전하게 변환됩니다.
- **`submit_workflow`로 생성된 에셋에는 워크플로 메타데이터가 포함되지 않을 수 있습니다.** ComfyUI에서 열 때 원본 워크플로를 다시 열지 못할 수 있습니다.
- **워크플로 구축은 에이전트의 정확성에 달려 있습니다.** 복잡한 다중 노드 워크플로는 재시도 또는 개선이 필요할 수 있습니다.

**파일 처리**

- **출력은 쉘 다운로드 단계가 필요합니다.** [출력 및 다운로드](#outputs-and-downloads)를 참조하세요.
- **업로드 크기 제한**은 MCP 클라이언트에 따라 적용될 수 있습니다. 일부 클라이언트는 파일 업로드에 자체 제한을 부과합니다.

**인증**

- **OAuth 또는 API 키.** Claude Code와 Claude Desktop은 일회성 브라우저 OAuth 흐름을 사용합니다. 헤드리스 클라이언트는 대신 `X-API-Key` 헤더를 통해 Comfy Cloud API 키를 전달할 수 있습니다. 브라우저를 열 수 없는 클라이언트를 위한 장치 코드 OAuth 흐름이 계획되어 있습니다.

## 자주 묻는 질문

<AccordionGroup>
  <Accordion title="지원되는 클라이언트는 무엇인가요?">
    현재 Claude Code와 Claude Desktop이 지원되며, 둘 다 OAuth로 로그인합니다. 더 많은 클라이언트 지원이 곧 추가될 예정입니다.
  </Accordion>
  <Accordion title="API 키가 필요한가요?">
    Claude Code나 Claude Desktop에서는 필요하지 않습니다. OAuth를 사용합니다. API 키는 브라우저가 없는 헤드리스 또는 CI 환경에서만 필요합니다.
  </Accordion>
  <Accordion title="에이전트가 생성 전에 확인을 요청하는 이유는 무엇인가요?">
    유료 생성에는 지출 게이트가 적용됩니다: 실행이 Comfy Cloud 크레딧을 소모하는 경우(파트너 모델 또는 유료 API 노드가 포함된 워크플로/템플릿), 서버는 실행 전에 사용자의 명시적 동의를 요구합니다. 자세한 내용은 위의 크레딧 및 지출 섹션을 참조하세요.
  </Accordion>
  <Accordion title="슬래시 명령이 Claude Desktop에서 작동하나요?">
    아니요. 슬래시 명령은 Claude Code 플러그인에 포함되어 있습니다. Claude Desktop은 동일한 MCP 서버에 연결됩니다(일반 언어로 요청하거나 프롬프트 선택기를 사용하면 도구가 작동합니다). 하지만 Claude Code 플러그인이나 슬래시 명령은 지원하지 않습니다.
  </Accordion>
  <Accordion title="/comfy 또는 /cloud를 입력했는데 아무것도 나타나지 않습니다.">
    `/comfy` 또는 `/cloud` 명령은 없습니다. 명령은 연결 방식에 따라 두 가지 접두사 중 하나로 나타납니다:

    - **플러그인 (권장):** `/comfy-cloud:generate-image`, `/comfy-cloud:generate-video`, … : 전체를 보려면 `/comfy-cloud:`를 입력하세요.
    - **직접 연결 (플러그인 없음):** `/mcp__comfy-cloud__generate-image`, … : 보려면 `/mcp__`를 입력하세요.

    어느 쪽이든 일반 언어로 요청할 수도 있습니다 ("...의 이미지 생성"). MCP 도구는 모델에 의해 호출되며 슬래시 명령이 필요하지 않습니다.
  </Accordion>
  <Accordion title="로그인 시 브라우저가 열리지 않았습니다.">
    Claude Code에서 `/mcp`를 실행하고 **comfy-cloud**를 **선택**한 다음 **인증**을 선택하세요. Claude Desktop에서는 **사용자 정의 → 커넥터**에서 커넥터를 다시 열고 로그인을 트리거하세요.
  </Accordion>
  <Accordion title="에이전트로 Comfy를 로컬에서 실행하는 것은 어떻습니까?">
    Comfy Cloud MCP는 저희가 호스팅하는 서비스이지만, 로컬 에이전트 워크플로는 ComfyUI가 오픈소스화된 이후로 존재해 왔습니다. Agent Tools 개요에서 [로컬 ComfyUI는 어떻습니까?](/ko/agent-tools#what-about-local-comfyui)를 참조하여 클라우드가 먼저 출시된 이유, 현재 사용할 수 있는 커뮤니티 MCP 서버, 그리고 또 다른 옵션으로 [Comfy CLI](/ko/agent-tools/comfy-cli)를 확인하세요.
  </Accordion>
</AccordionGroup>
