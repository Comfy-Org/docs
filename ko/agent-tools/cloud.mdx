## 개요

**Comfy Cloud MCP 서버**는 [Model Context Protocol](https://modelcontextprotocol.io)을 통해 AI 에이전트를 [Comfy Cloud](https://cloud.comfy.org)에 연결합니다. 연결 후에는 에이전트와의 채팅에서 이미지, 비디오, 오디오 및 3D를 생성하고, 모델, 노드 및 템플릿을 검색하며, ComfyUI 워크플로를 실행할 수 있습니다.

서버는 `cloud.comfy.org/mcp`에서 원격으로 실행됩니다. 워크플로는 Comfy Cloud GPU에서 실행되므로 로컬 GPU가 필요하지 않습니다. 에이전트는 HTTPS를 통해 서버와 통신하며, 서버는 도구 호출을 대신 인증된 요청으로 변환하여 Comfy Cloud에 전송합니다.

모든 **MCP 호환 클라이언트**가 동일한 서버 URL에 연결할 수 있습니다. **Claude Code**, **Claude Desktop**, **Cursor**, **Codex**는 아래에 전용 설정 안내가 있습니다. **Windsurf**, **Amp** 및 기타 에이전트는 OAuth 또는 API 키를 사용하여 동일한 URL을 사용합니다.

## Comfy Cloud MCP 설치

<Info>
  연결하기 전에 [Comfy Cloud](https://cloud.comfy.org) 계정이 필요합니다. 아직 계정이 없다면 [가입하기](https://cloud.comfy.org)를 클릭하세요. 설정 중 OAuth 로그인은 Comfy 계정을 사용합니다.
</Info>

Comfy Cloud MCP는 다음 주소에서 실행됩니다:

```
https://cloud.comfy.org/mcp
```

클라이언트를 선택하세요:

<Tabs>
  <Tab title="Claude Code">
    **comfy-cloud** 플러그인을 설치하세요. 이 플러그인은 MCP 연결과 슬래시 명령어를 한 번에 추가합니다. 이 플러그인은 Comfy의 Claude Code 플러그인과 커뮤니티 에이전트 스킬을 호스팅하는 [Comfy Skills](https://github.com/Comfy-Org/comfy-skills/) 저장소를 통해 게시됩니다.

    <Steps>
      <Step title="마켓플레이스 추가">
        Comfy Skills 마켓플레이스를 등록합니다(한 번만 수행):

        ```
        /plugin marketplace add Comfy-Org/comfy-skills
        ```
      </Step>
      <Step title="플러그인 설치">
        ```
        /plugin install comfy-cloud@comfy-skills
        ```
      </Step>
      <Step title="로그인">
        `/mcp`를 실행하고 **comfy-cloud** → **Authenticate**를 선택하세요. 브라우저가 열려 로그인하고 토큰은 자동으로 새로 고쳐집니다.
      </Step>
    </Steps>

    플러그인은 다음 슬래시 명령어를 추가합니다:

    | 명령어 | 설명 |
    | --- | --- |
    | `/comfy-cloud:generate-image` | 이미지 생성, 편집 또는 수정 |
    | `/comfy-cloud:generate-video` | 비디오 생성, 편집 또는 확장 |
    | `/comfy-cloud:generate-audio` | 오디오, 음악 또는 사운드 효과 생성 |
    | `/comfy-cloud:generate-3d` | 텍스트 또는 이미지에서 3D 모델 생성 |
    | `/comfy-cloud:remove-background` | 이미지에서 배경 제거 |
    | `/comfy-cloud:upscale-image` | 이미지를 더 높은 해상도로 업스케일 |
    | `/comfy-cloud:search-templates` | [comfy.org](https://comfy.org)에서 사전 구축된 워크플로 템플릿 검색 |
    | `/comfy-cloud:search-models` | 사용 가능한 모델 검색 |
    | `/comfy-cloud:search-nodes` | 노드 검색 및 연결 제안 받기 |
    | `/comfy-cloud:help` | ComfyUI Cloud로 할 수 있는 작업 확인 |

    <Accordion title="플러그인 없이 연결만 원하시나요?">
      서버를 직접 추가하세요:

      ```bash
      claude mcp add --transport http comfy-cloud https://cloud.comfy.org/mcp
      ```

      그런 다음 `/mcp`를 실행하고 **comfy-cloud** → **Authenticate**를 선택하세요. `-s user`를 추가하면 모든 프로젝트에서 사용할 수 있습니다.

      이 방법도 MCP 프롬프트로 동일한 워크플로를 제공합니다: `/mcp__comfy-cloud__generate-image`, `/mcp__comfy-cloud__search-models` 등(`/mcp__<이름>__` 접두사는 `claude mcp add`에 전달한 이름을 사용합니다). 위 플러그인은 이를 더 친숙한 `/comfy-cloud:*` 명령어로 래핑하므로 권장됩니다.
    </Accordion>
  </Tab>

  <Tab title="Cursor">
    Cursor는 원격 MCP 서버에 HTTP로 연결됩니다. **Cursor Settings → Tools & MCP**에서 Comfy Cloud를 추가하거나 MCP 구성 파일을 직접 편집하세요.

    <Steps>
      <Step title="서버 URL 추가">
        프로젝트에 `.cursor/mcp.json`을 생성하거나 편집하거나, 전역 설정을 위해 `~/.cursor/mcp.json`을 편집하세요:

        ```json
        {
          "mcpServers": {
            "comfy-cloud": {
              "url": "https://cloud.comfy.org/mcp"
            }
          }
        }
        ```
      </Step>
      <Step title="로그인">
        **Cursor Settings → Tools & MCP**를 엽니다. **comfy-cloud**를 선택하고 **Connect**를 클릭하여 브라우저를 통해 로그인합니다(OAuth). 토큰은 자동으로 새로 고쳐집니다.
      </Step>
      <Step title="생성 시작">
        일반 언어로 요청하세요("고양이 우주비행사 이미지 생성", "Wan 2.2 비디오 템플릿 찾기"). Cursor가 대신 MCP 도구를 호출합니다.
      </Step>
    </Steps>

    <Accordion title="OAuth 대신 API 키 사용(헤드리스 또는 CI)">
      브라우저를 사용할 수 없는 경우 `X-API-Key` 헤더에 Comfy Cloud API 키를 전달하세요. [platform.comfy.org/profile/api-keys](https://platform.comfy.org/profile/api-keys)에서 키를 생성하세요(`comfyui-`로 시작):

      ```json
      {
        "mcpServers": {
          "comfy-cloud": {
            "url": "https://cloud.comfy.org/mcp",
            "headers": {
              "X-API-Key": "${env:COMFY_API_KEY}"
            }
          }
        }
      }
      ```

      `COMFY_API_KEY`를 셸 또는 시스템 환경에 설정하세요. 키를 버전 관리에 커밋하지 마세요.
    </Accordion>

      터미널에서 등록 및 로그인:

      ```bash
      codex mcp add comfy-cloud --url https://cloud.comfy.org/mcp
      codex mcp login comfy-cloud
      ```

      이렇게 하면 `~/.codex/config.toml`(또는 신뢰할 수 있는 프로젝트의 `.codex/config.toml`)에 항목이 기록됩니다.

      ```toml
      [mcp_servers.comfy-cloud]
      url = "https://cloud.comfy.org/mcp"
      env_http_headers = { "X-API-Key" = "COMFY_API_KEY" }
      ```

      `COMFY_API_KEY`를 셸 또는 시스템 환경에 설정하세요. 키를 버전 관리에 커밋하지 마세요.

    **원격 HTTP** 전송을 지원하는 모든 MCP 클라이언트는 Comfy Cloud에 연결할 수 있습니다. 서버 URL은 항상 `https://cloud.comfy.org/mcp`입니다.

    <Steps>
      <Step title="서버 URL 추가">
        `https://cloud.comfy.org/mcp`를 가리키는 원격 MCP 항목을 추가하세요. 대부분의 클라이언트는 `url` 필드가 있는 JSON 구성을 사용합니다:

        ```json
        {
          "mcpServers": {
            "comfy-cloud": {
              "url": "https://cloud.comfy.org/mcp"
            }
          }
        }
        ```

        **Windsurf**는 `url` 대신 `serverUrl`을 사용합니다. **Amp**는 **Settings → MCP Servers** 또는 `~/.config/amp/settings.json`에서 동일한 `mcpServers` 형식을 사용합니다.
      </Step>
      <Step title="인증">
        클라이언트가 MCP OAuth를 지원하는 경우 브라우저에서 로그인하라는 메시지가 표시됩니다. 그렇지 않으면 API 키 헤더를 추가하세요(아래 참조).
      </Step>
      <Step title="재시작 및 확인">
        클라이언트를 재시작하세요. **comfy-cloud** 서버 아래에 Comfy Cloud 도구(`search_templates`, `submit_workflow`, `get_output` 등)가 등록된 것을 확인할 수 있습니다.
      </Step>
    </Steps>

    **헤드리스 / CI (API 키):** OAuth를 사용할 수 없는 경우 [platform.comfy.org/profile/api-keys](https://platform.comfy.org/profile/api-keys)에서 키를 생성하고 `X-API-Key` 헤더로 전달하세요:

    ```json
    {
      "mcpServers": {
        "comfy-cloud": {
          "url": "https://cloud.comfy.org/mcp",
          "headers": {
            "X-API-Key": "comfyui-…"
          }
        }
      }
    }
    ```

    터미널에서 Claude Code의 경우:

    ```bash
    claude mcp add --transport http comfy-cloud https://cloud.comfy.org/mcp -H "X-API-Key: comfyui-…"
    ```

    Git에 커밋하는 구성 파일에 키를 하드코딩하는 대신 `${env:COMFY_API_KEY}` 또는 클라이언트의 비밀 변수 보간을 사용하는 것이 좋습니다.
  </Tab>
</Tabs>

## MCP 도구

연결되면 에이전트가 액세스할 수 있는 도구입니다. 이름은 MCP 클라이언트 로그 및 디버깅에 표시되는 것과 일치합니다.

### Discovery

| 도구 | 설명 |
| --- | --- |
| `search_templates` | 텍스트, 태그, 미디어 유형 또는 모델로 [comfy.org](https://comfy.org/workflows)의 사전 제작된 워크플로 템플릿 검색 |
| `get_template` | 검사 또는 직접 편집을 위해 템플릿의 전체 워크플로 JSON 가져오기 |
| `get_template_schema` | 실행 시 재정의할 수 있는 템플릿 매개변수 확인 |
| `search_models` | 텍스트, 유형, 베이스 모델 또는 소스로 모델 카탈로그 검색 |
| `search_nodes` | 텍스트, 카테고리 또는 입력/출력 유형으로 사용 가능한 노드 검색. 또한 **서브그래프 블루프린트**(예: 텍스트 → 이미지 또는 배경 제거와 같은 사전 연결된 노드 그룹)를 표시합니다. |
| `get_node` | 전체 옵션 목록을 포함하여 특정 노드의 완전한 입력 사양 가져오기 |
| `cql` | 구조적 질문(예: 어떤 노드가 이미지를 생성하는지, 또는 노드 유형 간의 경로)을 위해 [CQL](https://github.com/Comfy-Org/cql) 그래프 쿼리 실행. 특정 노드에 대한 전체 입력 사양이 필요한 경우 `search_nodes` 사용 |
| `get_prompting_guide` | 모델 계열별 프롬프트 스타일 및 권장 설정(steps, cfg, sampler, 해상도) 확인 |

## 크레딧 및 지출

탐색은 무료입니다: `search_templates`, `search_models`, `search_nodes`는 [Comfy 계정](https://cloud.comfy.org)만으로 작동합니다. 생성을 실행하면 Comfy Cloud 크레딧이 소모되며 구독 또는 크레딧 잔액이 필요합니다.

실제 비용이 발생하는 도구에는 **지출 게이트(spend-gate)**가 적용됩니다. 생성에 유료 파트너 모델(`partner_generate`)이나 유료 API 노드가 포함된 워크플로/템플릿(`run_template`, `run_saved_workflow`, `submit_batch`)이 사용되는 경우, 서버는 사용자가 크레딧 지출에 동의했음을 에이전트가 확인할 때까지 실행을 차단합니다. 따라서 해당 실행 전에 에이전트가 사용자에게 확인을 요청합니다. 무료 로컬 전용 워크플로는 확인을 요청하지 않습니다.

언제든지 에이전트에게 잔액을 물어볼 수 있습니다("크레딧이 얼마나 남았나요?"). 에이전트는 `get_billing_status`를 통해 답변합니다.

**comfy-cloud** 플러그인을 설치합니다. MCP 연결과 슬래시 명령이 한 번에 추가됩니다. 이 플러그인은 Comfy용 Claude Code 플러그인과 커뮤니티 에이전트 스킬을 호스팅하는 [Comfy Skills](https://github.com/Comfy-Org/comfy-skills/) 저장소를 통해 게시됩니다.

  </Step>

플러그인은 다음 슬래시 명령을 추가합니다:

| 명령 | 기능 |
| --- | --- |
| `/comfy-cloud:generate-image` | 이미지 생성, 편집 또는 수정 |

| `/comfy-cloud:search-models` | 사용 가능한 모델 검색 |

<Accordion title="플러그인 없이 연결만 선호하시나요?">

  ```bash
  claude mcp add --transport http comfy-cloud https://cloud.comfy.org/mcp
  ```

</Accordion>

## 출력 및 다운로드

MCP 서버는 클라우드에서 실행되며 **사용자 기기에 파일을 절대 쓰지 않습니다**. 생성이 완료되면 에이전트가 `get_output`을 호출하며, 다음을 반환합니다:

1. **임시 서명된 다운로드 URL**(짧은 기간 동안 유효).
2. 바로 실행 가능한 **셸 명령**(macOS 및 Linux에서는 `curl`, Windows에서는 `curl.exe`).

에이전트는 해당 명령을 셸에서 실행해야 합니다. 명령에는 대상 경로와 파일 이름이 포함됩니다.

<Warning>
  반환된 명령을 **그대로** 실행하세요. 서명된 URL을 다시 인코딩하거나 편집하지 마세요. 서명은 쿼리 문자열에 있으며 URL이 수정되면 깨집니다.
</Warning>

MCP 클라이언트가 셸 명령을 실행할 수 없는 경우(일부 GUI 전용 설정), 명령을 복사하여 터미널에서 직접 실행하세요.

## 알려진 제한 사항

Comfy Cloud MCP는 초기 릴리스입니다. 다음 제한 사항이 알려져 있으며 현재 작업 중입니다:

**워크플로**

- **에디터 형식에서 실행 가능한 형식으로의 변환에는 아직 미흡한 부분이 있습니다.** 저장된 워크플로는 `run_saved_workflow`를 통해 파일 이름으로 실행되지만(서버가 자동으로 변환), 일부 노드 구성은 여전히 불완전하게 변환됩니다.
- **`submit_workflow`로 생성된 에셋에는 워크플로 메타데이터가 포함되지 않을 수 있습니다.** ComfyUI에서 열 때 원본 워크플로를 다시 열지 못할 수 있습니다.
- **워크플로 구축은 에이전트의 정확성에 달려 있습니다.** 복잡한 다중 노드 워크플로는 재시도 또는 개선이 필요할 수 있습니다.

**파일 처리**

- **출력은 쉘 다운로드 단계가 필요합니다.** [출력 및 다운로드](#outputs-and-downloads)를 참조하세요.
- **업로드 크기 제한**은 MCP 클라이언트에 따라 적용될 수 있습니다. 일부 클라이언트는 파일 업로드에 자체 제한을 부과합니다.

**인증**

- **OAuth 또는 API 키.** Claude Code와 Claude Desktop은 일회성 브라우저 OAuth 흐름을 사용합니다. 헤드리스 클라이언트는 대신 `X-API-Key` 헤더를 통해 Comfy Cloud API 키를 전달할 수 있습니다. 브라우저를 열 수 없는 클라이언트를 위한 장치 코드 OAuth 흐름이 계획되어 있습니다.

## 자주 묻는 질문

<AccordionGroup>
  <Accordion title="지원되는 클라이언트는 무엇인가요?">
    원격 HTTP 전송을 지원하는 모든 MCP 호환 클라이언트는 `https://cloud.comfy.org/mcp`에 연결할 수 있습니다. **Claude Code**, **Claude Desktop**, **Cursor** 및 **Codex**에 대한 일급 설정을 문서화합니다. **Windsurf**, **Amp**, **OpenClaw** 및 기타 에이전트는 OAuth 또는 API 키를 사용하여 동일한 URL을 따릅니다. [Comfy Cloud MCP 설치](#install-comfy-cloud-mcp)를 참조하세요.
  </Accordion>
  <Accordion title="API 키가 필요한가요?">
    OAuth를 지원하는 대화형 클라이언트(Claude Code, Claude Desktop, Cursor, Codex 등)에는 필요하지 않습니다. API 키는 브라우저 없이 헤드리스 또는 CI 설정에서만 필요합니다. [Comfy Cloud MCP 설치](#install-comfy-cloud-mcp)의 **기타 클라이언트** 탭을 참조하세요.
  </Accordion>
  <Accordion title="에이전트가 생성 전에 확인을 요청하는 이유는 무엇인가요?">
    유료 생성에는 지출 게이트가 적용됩니다: 실행이 Comfy Cloud 크레딧을 소모하는 경우(파트너 모델 또는 유료 API 노드가 포함된 워크플로/템플릿), 서버는 실행 전에 사용자의 명시적 동의를 요구합니다. 자세한 내용은 [크레딧 및 지출](#credits-and-spending)을 참조하세요.
  </Accordion>
  <Accordion title="슬래시 명령이 Claude Desktop에서 작동하나요?">
    아니요. 슬래시 명령은 Claude Code 플러그인에 포함되어 있습니다. Claude Desktop은 동일한 MCP 서버에 연결됩니다(일반 언어로 요청하거나 프롬프트 선택기를 사용하면 도구가 작동합니다). 하지만 Claude Code 플러그인이나 슬래시 명령은 지원하지 않습니다.
  </Accordion>
  <Accordion title="/comfy 또는 /cloud를 입력했는데 아무것도 나타나지 않습니다.">
    `/comfy` 또는 `/cloud` 명령은 없습니다. 명령은 연결 방식에 따라 두 가지 접두사 중 하나로 나타납니다:

    - **플러그인 (권장):** `/comfy-cloud:generate-image`, `/comfy-cloud:generate-video`, … — 전체를 보려면 `/comfy-cloud:`를 입력하세요.
    - **직접 연결 (플러그인 없음):** `/mcp__comfy-cloud__generate-image`, … — 보려면 `/mcp__`를 입력하세요.

    어느 쪽이든 일반 언어로 요청할 수도 있습니다 ("...의 이미지 생성"). MCP 도구는 모델에 의해 호출되며 슬래시 명령이 필요하지 않습니다.
  </Accordion>
  <Accordion title="로그인 시 브라우저가 열리지 않았습니다.">
    Claude Code에서 `/mcp`를 실행하고 **comfy-cloud**를 **선택**한 다음 **인증**을 선택하세요. Claude Desktop에서는 **사용자 정의 → 커넥터**에서 커넥터를 다시 열고 로그인을 트리거하세요.
  </Accordion>
  <Accordion title="에이전트로 Comfy를 로컬에서 실행하는 것은 어떻습니까?">
    Comfy Cloud MCP는 당사의 호스팅 서비스이지만, 로컬 에이전트 워크플로는 ComfyUI가 오픈소스화된 이후로 존재해 왔습니다. Agent Tools 개요에서 [로컬 ComfyUI는 어떻습니까?](/ko/agent-tools#what-about-local-comfyui)를 참조하여 클라우드가 먼저 출시된 이유, 현재 사용할 수 있는 커뮤니티 MCP 서버, 그리고 또 다른 옵션인 [Comfy CLI](/ko/agent-tools/comfy-cli)를 확인하세요.
  </Accordion>
</AccordionGroup>
