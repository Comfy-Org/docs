## Comfy Cloud란?

ComfyUI Cloud는 로컬 버전과 동일한 기능을 갖춘 ComfyUI의 클라우드 버전입니다. 모든 것이 사전 설치되어 즉시 사용할 수 있습니다.

## 클라우드 vs 로컬

ComfyUI는 공식 클라우드 버전인 [Comfy Cloud](https://comfy.org/cloud)와 오픈소스 자체 호스팅 버전을 제공합니다. 강력한 GPU를 보유한 경우 로컬에서 ComfyUI를 실행하는 것이 좋은 선택입니다. 반면 클라우드 버전은 즉시 사용 가능한 온라인 서비스로, URL만 열면 설치나 설정이 필요 없습니다.


| 카테고리 | Comfy Cloud | 자체 호스팅 (로컬 ComfyUI) |
|----------|-------------|------------------------------|
| **비용** | 월간 구독 | 무료 |
| **GPU** | 강력한 Blackwell RTX 6000 Pro | 직접 GPU 준비 필요 |
| **기술 지식** | 기술 지식이 필요 없습니다. | 데스크톱 및 휴대용 버전은 쉽게 시작할 수 있지만, 커스텀 노드 설치 및 로컬 설치 문제 해결이 필요합니다. |
| **커스텀 노드** | 사전 설치된 커스텀 노드를 사용하며 호환성 문제를 걱정할 필요가 없습니다. | 원하는 커스텀 노드를 설치할 수 있지만 직접 관리해야 합니다. |
| **모델** | 사전 설치된 모델을 사용합니다. Civitai에서 LoRA 모델을 가져올 수 있습니다. Hugging Face에서 모델을 가져올 예정입니다(곧 출시). | 원하는 모델을 사용할 수 있지만 먼저 다운로드해야 합니다. |
| **주목할 만한 차이점** | 팀 구성이 용이함 | 오프라인에서도 작동하며 무한히 커스터마이징 가능 |
| **시작하기** | [ComfyUI Cloud 실행하기](https://comfy.org/cloud) | [로컬 ComfyUI 설치하기](/ko/installation/system_requirements) |

## ComfyUI Cloud 사용 방법

ComfyUI Cloud를 사용하는 방법은 기본적으로 로컬 ComfyUI를 사용하는 것과 같습니다. ComfyUI를 처음 사용하신다면, 다음의 간단한 팁을 참고해보세요:

<Steps>
  <Step title="템플릿 선택하기">
    왼쪽 사이드바의 템플릿 아이콘을 클릭해 사용 가능한 워크플로우를 둘러보세요.
    ![워크플로우 템플릿 열기](/images/cloud/open_template.webp)
  </Step>
  
  <Step title="실행할 워크플로우 선택하기">
    실행하고자 하는 템플릿을 클릭하세요.
    ![워크플로우 템플릿 선택하기](/images/cloud/template_library.webp)
  </Step>
  
  <Step title="템플릿 입력값 업데이트하기">
    1. 로드된 워크플로우에서, 클라우드 버전에서는 모든 모델이 사전 설치되어 있으므로 로드 후 바로 실행할 준비가 됩니다.
    2. 업데이트가 필요한 경우 이미지 입력이나 텍스트 프롬프트만 제공하면 됩니다. 'Load Image' 노드 또는 'CLIP Text Encode' 노드를 확인하세요.
    ![입력값 확인하기](/images/cloud/workflow_input_or_loader.webp)
  </Step>
  
  <Step title="워크플로우 실행하기">
    모든 것이 정확하다면, 'Run' 아이콘을 클릭하거나 단축키 'Ctrl + Enter'를 사용해 워크플로우를 실행하세요.
    ![워크플로우 실행하기](/images/cloud/workflow_run_workflow.webp)
  </Step>

  <Step title="출력물 보기">
    실행을 클릭한 후, 저희 서비스는 워크플로우를 위한 머신을 할당하기 시작합니다. 대기열 패널에서 해당 워크플로우의 실행 상태를 확인할 수 있습니다.
    ![대기열 보기](/images/cloud/workflow_check_queue.webp)
  </Step>

  <Step title="내부 콘텐츠 저장하기">
    워크플로우 실행이 완료된 후, 생성된 콘텐츠를 로컬에 저장할 수 있습니다. 자산 유형에 따라 저장 방법은 다음과 같습니다:

    <Tabs>
      <Tab title="이미지 저장하기">
        대기열 패널 또는 이미지 저장 노드에서 생성된 이미지를 마우스 오른쪽 버튼으로 클릭하고 '이미지 저장'을 선택해 로컬에 저장하세요.
        
        ![이미지 자산 저장하기](/images/cloud/download_image_assets.jpg)
      </Tab>
      
      <Tab title="비디오 저장하기">
        대기열 패널 또는 비디오 저장 노드에서:
        1. 브라우저 플레이어 구성 요소의 세 개 점을 클릭해 메뉴를 열고
        2. '다운로드'를 선택해 비디오를 로컬에 저장하세요.
        
        ![비디오 자산 저장하기](/images/cloud/download_video_assets.webp)
      </Tab>
      
      <Tab title="오디오 저장하기">
        대기열 패널 또는 오디오 저장 노드에서:
        1. 브라우저 플레이어 구성 요소의 세 개 점을 클릭해 메뉴를 열고
        2. '다운로드'를 선택해 오디오 파일을 로컬에 저장하세요.
        
        ![오디오 자산 저장하기](/images/cloud/download_audio_assets.webp)
      </Tab>
      
      <Tab title="3D 자산 저장하기">
        3D 브라우저 노드 메뉴에서 'Export' 옵션을 선택하고 원하는 형식을 고르면 3D 파일이 로컬에 저장됩니다.
        
        ![3D 자산 저장하기](/images/cloud/download_3d_assets.webp)
      </Tab>
    </Tabs>
  </Step>
</Steps>
