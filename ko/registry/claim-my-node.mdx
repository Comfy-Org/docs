## 개요

**내 노드 신청하기** 기능을 통해 개발자는 ComfyUI 레지스트리의 맞춤형 노드에 대한 소유권을 주장할 수 있습니다. 이 시스템은 정당한 저자만이 자신이 게시한 노드를 관리하고 업데이트할 수 있도록 보장하며, 커뮤니티 내에서 보안과 책임성을 제공합니다.

## 미신청 노드란?

ComfyUI 매니저에서 새로운 표준을 갖춘 Comfy 레지스트리로 이전하면서, 이전 ComfyUI 매니저 레거시 시스템에 등록되었던 많은 맞춤형 노드가 이제 레지스트리에서 '미신청' 상태로 나타납니다. 이러한 노드는 다음과 같은 특징을 가집니다:

- 원래 ComfyUI 매니저 레거시 시스템에 게시되었던 것들
- 최신 표준을 충족하도록 Comfy 레지스트리로 이전된 것들
- 원래 저자가 소유권을 주장하기를 기다리는 것들

개발자들이 이러한 이전된 노드를 손쉽게 신청할 수 있는 방법을 제공함으로써, 레거시 시스템에서 새로운 레지스트리 표준으로 원활하게 전환하면서도 적절한 소유권과 관리를 유지할 수 있도록 합니다.

## 시작하기

노드를 신청하려면:

1. **미신청 노드 페이지로 이동**: 미신청 노드 페이지에 방문하여 '내 노드 신청하기!' 버튼을 클릭하세요.

2. **퍼블리셔 생성 (아직 없으신 경우)**:
   - 아직 퍼블리셔를 생성하지 않았다면, 퍼블리셔를 생성하라는 안내가 표시됩니다. 노드를 신청하고 효과적으로 관리하려면 퍼블리셔가 필요합니다.

3. **퍼블리셔 선택**: 노드를 신청하고자 하는 퍼블리셔를 선택하세요. 이 단계에서는 신청 페이지로 이동합니다.

1. **노드 정보 확인**:
   - 화면에 표시된 노드의 이름, 저장소 링크, 퍼블리셔 상태 등 노드 상세 정보를 확인하세요.

2. **GitHub 인증**:
   - 'GitHub로 계속하기' 버튼을 클릭해 인증 과정을 시작하세요. 저장소의 관리자 권한이 있는 올바른 GitHub 계정에 로그인되어 있는지 확인하세요.

3. **관리자 접근 권한 확인**:
   - 로그인 후, 시스템은 지정된 GitHub 저장소에 대한 관리자 권한이 있는지 확인합니다. 이 단계는 필요한 권한을 확보하는 데 매우 중요합니다.

4. **노드 신청**:
   - 인증이 성공하면 '신청'을 클릭해 노드를 신청하세요. 그러면 퍼블리셔 상태가 변경되어 소유권이 명시됩니다.

5. **완료!**:
   - 노드를 성공적으로 신청한 후에는 [게시](./publishing) 및 노드 관리를 정당한 소유자로서 계속 진행하실 수 있습니다.

## 자주 묻는 질문

<AccordionGroup>

  <Accordion title="누가 노드를 신청할 수 있나요?">
    모든 GitHub 사용자가 신청 요청을 제출할 수 있지만, 신청은 원래 노드 저장소와 저자 정보를 기반으로 검증됩니다.
  </Accordion>

  <Accordion title="신청 과정은 얼마나 걸리나요?">
    GitHub 저장소에 대한 관리자 권한이 확인되면 신청은 자동으로 승인됩니다. 이 과정은 신청서 제출 즉시 이루어집니다.
  </Accordion>

  <Accordion title="신청이 승인된 후 어떻게 되나요?">
    승인되면 노드의 관리에 대한 완전한 권한을 가지게 되며, 버전 게시, 업데이트 메타, 폐기 및 정책 설정 등을 포함한 모든 관리 작업을 수행할 수 있습니다.
  </Accordion>

  <Accordion title="내가 만들지 않은 노드를 신청할 수 있나요?">
    아니요, 신청은 GitHub 저장소 관리자 권한을 기반으로 자동으로 검증됩니다. 원래 저장소의 관리자 권한을 가진 사용자만이 노드를 성공적으로 신청할 수 있습니다.
  </Accordion>

  <Accordion title="신청이 거부된다면 어떻게 하나요?">
    신청은 GitHub 관리자 권한을 기반으로 자동 처리됩니다. 만약 저장소에 관리자 권한이 없다면 신청은 자동으로 거부됩니다. 소유권을 가져야 한다고 생각된다면 저장소 소유자에게 연락해 관리자 권한을 요청하세요.
  </Accordion>
</AccordionGroup>
