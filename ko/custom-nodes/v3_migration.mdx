## 개요

ComfyUI V3 스키마는 노드를 정의하는 방식을 보다 체계적으로 개선했으며, 향후 노드 기능 확장은 오직 V3 스키마에만 추가됩니다. 이 가이드를 사용해 기존 V1 노드를 새 V3 스키마로 마이그레이션할 수 있습니다.

## 핵심 개념

V3 스키마는 새로운 버전화된 Comfy API에 유지되며, 이는 스키마의 향후 수정사항이 하위 호환성을 보장한다는 것을 의미합니다. ```comfy_api.latest```는 여전히 개발 중인 최신 번호의 API를 가리키며, 최신 버전 바로 전 버전이 '안정적'이라고 볼 수 있습니다. 현재(첫 번째) API 버전은 ```v0_0_2```이며, 이 버전에는 경고 없이 더 많은 변경이 이루어질 것입니다. 안정적인 버전으로 간주되면, ```latest```가 가리키는 버전이 새롭게 ```v0_0_3```으로 변경됩니다.

```python
# 최신 ComfyUI API 사용
from comfy_api.latest import ComfyExtension, io, ui

# 특정 버전의 ComfyUI API 사용
from comfy_api.v0_0_2 import ComfyExtension, io, ui
```

### V1 vs V3 아키텍처

V3 스키마에서 가장 큰 변화는 다음과 같습니다:
- 입력과 출력이 사전 대신 객체로 정의됨.
- 실행 방식이 'execute'라는 이름으로 고정되고 클래스 메서드임.
- ```def comfy_entrypoint()``` 함수는 ComfyExtension 객체를 반환하며, 노출되는 노드를 NODE_CLASS_MAPPINGS/NODE_DISPLAY_NAME_MAPPINGS 대신 정의함.
- 노드 객체는 '상태'를 노출하지 않음 - ```def __init__(self)```는 노드의 함수에서 노출되는 내용에 아무런 영향을 미치지 않으며, 모든 함수는 클래스 메서드임. 또한 노드 클래스는 실행 전에 정제됨.

#### V1 (레거시)
```python
class MyNode:
    @classmethod
    def INPUT_TYPES(s):
        return {"required": {...}}

    RETURN_TYPES = ("IMAGE",)
    FUNCTION = "execute"
    CATEGORY = "my_category"

    def execute(self, ...):
        return (result,)

NODE_CLASS_MAPPINGS = {"MyNode": MyNode}
```

#### V3 (현대)
```python
from comfy_api.latest import ComfyExtension, io

class MyNode(io.ComfyNode):
    @classmethod
    def define_schema(cls) -> io.Schema:
        return io.Schema(
            node_id="MyNode",
            display_name="My Node",
            category="my_category",
            inputs=[...],
            outputs=[...]
        )

    @classmethod
    def execute(cls, ...) -> io.NodeOutput:
        return io.NodeOutput(result)

class MyExtension(ComfyExtension):
    async def get_node_list(self) -> list[type[io.ComfyNode]]:
        return [MyNode]

async def comfy_entrypoint() -> ComfyExtension:
    return MyExtension()
```

## 마이그레이션 단계

대부분의 경우 V1에서 V3으로의 이동은 간단하며, 단순한 문법 변경에 불과합니다.

### 단계 1: 기본 클래스 변경

모든 V3 스키마 노드는 ```ComfyNode```를 상속해야 합니다. 여러 계층의 상속도 괜찮으며, 체인의 맨 위에 ```ComfyNode``` 부모가 있으면 됩니다.

**V1:**
```python
class Example:
    def __init__(self):
        pass
```

**V3:**
```python
from comfy_api.latest import io

class Example(io.ComfyNode):
    # __init__ 필요 없음
```

### 단계 2: INPUT_TYPES를 define_schema로 변환

노드 ID, 표시 이름, 카테고리 등 코드 내 다양한 위치에서 할당되었던 노드 속성들은 이제 ```Schema``` 클래스를 통해 함께 관리됩니다.

```define_schema(cls)``` 함수는 V1에서 INPUT_TYPES(s)가 작동했던 것과 비슷하게 ```Schema``` 객체를 반환하도록 예상됩니다.

지원되는 핵심 입력/출력 유형은 ```comfy_api/{version}```의 ```_io.py```에 저장되고 문서화되어 있으며, 기본적으로 ```io```라는 네임스페이스로 설정됩니다. 이제 입력과 출력은 사전이나 문자열 대신 클래스로 정의되기 때문에, 맞춤형 유형은 자체 클래스를 정의하거나 ```io```의 도우미 함수 ```Custom```을 사용하여 지원됩니다.

맞춤형 유형은 아래 섹션에서 자세히 설명됩니다.

유형 클래스는 다음과 같은 속성을 가집니다:
- ```class Input```은 입력용 (예: ```Model.Input(...)```)
- ```class Output```은 출력용 (예: ```Model.Output(...)```). 모든 유형이 출력 가능하지는 않다는 점에 유의하세요.
- ```Type```은 유형의 타입힌트를 얻는 데 사용됩니다 (예: ```Model.Type```). 일부 타입힌트는 단순히 ```any```일 수 있으며, 이는 향후 업데이트될 수 있습니다. 이러한 타입힌트는 강제되지 않으며, 단순히 유용한 문서화 역할을 합니다.

**V1:**
```python
@classmethod
def INPUT_TYPES(s):
    return {
        "required": {
            "image": ("IMAGE",),
            "int_field": ("INT", {
                "default": 0,
                "min": 0,
                "max": 4096,
                "step": 64,
                "display": "number"
            }),
            "string_field": ("STRING", {
                "multiline": False,
                "default": "Hello"
            }),
            # V1 임의 유형 처리
            "custom_field": ("MY_CUSTOM_TYPE",),
        },
        "optional": {
            "mask": ("MASK",)
        }
    }
```

**V3:**
```python
@classmethod
def define_schema(cls) -> io.Schema:
    return io.Schema(
        node_id="Example",
        display_name="Example Node",
        category="examples",
        description="노드 설명 여기",
        inputs=[
            io.Image.Input("image"),
            io.Int.Input("int_field",
                default=0,
                min=0,
                max=4096,
                step=64,
                display_mode=io.NumberDisplay.number
            ),
            io.String.Input("string_field",
                default="Hello",
                multiline=False
            ),
            # V3 임의 유형 처리
            io.Custom("my_custom_type").Input("custom_input"),
            io.Mask.Input("mask", optional=True)
        ],
        outputs=[
            io.Image.Output()
        ]
    )
```

### 단계 3: Execute 메서드 업데이트

V3의 모든 실행 함수는 이름이 ```execute```이고 클래스 메서드입니다.

**V1:**
```python
def test(self, image, string_field, int_field):
    # 처리
    image = 1.0 - image
    return (image,)
```

**V3:**
```python
@classmethod
def execute(cls, image, string_field, int_field) -> io.NodeOutput:
    # 처리
    image = 1.0 - image

    # 선택적 UI 미리보기와 함께 반환
    return io.NodeOutput(image, ui=ui.PreviewImage(image, cls=cls))
```

### 단계 4: 노드 속성 변환

다음은 몇 가지 속성명의 예시입니다; 더 자세한 내용은 ```comfy_api.latest._io```의 소스코드를 참고하세요.

| V1 속성 | V3 스키마 필드 | 참고 |
|------------|-----------------|-------|
| `RETURN_TYPES` | Schema의 `outputs` | 출력 객체 리스트 |
| `RETURN_NAMES` | Output의 `display_name` | 각 출력별 표시 이름 |
| `FUNCTION` | 항상 `execute` | 메서드 이름 표준화 |
| `CATEGORY` | Schema의 `category` | 문자열 값 |
| `OUTPUT_NODE` | Schema의 `is_output_node` | 불린 플래그 |
| `DEPRECATED` | Schema의 `is_deprecated` | 불린 플래그 |
| `EXPERIMENTAL` | Schema의 `is_experimental` | 불린 플래그 |

### 단계 5: 특수 메서드 처리

V1과 동일한 특수 메서드가 지원되지만, 소문자로 바뀌거나 완전히 이름이 변경되어 명확성을 높였습니다. 사용법은 그대로입니다.

#### 유효성 검사 (V1 → V3)

입력 유효성 검사 함수는 ```validate_inputs```로 이름이 변경되었습니다.

**V1:**
```python
@classmethod
def VALIDATE_INPUTS(s, **kwargs):
    # 유효성 검사 로직
    return True
```

**V3:**
```python
@classmethod
def validate_inputs(cls, **kwargs) -> bool | str:
    # 유효하면 True, 아니면 에러 문자열 반환
    if error_condition:
        return "에러 메시지"
    return True
```

#### 지연 평가 (V1 → V3)

```check_lazy_status``` 함수는 클래스 메서드이며, 다른 부분은 그대로입니다.

**V1:**
```python
def check_lazy_status(self, image, string_field, ...):
    if condition:
        return ["string_field"]
    return []
```

**V3:**
```python
@classmethod
def check_lazy_status(cls, image, string_field, ...):
    if condition:
        return ["string_field"]
    return []
```

#### 캐시 제어 (V1 → V3)

캐시 제어의 기능은 V1과 동일하지만, 원래 이름은 작동 방식에 대해 매우 혼란스러웠습니다.

V1의 ```IS_CHANGED``` 함수는 반환값이 마지막 실행 시와 동일하면 노드의 재실행을 트리거하지 않도록 실행을 신호합니다.

따라서 ```IS_CHANGED``` 함수는 ```fingerprint_inputs```로 이름이 변경되었습니다. 개발자들이 가장 흔히 하는 실수 중 하나는 ```True```를 반환하면 노드가 항상 재실행된다고 생각하는 것이었습니다. 하지만 ```True```는 항상 반환되므로, 노드가 한 번만 실행되고 캐시된 값을 재사용하는 반대 효과를 가져옵니다.

이 함수를 사용하는 예시는 LoadImage 노드입니다. 선택한 파일의 해시를 반환하므로, 파일이 변경되면 노드가 강제로 재실행됩니다.

**V1:**
```python
@classmethod
def IS_CHANGED(s, **kwargs):
    return "unique_value"
```

**V3:**
```python
@classmethod
def fingerprint_inputs(cls, **kwargs):
    return "unique_value"
```

### 단계 6: 확장 및 진입점 생성

노드 ID와 노드 클래스/표시 이름을 연결하는 사전을 정의하는 대신, 이제 ```ComfyExtension``` 클래스와 정의해야 하는 예상되는 ```comfy_entrypoint``` 함수가 있습니다.

향후 더 많은 함수가 ComfyExtension에 추가되어 ```get_node_list```를 통해 노드뿐만 아니라 다른 항목도 등록할 수 있을 것입니다.

```comfy_entrypoint```는 비동기 또는 동기 모두 가능하지만, ```get_node_list```는 반드시 비동기로 정의해야 합니다.

**V1:**
```python
NODE_CLASS_MAPPINGS = {
    "Example": Example
}

NODE_DISPLAY_NAME_MAPPINGS = {
    "Example": "Example Node"
}
```

**V3:**
```python
from comfy_api.latest import ComfyExtension

class MyExtension(ComfyExtension):
    # 반드시 비동기로 선언해야 함
    async def get_node_list(self) -> list[type[io.ComfyNode]]:
        return [
            Example,
            # 여기에 더 많은 노드 추가
        ]

# 비동기 또는 동기로 선언 가능하며, 둘 다 작동함
async def comfy_entrypoint() -> MyExtension:
    return MyExtension()
```

## 입력 유형 참조

이미 2단계에서 설명했지만, 여기에 V1과 V3의 유형 참조 비교를 나열합니다. 전체 유형 선언은 ```comfy_api.latest._io```를 참고하세요.

### 기본 유형

| V1 유형 | V3 유형 | 예시 |
|---------|---------|---------|
| `"INT"` | `io.Int.Input()` | `io.Int.Input("count", default=1, min=0, max=100)` |
| `"FLOAT"` | `io.Float.Input()` | `io.Float.Input("strength", default=1.0, min=0.0, max=10.0)` |
| `"STRING"` | `io.String.Input()` | `io.String.Input("text", multiline=True)` |
| `"BOOLEAN"` | `io.Boolean.Input()` | `io.Boolean.Input("enabled", default=True)` |

#### control_after_generate

Int 및 Combo 입력은 `control_after_generate` 매개변수를 지원하며, 이는 각 생성 후 값이 자동으로 변경되는 컨트롤 위젯을 추가합니다. V1에서는 단순한 `bool`이었으나, V3에서는 `io.ControlAfterGenerate` 열거형을 사용해 명확한 제어가 가능합니다. `True`를 전달하면 `io.ControlAfterGenerate.randomize`와 동일합니다.

| 값 | 동작 |
|-------|----------|
| `io.ControlAfterGenerate.fixed` | 값은 각 생성 후에도 동일하게 유지됨. |
| `io.ControlAfterGenerate.increment` | 값은 각 생성 후 단계만큼 증가함. |
| `io.ControlAfterGenerate.decrement` | 값은 각 생성 후 단계만큼 감소함. |
| `io.ControlAfterGenerate.randomize` | 값은 각 생성 후 무작위로 변경됨. |

```python
# 컨트롤 위젯 활성화 (사용자가 UI에서 모드 선택)
io.Int.Input("seed", default=0, min=0, max=0xFFFFFFFFFFFFFFFF, control_after_generate=True)

# 특정 기본 모드 설정
io.Int.Input("seed", default=0, min=0, max=0xFFFFFFFFFFFFFFFF,
    control_after_generate=io.ControlAfterGenerate.randomize)
```

### ComfyUI 유형

| V1 유형 | V3 유형 | 예시 |
|---------|---------|---------|
| `"IMAGE"` | `io.Image.Input()` | `io.Image.Input("image", tooltip="입력 이미지")` |
| `"MASK"` | `io.Mask.Input()` | `io.Mask.Input("mask", optional=True)` |
| `"LATENT"` | `io.Latent.Input()` | `io.Latent.Input("latent")` |
| `"CONDITIONING"` | `io.Conditioning.Input()` | `io.Conditioning.Input("positive")` |
| `"MODEL"` | `io.Model.Input()` | `io.Model.Input("model")` |
| `"VAE"` | `io.VAE.Input()` | `io.VAE.Input("vae")` |
| `"CLIP"` | `io.CLIP.Input()` | `io.CLIP.Input("clip")` |

### 콤보 (드롭다운/선택 목록)

V3의 콤보 유형은 명시적인 클래스 정의가 필요합니다.

**V1:**
```python
"mode": (["option1", "option2", "option3"],)
```

**V3:**
```python
io.Combo.Input("mode", options=["option1", "option2", "option3"])
```

## 스키마 참조

```Schema``` 데이터클래스는 V3 노드의 모든 속성을 정의합니다. 다음은 사용 가능한 모든 필드의 전체 참조입니다:

| 필드 | 유형 | 기본값 | 설명 |
|-------|------|---------|-------------|
| `node_id` | `str` | *필수* | 노드의 전역적으로 고유한 ID. 맞춤형 노드는 충돌을 피하기 위해 접두사/접미사를 추가해야 합니다. |
| `display_name` | `str` | `None` | UI에 표시되는 표시 이름. 설정되지 않은 경우 `node_id`로 돌아갑니다. |
| `category` | `str` | `"sd"` | "노드 추가" 메뉴의 카테고리 (예: `"image/transform"`). |
| `description` | `str` | `""` | 노드 위에 마우스를 올리면 표시되는 툴팁. |
| `inputs` | `list[Input]` | `[]` | 입력 정의 목록. |
| `outputs` | `list[Output]` | `[]` | 출력 정의 목록. |
| `hidden` | `list[Hidden]` | `[]` | 요청할 숨겨진 입력 목록 (참조: [숨겨진 입력](#숨겨진-입력)). |
| `search_aliases` | `list[str]` | `[]` | 검색을 위한 대체 이름. 동의어나 이름 변경 후 이전 이름에 유용합니다. |
| `is_output_node` | `bool` | `False` | 노드를 출력 노드로 표시하여 해당 노드와 의존성을 실행하도록 함. |
| `is_input_list` | `bool` | `False` | True일 경우, 모든 입력이 `list[type]`로 변환되며, 전달된 항목 수에 관계없이 적용됩니다. |
| `is_deprecated` | `bool` | `False` | 노드를 추천되지 않는 것으로 표시하여 사용자에게 대체품을 찾도록 알림. |
| `is_experimental` | `bool` | `False` | 노드를 실험적이라고 표시하여 사용자에게 변경 가능성을 경고함. |
| `is_dev_only` | `bool` | `False` | 개발 모드가 활성화되지 않으면 노드를 검색/메뉴에서 숨김. |
| `is_api_node` | `bool` | `False` | 노드를 Comfy API 서비스용 API 노드로 표시함. |
| `not_idempotent` | `bool` | `False` | True일 경우, 노드는 항상 재실행되며 그래프 내 다른 동일한 노드의 캐시된 출력을 절대 재사용하지 않습니다. |
| `enable_expand` | `bool` | `False` | `NodeOutput`에 노드 확장을 위한 `expand` 속성을 포함하도록 허용함. |
| `accept_all_inputs` | `bool` | `False` | True일 경우, 스키마에 정의되지 않았더라도 프롬프트의 모든 입력이 kwargs로 전달됨. |

### 공통 입력 매개변수

모든 입력 유형은 다음 기본 매개변수를 공유합니다:

| 매개변수 | 유형 | 기본값 | 설명 |
|-----------|------|---------|-------------|
| `id` | `str` | *필수* | 입력의 고유 식별자로, `execute`에서 kwargs 이름으로 사용됨. |
| `display_name` | `str` | `None` | UI에 표시되는 레이블. 기본값은 `id`임. |
| `optional` | `bool` | `False` | 입력이 선택적인지 여부. |
| `tooltip` | `str` | `None` | 마우스오버 툴팁 텍스트. |
| `lazy` | `bool` | `None` | 지연 평가를 위한 입력 표시 (참조: [지연 평가](#지연-평가-v1--v3)). |
| `raw_link` | `bool` | `None` | True일 경우, 해결된 값 대신 원본 링크 정보를 전달함. |
| `advanced` | `bool` | `None` | True일 경우, UI에서 "고급" 토글 뒤에 숨겨짐. |

위젯 입력(Int, Float, String, Boolean, Combo)은 추가로 다음을 지원합니다:

| 매개변수 | 유형 | 기본값 | 설명 |
|-----------|------|---------|-------------|
| `default` | 다양함 | `None` | 위젯의 기본값. |
| `socketless` | `bool` | `None` | True일 경우, 입력 소켓을 숨김 (위젯 전용, 들어오는 연결 없음). |
| `force_input` | `bool` | `None` | True일 경우, 위젯을 소켓 입력으로 표시하도록 강제함. |

## 고급 기능

### 숨겨진 입력

숨겨진 입력은 프롬프트 메타데이터, 노드 ID 및 기타 내부 값과 같은 실행 컨텍스트에 접근할 수 있도록 제공합니다. UI에는 표시되지 않습니다.

V1에서는 숨겨진 입력을 `INPUT_TYPES`의 `"hidden"` 키로 선언했습니다. V3에서는 Schema의 `hidden` 매개변수를 통해 선언하며, 값은 `cls.hidden`를 통해 접근합니다.

**V1:**
```python
@classmethod
def INPUT_TYPES(s):
    return {
        "required": {...},
        "hidden": {
            "unique_id": "UNIQUE_ID",
            "prompt": "PROMPT",
            "extra_pnginfo": "EXTRA_PNGINFO",
        }
    }

def execute(self, unique_id, prompt, extra_pnginfo, ...):
    # 숨겨진 값이 일반 인자로 전달됨
    print(unique_id)
```

**V3:**
```python
@classmethod
def define_schema(cls) -> io.Schema:
    return io.Schema(
        node_id="MyNode",
        inputs=[...],
        hidden=[io.Hidden.unique_id, io.Hidden.prompt, io.Hidden.extra_pnginfo],
    )

@classmethod
def execute(cls, ...) -> io.NodeOutput:
    # 숨겨진 값은 cls.hidden를 통해 접근함
    print(cls.hidden.unique_id)
    print(cls.hidden.prompt)
    print(cls.hidden.extra_pnginfo)
```

사용 가능한 숨겨진 값:

| 숨겨진 열거형 | 설명 |
|-------------|-------------|
| `io.Hidden.unique_id` | 노드의 고유 식별자로, 클라이언트 측의 id와 일치함. |
| `io.Hidden.prompt` | 클라이언트가 보낸 전체 프롬프트. |
| `io.Hidden.extra_pnginfo` | 저장된 `.png` 파일의 메타데이터에 복사된 딕셔너리. |
| `io.Hidden.dynprompt` | 실행 중에 변형될 수 있는 `DynamicPrompt` 인스턴스. |
| `io.Hidden.auth_token_comfy_org` | 프론트엔드에서 ComfyOrg 계정에 로그인하여 획득한 토큰. |
| `io.Hidden.api_key_comfy_org` | ComfyOrg에서 생성한 API 키로, 프론트엔드 로그인 생략 가능. |

<Note>
일부 숨겨진 값은 Schema 플래그에 따라 자동으로 추가됩니다. 출력 노드(`is_output_node=True`)는 자동으로 `prompt`와 `extra_pnginfo`를 받습니다. API 노드(`is_api_node=True`)는 자동으로 인증 토큰을 받습니다.
</Note>

### UI 도우미

V3는 `ui` 모듈에 미리보기 및 파일 저장과 같은 일반 패턴을 처리하는 내장 UI 도우미를 제공합니다. `io.NodeOutput`에 `ui` 매개변수를 통해 전달하세요.

#### 미리보기 도우미

미리보기 도우미는 임시 파일을 저장하고 노드 내 표시를 위한 UI 데이터를 반환합니다.

```python
from comfy_api.latest import ui

# 노드에서 이미지 미리보기
return io.NodeOutput(images, ui=ui.PreviewImage(images, cls=cls))

# 마스크 미리보기 (자동으로 3채널로 변환하여 표시)
return io.NodeOutput(mask, ui=ui.PreviewMask(mask, cls=cls))

# 오디오 미리보기
return io.NodeOutput(audio, ui=ui.PreviewAudio(audio, cls=cls))

# 텍스트 미리보기
return io.NodeOutput(ui=ui.PreviewText("Some text value"))

# 3D 모델 미리보기
return io.NodeOutput(ui=ui.PreviewUI3D(model_file, camera_info))
```

#### 저장 도우미

저장 도우미는 적절한 메타데이터를 포함해 출력 디렉토리에 파일을 저장하는 방법을 제공합니다. 주로 출력 노드에서 사용됩니다.

```python
from comfy_api.latest import ui, io

# 이미지를 저장하고 UI 데이터 반환 (가장 일반적인 패턴)
return io.NodeOutput(
    ui=ui.ImageSaveHelper.get_save_images_ui(
        images=images,
        filename_prefix=filename_prefix,
        cls=cls,  # 숨겨진 프롬프트/extra_pnginfo를 메타데이터에 전달
    )
)

# 애니메이션 PNG 저장
return io.NodeOutput(
    ui=ui.ImageSaveHelper.get_save_animated_png_ui(
        images=images,
        filename_prefix=filename_prefix,
        cls=cls,
        fps=6.0,
        compress_level=4,
    )
)

# 애니메이션 WebP 저장
return io.NodeOutput(
    ui=ui.ImageSaveHelper.get_save_animated_webp_ui(
        images=images,
        filename_prefix=filename_prefix,
        cls=cls,
        fps=6.0,
        lossless=True,
        quality=80,
        method=4,
    )
)

# 오디오 저장 (flac, mp3, opus 지원)
return io.NodeOutput(
    ui=ui.AudioSaveHelper.get_save_audio_ui(
        audio=audio,
        filename_prefix=filename_prefix,
        cls=cls,
        format="flac",
    )
)
```

<Tip>
저장/미리보기 도우미에 `cls=cls`를 전달하면, 저장된 파일에 워크플로우 메타데이터(프롬프트, extra_pnginfo)를 자동으로 포함할 수 있습니다. Schema의 숨겨진 목록에 `io.Hidden.prompt`와 `io.Hidden.extra_pnginfo`를 포함하거나, `is_output_node=True`를 설정해 자동으로 추가하도록 하세요.
</Tip>

#### Raw UI Dict 반환

도우미가 없는 UI 데이터를 직접 반환해야 한다면, 딕셔너리를 직접 전달할 수 있습니다:

```python
return io.NodeOutput(ui={"images": results})
```

### 출력 노드

파일 저장과 같은 부수적 효과를 발생시키는 노드의 경우, V1과 동일하게 출력 노드로 표시하면 노드의 컨텍스트 창에 `run` 재생 버튼이 표시되어 그래프의 부분 실행이 가능합니다.

```python
@classmethod
def define_schema(cls) -> io.Schema:
    return io.Schema(
        node_id="SaveNode",
        inputs=[...],
        outputs=[],  # 비워둘 필요 없음.
        is_output_node=True  # 출력 노드로 표시
    )
```

### 맞춤형 유형

맞춤형 입력/출력 유형은 클래스 정의 또는 `Custom` 도우미 함수를 통해 생성합니다.

```python
from comfy_api.latest import io

# 방법 1: 클래스와 함께 데코레이터 사용
@io.comfytype(io_type="MY_CUSTOM_TYPE")
class MyCustomType:
    Type = torch.Tensor  # 파이썬 타입 힌트

    class Input(io.Input):
        def __init__(self, id: str, **kwargs):
            super().__init__(id, **kwargs)

    class Output(io.Output):
        def __init__(self, **kwargs):
            super().__init__(**kwargs)

# 방법 2: Custom 도우미 사용
# 도우미는 변수에 저장하지 않고 바로 사용해 편리함
MyCustomType = io.Custom("MY_CUSTOM_TYPE")
```

### 멀티타입 입력

`MultiType`는 입력이 하나 이상의 유형을 허용하도록 합니다. 이는 노드가 동일한 입력 슬롯을 통해 서로 다른 데이터 유형으로 작동할 때 유용합니다.

첫 번째 인자(`id`)가 문자열이 아닌 `Input` 클래스의 인스턴스라면, 해당 입력은 오버라이드된 값을 가진 위젯으로 사용됩니다. 그렇지 않으면 입력은 소켓 전용입니다.

```python
# 소켓 전용 멀티타입 입력 (위젯 없음)
io.MultiType.Input("input", types=[io.Image, io.Mask])

# 위젯 백업이 있는 멀티타입 입력 (연결되지 않으면 문자열 위젯 표시)
io.MultiType.Input(
    io.String.Input("model_file", default="", multiline=False),
    types=[io.File3DGLB, io.File3DGLTF, io.File3DOBJ],
    tooltip="3D 모델 파일 또는 경로 문자열",
)
```

### MatchType (유형 연관 매칭)

`MatchType`는 유형 연관 입력과 출력을 만듭니다. 사용자가 특정 유형을 MatchType 입력에 연결하면, 같은 템플릿을 공유하는 모든 입력과 출력은 자동으로 해당 유형으로 제한됩니다. Switch 및 Create List와 같은 노드가 어떤 유형에서도 작동하는 방식입니다.

```python
@classmethod
def define_schema(cls):
    # 템플릿 생성 - 같은 템플릿을 공유하는 모든 입력/출력은 유형을 매칭함
    template = io.MatchType.Template("switch")
    return io.Schema(
        node_id="SwitchNode",
        display_name="Switch",
        category="logic",
        inputs=[
            io.Boolean.Input("switch"),
            io.MatchType.Input("on_false", template=template, lazy=True),
            io.MatchType.Input("on_true", template=template, lazy=True),
        ],
        outputs=[
            io.MatchType.Output(template=template, display_name="output"),
        ],
    )
```

또한 허용되는 유형을 제한할 수도 있습니다:

```python
# Image, Mask, Latent 유형만 허용
template = io.MatchType.Template("input_type", allowed_types=[io.Image, io.Mask, io.Latent])
```

### 동적 입력

V3는 사용자 상호작용에 따라 사용 가능한 입력을 변경하는 동적 입력 유형을 도입했습니다. 이러한 기능에 대한 V1 equivalent는 없습니다.

#### Autogrow

`Autogrow`는 사용자가 더 많이 연결할수록 자동으로 늘어나는 가변 수의 입력을 생성합니다. 두 가지 템플릿 유형이 있습니다:

**TemplatePrefix**는 번호가 붙은 접두사를 가진 입력을 생성합니다 (예: `image0`, `image1`, `image2`...):

```python
@classmethod
def define_schema(cls):
    autogrow_template = io.Autogrow.TemplatePrefix(
        input=io.Image.Input("image"),  # 각 입력의 템플릿
        prefix="image",                  # 생성된 입력 이름의 접두사
        min=2,                           # 최소 표시 입력 수
        max=50,                          # 최대 허용 입력 수
    )
    return io.Schema(
        node_id="BatchImagesNode",
        display_name="배치 이미지",
        category="image",
        inputs=[io.Autogrow.Input("images", template=autogrow_template)],
        outputs=[io.Image.Output()],
    )

@classmethod
def execute(cls, images: io.Autogrow.Type) -> io.NodeOutput:
    # 'images'는 입력 이름과 값을 매핑한 딕셔너리
    image_list = list(images.values())
    return io.NodeOutput(batch(image_list))
```

**TemplateNames**는 특정 이름을 가진 입력을 생성합니다:

```python
template = io.Autogrow.TemplateNames(
    input=io.Float.Input("float"),
    names=["x", "y", "z"],  # 각 입력의 명시적 이름
    min=1,                    # 최소 표시 입력 수
)
```

Autogrow는 MatchType과 결합해 유형 매칭된 입력 목록을 만들 수 있습니다:

```python
@classmethod
def define_schema(cls):
    template_matchtype = io.MatchType.Template("type")
    template_autogrow = io.Autogrow.TemplatePrefix(
        input=io.MatchType.Input("input", template=template_matchtype),
        prefix="input",
    )
    return io.Schema(
        node_id="CreateList",
        display_name="목록 생성",
        category="logic",
        is_input_list=True,
        inputs=[io.Autogrow.Input("inputs", template=template_autogrow)],
        outputs=[
            io.MatchType.Output(
                template=template_matchtype,
                is_output_list=True,
                display_name="목록",
            ),
        ],
    )
```

#### DynamicCombo

`DynamicCombo`는 선택한 옵션에 따라 다른 입력을 표시하거나 숨기는 드롭다운을 만듭니다. 이는 서로 다른 모드마다 다른 매개변수가 필요한 노드에 유용합니다.

```python
@classmethod
def define_schema(cls):
    return io.Schema(
        node_id="ResizeNode",
        display_name="크기 조정",
        category="변환",
        inputs=[
            io.Image.Input("image"),
            io.DynamicCombo.Input("resize_type", options=[
                io.DynamicCombo.Option("차원별 스케일", [
                    io.Int.Input("width", default=512, min=0, max=8192),
                    io.Int.Input("height", default=512, min=0, max=8192),
                ]),
                io.DynamicCombo.Option("승수별 스케일", [
                    io.Float.Input("multiplier", default=1.0, min=0.01, max=8.0),
                ]),
                io.DynamicCombo.Option("메가픽셀로 스케일", [
                    io.Float.Input("megapixels", default=1.0, min=0.01, max=16.0),
                ]),
            ]),
        ],
        outputs=[io.Image.Output()],
    )

@classmethod
def execute(cls, image, resize_type: dict) -> io.NodeOutput:
    # resize_type는 선택된 옵션 키와 그 입력을 포함한 딕셔너리
    selected = resize_type["resize_type"]
    if selected == "차원별 스케일":
        width = resize_type["width"]
        height = resize_type["height"]
        # ...
    elif selected == "승수별 스케일":
        multiplier = resize_type["multiplier"]
        # ...
```

DynamicCombo 옵션은 중첩될 수도 있습니다:

```python
io.DynamicCombo.Input("combo", options=[
    io.DynamicCombo.Option("옵션1", [io.String.Input("문자열")]),
    io.DynamicCombo.Option("옵션2", [
        io.DynamicCombo.Input("하위콤보", options=[
            io.DynamicCombo.Option("하위옵션1", [io.Float.Input("x"), io.Float.Input("y")]),
            io.DynamicCombo.Option("하위옵션2", [io.Mask.Input("마스크", optional=True)]),
        ])
    ]),
])
```

### 비동기 실행

V3는 비동기 `execute` 메서드를 지원합니다. I/O 작업, API 호출 또는 기타 비동기 작업을 수행하는 노드에 유용합니다. 간단히 `execute`를 `async`로 선언하세요:

```python
@classmethod
async def execute(cls, prompt, **kwargs) -> io.NodeOutput:
    result = await some_async_operation(prompt)
    return io.NodeOutput(result)
```

### ComfyAPI

`ComfyAPI` 클래스는 진행률 보고 및 노드 교체 등록과 같은 ComfyUI 런타임 서비스에 접근할 수 있게 합니다. 이를 가져오고 인스턴스를 생성하세요:

```python
from comfy_api.latest import ComfyAPI

api =
