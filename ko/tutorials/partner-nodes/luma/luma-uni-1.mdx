**ComfyUI**에서 Luma **Uni-1**는 통합 이미지 작업을 위한 **파트너 파트너 노드**로 나타납니다. **생성** 그래프는 프롬프트(선택적 참조 포함)를 기반으로 새 이미지를 생성하며, **수정** 그래프는 기존 이미지를 입력으로 받아 그대로 편집합니다. 두 작업 모두 일반적인 워크플로우를 통해 진행됩니다—Luma 노드에서 **이미지 로드**/ **이미지 저장**을 연결하고, 프롬프트, 시드, 종횡비 및 참조 슬롯을 설정한 다음, 그래프를 로컬로 대기시키거나 **Comfy Cloud**에서 템플릿을 열어보세요.

Luma는 Uni-1을 비확산, 디코더 전용 자가회귀 모델로 설명하며, 그림을 그리기 전에 프롬프트를 기반으로 추론합니다. 캔버스에서 가장 중요한 것은 **생성 vs 수정**을 선택하고, 참조를 명확히 표시하며, 시드를 활용해 반복하는 것입니다.

이 페이지에서는 간단한 그래프부터 다중 참조 설정까지, Uni-1을 **ComfyUI 내부에서** 어떻게 사용하는지 중점적으로 다룹니다.

## Uni-1의 차별점

- **두 가지 명확한 모드**: 이미지 생성(새로운 것을 생성)과 이미지 수정(기존 것을 편집)
- 최대 9개의 참조 이미지, 각각 정해진 역할을 가짐
- **틈새 및 특정 시각 스타일에 대한 강력한 제어**
- 재현성과 제어된 반복을 위한 **시드 지원**
- 초장형부터 초광형까지 **아홉 가지 종횡비**
- 실제로 읽을 수 있는 **텍스트 렌더링**
- 현실적 맥락을 제공하는 **웹 검색 기반**
- **다국어 프롬프트**
- 시간적 일관성을 갖춘 **멀티패널 출력**

모드와 참조 제어 방법을 이해하면 Uni-1을 완벽히 활용할 수 있습니다.

## 강점

Uni-1은 다양한 작업에서 뛰어난 성능을 발휘합니다:

- 재료의 정확성을 갖춘 **사실주의**
- 강력한 미적 제어를 통한 **일러스트 및 스타일화된 아트**
- 오래된 사진 복원 및 빈티지 재현
- **초현실적이고 개념적인** 구성
- 이미지 내에서 실제로 읽힐 수 있는 **텍스트 렌더링**—인포그래픽 및 포스터에 적합
- 이미지 편집 및 다중턴 개선
- 정체성 보존을 위한 **참조 유도 생성**
- 여러 프레임에서 일관된 캐릭터/장면을 갖춘 **멀티패널 출력**

## 핵심 구분: 생성 vs 수정

Uni-1의 모든 작업은 한 가지 질문에서 시작됩니다: **새로운 것을 만들고 있나요, 아니면 이미 존재하는 것을 바꾸고 있나요?**

| 모드 | 무엇을 하는가 | 언제 사용하나요 |
| --- | --- | --- |
| **이미지 생성** | 완전히 새로운 구성물을 생성합니다. 참조를 통해 영감을 받을 수 있습니다. | "이 사진 스타일로 새로운 장면을 만들어라" |
| **이미지 수정** | 특정 입력 이미지를 편집합니다. 다른 지시가 없으면 구성과 구조를 유지합니다. | "이 사진을 야간처럼 만들어라" |

의문이 들 때:

- 출력물이 **입력의 버전처럼 보여야 한다면** 수정을 사용하세요.
- **영감을 받았지만 새롭게 느껴져야 한다면** 생성을 사용하세요.

## 핵심 매개변수

| 매개변수 | 설명 |
| --- | --- |
| **프롬프트** | 주요 제어 요소입니다. 최대 6,000자까지 가능합니다. 정확하게 작성하세요. |
| **종횡비** | 화질이 아니라 프레이밍을 제어합니다. 사용 사례에 따라 선택하세요. |
| **시드** | 동일한 시드 + 동일한 프롬프트 → 동일한 결과. 동일한 시드 + 변경된 프롬프트 → 제어된 변형. 시드 없음 → 탐색. |
| **참조 이미지**(생성) | 서로 다른 측면을 안내하기 위해 최대 9개의 이미지를 사용합니다. |
| **소스 이미지**(수정) | 편집할 이미지입니다. 크기는 자동으로 유지됩니다. |

## 참조 이미지 사용법

참조는 모델에게 그것들이 무엇을 위한 것인지 알려줘야만 작동합니다. 다음과 같은 구조를 사용하세요:

```text
IMAGE1([설명])을 [역할] 참조로 사용하세요.
```

가능한 역할: **스타일**, **캐릭터**, **구성**, **색상 팔레트**, **조명**, **텍스처**, **분위기**.

역할을 지정하지 않으면 모델이 추측하게 되며, 추측은 신뢰할 수 없습니다.

### 생성 모드 예시

**스타일 참조**

```text
IMAGE1(느슨한 붓놀림과 따뜻한 석양 색조의 인상파 유화)을 스타일 참조로 사용하세요. IMAGE1의 색상 팔레트와 화가의 텍스처를 적용해 30대 여성의 초상화를 생성하세요. 피사체는 새로 만들어야 합니다.
```

**캐릭터 참조**

```text
IMAGE1(짧은 구리빛 머리카락과 주근깨가 있는 여성)을 캐릭터 참조로 사용하세요. 그녀의 특징을 유지하세요. 새로운 장면을 생성하세요: 그녀가 은은한 조명의 카페에 앉아 있습니다.
```

**다중 참조**

```text
IMAGE1을 색상 팔레트 참조로, IMAGE2를 조명 참조로, IMAGE3를 구성 참조로 사용하세요. 밤에 비에 젖은 거리를 걷는 외로운 인물을 생성하세요.
```

### 수정 모드 예시

수정 모드에서는 명확성이 중요합니다. 항상 **무엇을 바꿀지**와 **변경되지 말아야 할 것**을 명확히 지정하세요.

```text
시간대를 골든아워로 변경하세요. 하늘, 빛의 방향, 그림자, 색온도를 업데이트하세요. 모든 피사체와 구성은 그대로 유지하세요.
```

```text
비를 더 세게 만들어 지면에 반사를 추가하세요. 다른 모든 요소는 그대로 유지하세요.
```

## 프롬프트 작성 가이드라인

권장 길이:

- **텍스트-to-이미지** → 80~250자
- **참조 유도** → 100~300자
- **수정** → 30~100자

모호한 표현("아름답다", "놀랍다"), 중복된 문구, 상충되는 지시사항은 피하세요. 대신 구체적이고 명확한 미학을 사용하세요:

- `골든아워, 85mm 렌즈, 얕은 심도`
- `1970년대 이탈리아 지알로 영화 포스터, 고대비 컬러 블록`

정밀함이 일반성을 능가합니다.

## 종횡비

이미지가 어디에 사용될지에 따라 선택하세요:

| 종횡비 | 사용 사례 |
| --- | --- |
| 1:1 | 소셜 게시물 |
| 9:16 | 세로 동영상 |
| 16:9 | 와이드스크린 |
| 3:2 / 2:3 | 사진 촬영 |
| 2:1 / 3:1 | 시네마틱 / 파노라마 |
| 1:2 / 1:3 | 초장형 |

수정 모드에서는 종횡비가 소스 이미지에 고정됩니다.

## 시드: 제어와 재현성

시드는 실험을 체계로 만듭니다.

- **고정된 시드** → 일관성
- **시드 없음** → 탐색

워크플로우:

1. 시드 없이 탐색하세요
2. 강력한 결과를 찾으세요
3. 시드를 고정하세요
4. 한 번에 하나씩 변수를 변경하세요

프롬프트와 시드를 함께 저장하세요. 이것이 바로 재사용 가능한 레시피입니다.

## 고급 기술

### 캐릭터 일관성

1. 깨끗하고 정면을 보이는 참조 이미지를 생성하세요
2. 이를 모든 장면에서 `IMAGE1(캐릭터)`로 재사용하세요
3. 프롬프트에서 레이블을 동일하게 유지하세요

### 다중 참조 아키텍처

각 이미지에 하나의 역할을 부여하세요:

- IMAGE1 → 캐릭터
- IMAGE2 → 스타일
- IMAGE3 → 조명
- IMAGE4 → 환경

프롬프트를 다음과 같이 마무리하세요:

```text
각 참조를 자신이 맡은 레이어에만 권한을 가진다고 생각하세요.
```

### 생성 → 수정 체인

생성을 통해 구성물을 탐색한 후, 수정을 통해 세부사항을 다듬으세요. 이것이 가장 강력한 워크플로우 중 하나입니다.

### 반복적 개선

1. 탐색(시드 없음)
2. 시드 고정
3. 한 번에 하나씩 변수를 변경하세요
4. 결과를 문서화하세요

느리게 느껴지지만, 실제로는 더 빠릅니다. 왜냐하면 무엇이 바뀌었는지 항상 알 수 있기 때문입니다.

## 문제 해결

| 문제 | 해결책 |
| --- | --- |
| 참조가 무시됨 | 각 참조를 명확히 레이블 지정하세요 |
| 수정이 너무 많이 변경됨 | 변경되지 말아야 할 것을 명확히 지정하세요 |
| 출력이 일관되지 않음 | 시드를 고정하세요 |
| 프롬프트가 부분적으로 무시됨 | 충돌을 제거하거나 단계로 나누세요 |
| 출력이 참조 이미지와 같음 | 수정 모드에 있을 수 있습니다 |
| 캐릭터가 일관되지 않음 | 표준 참조 이미지를 재사용하세요 |

## 빠른 참고

**이미지 생성**

- 새로운 구성
- 텍스트 + 최대 9개의 참조
- 묘사적인 프롬프트

**이미지 수정**

- 기존 이미지 편집
- 소스 이미지 + 참조
- 직접적이고 정교한 프롬프트

## 황금률

1. 모든 참조에 레이블을 붙이세요
2. 수정 모드에서는 반드시 변경되지 말아야 할 것을 명시하세요
3. 개선할 때 한 번에 하나씩 변수를 변경하세요
4. 재현성을 위해 프롬프트와 시드를 저장하세요
5. 생성 = 새로운 장면, 수정 = 편집

## 핵심 요약

Uni-1을 완벽히 활용하려면 세 가지를 잘 알아야 합니다: **올바른 모드 선택 → 참조 제어 → 의도를 가지고 반복하기**. 이 세 가지가 확실히 자리 잡으면 더 이상 추측하지 않고 직접 제어하게 됩니다.
