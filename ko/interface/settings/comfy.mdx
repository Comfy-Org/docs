## 모델 라이브러리

모델 라이브러리는 ComfyUI 사이드바 메뉴의 모델 관리 기능을 말합니다. 이 기능을 사용해 `ComfyUI/models` 및 추가로 구성된 모델 폴더의 모델을 볼 수 있습니다.

![모델 라이브러리](/images/interface/sidepanel/model_library.jpg)

### 모델 라이브러리 트리 보기에서 표시할 이름
- **기본값**: 제목
- **기능**: 모델 라이브러리 트리 보기에서 표시할 이름 형식을 선택하며, 현재는 파일명과 제목만 지원합니다.

### 모든 모델 폴더 자동 로드
- **기본값**: 비활성화됨
- **기능**: 모델 라이브러리를 클릭할 때 모든 폴더의 모델 파일을 자동으로 탐지할지 여부입니다. 활성화하면 로딩 지연이 발생할 수 있습니다(모든 폴더를 순회해야 함). 비활성화되면 해당 폴더 이름을 클릭할 때만 폴더 내 파일이 로드됩니다.

## 노드

ComfyUI의 반복 과정에서 일부 노드를 조정하고 일부 노드를 활성화할 것입니다. 이러한 노드는 향후 버전에서 주요 변경이나 제거될 수 있습니다. 그러나 호환성을 보장하기 위해, 폐기된 노드는 삭제되지 않았습니다. 아래 설정을 사용해 **실험적 노드**와 **폐기된 노드**를 표시할지 여부를 활성화할 수 있습니다.

### 검색에서 폐기된 노드 표시
- **기본값**: 비활성화됨
- **기능**: 검색에서 폐기된 노드를 표시할지 여부를 제어합니다. 폐기된 노드는 기본적으로 UI에서 숨겨져 있지만, 기존 워크플로우에서는 여전히 유효합니다.

![검색에서 폐기된 노드 표시](/images/interface/setting/comfy/depr_node.jpg)


### 검색에서 실험적 노드 표시
- **기본값**: 활성화됨
- **기능**: 검색에서 실험적 노드를 표시할지 여부를 제어합니다. 실험적 노드는 일부 신기능 지원이지만 완벽히 안정적이지 않으며, 향후 버전에서 변경되거나 제거될 수 있습니다.

![검색에서 실험적 노드 표시](/images/interface/setting/comfy/beta_node.jpg)


## 노드 검색창

### 노드 제안 개수
- **기본값**: 5
- **기능**: 관련 노드 컨텍스트 메뉴에서 추천 노드의 개수를 수정하는 데 사용됩니다. 값이 클수록 더 많은 관련 추천 노드가 표시됩니다.

![노드 제안 개수](/images/interface/setting/comfy/node_suggestions.jpg)

### 검색 결과에 노드 빈도 표시
- **기본값**: 비활성화됨
- **기능**: 검색 결과에 노드 사용 빈도를 표시할지 여부를 제어합니다.

![검색 결과에 노드 빈도 표시](/images/interface/setting/comfy/node_frequency.png)

### 검색 결과에 노드 ID 이름 표시
- **기본값**: 비활성화됨
- **기능**: 검색 결과에 노드 ID 이름을 표시할지 여부를 제어합니다.

![검색 결과에 노드 ID 이름 표시](/images/interface/setting/comfy/node_id_name.jpg)


### 검색 결과에 노드 카테고리 표시
- **기본값**: 활성화됨
- **기능**: 검색 결과에 노드 카테고리를 표시할지 여부를 제어하며, 사용자가 노드 분류 정보를 이해하는 데 도움을 줍니다.

### 노드 미리보기
- **기본값**: 활성화됨
- **기능**: 검색 결과에 노드 미리보기를 표시할지 여부를 제어하며, 이를 통해 빠르게 노드를 미리볼 수 있어 편리합니다.

### 노드 검색창 구현
- **기본값**: 기본
- **기능**: 노드 검색창의 구현 방식을 선택합니다(실험적 기능). `litegraph (레거시)`를 선택하면 초기 ComfyUI 검색창으로 전환됩니다.

## 노드 위젯

### 위젯 제어 모드
- **옵션**: 전, 후
- **기능**: 노드 위젯 값 업데이트 시점이 워크플로우 실행 전인지 후인지 여부를 제어하며, 예를 들어 시드 값 업데이트 등을 제어합니다.

### 텍스트 영역 위젯 맞춤법 검사
- **기본값**: 비활성화됨
- **기능**: 텍스트 영역 위젯이 맞춤법 검사를 활성화할지 여부를 제어하며, 텍스트 입력 중 맞춤법 검사 기능을 제공합니다. 이 기능은 브라우저의 맞춤법 검사 속성을 통해 구현됩니다.

## 큐

### 큐 기록 크기
- **기본값**: 100
- **기능**: 사이드바 큐 기록 패널에 기록되는 큐 기록 크기를 제어합니다. 값이 클수록 더 많은 큐 기록이 기록됩니다. 숫자가 많으면 페이지 로딩 시 메모리 소모도 증가합니다.

## 큐 버튼

### 일괄 처리 한도
- **기본값**: 100
- **기능**: 한 번의 클릭으로 큐에 추가할 수 있는 최대 작업 수를 설정하며, 실수로 너무 많은 작업을 큐에 추가하는 것을 방지합니다.

## 검증

### 노드 정의 검증 (느림)
- **기본값**: 비활성화됨
- **기능**: 시작 시 모든 노드 정의를 검증할지 여부를 제어합니다(느림). 노드 개발자에게만 권장됩니다. 활성화되면 시스템은 Zod 스키마를 사용해 각 노드 정의를 엄격하게 검증합니다. 이 기능은 더 많은 메모리와 시간을 소모합니다.
- **오류 처리**: 실패한 노드 정의는 건너뛰며 경고 정보가 콘솔에 출력됩니다.

<Tip>
    모든 노드 정의에 대해 상세한 스키마 검증을 수행해야 하므로 이 기능은 시작 시간을 크게 늘릴 수 있으므로 기본적으로 비활성화되어 있으며, 노드 개발자에게만 권장됩니다.
</Tip>

### 워크플로우 검증
- **기본값**: 활성화됨
- **기능**: 워크플로우의 구조와 연결이 올바른지 확인합니다. 활성화되면 시스템은 `useWorkflowValidation().validateWorkflow()`를 호출해 워크플로우 데이터를 검증합니다.
- **검증 과정**: 검증 과정은 두 단계로 이루어집니다:
    - 스키마 검증: Zod 스키마를 사용해 워크플로우 구조를 검증합니다.
    - 링크 복구: 노드 간 연결 문제를 확인하고 복구합니다.
- **오류 처리**: 검증 실패 시 오류 메시지가 표시되지만, 워크플로우 로딩이 차단되지는 않습니다.

## 창

### 창 닫을 때 확인 표시
- **기본값**: 활성화됨
- **기능**: 수정되었으나 저장되지 않은 워크플로우가 있을 때, 브라우저 창이나 탭을 닫을 때 확인을 표시할지 여부를 제어하며, 실수로 창을 닫아 저장되지 않은 워크플로우가 손실되는 것을 방지합니다.

## 워크플로우

### 워크플로우 상태 유지 및 페이지 재로드 시 복원
- **기본값**: 활성화됨
- **기능**: 페이지 재로드 시 워크플로우 상태를 복원해 페이지 새로고침 후에도 워크플로우 내용을 유지합니다.

### 자동 저장
- **기본값**: 비활성화됨
- **기능**: 워크플로우의 자동 저장 동작을 제어하며, 워크플로우 변경 사항을 자동으로 저장해 데이터 손실을 방지합니다.

### 자동 저장 지연 시간 (ms)
- **기본값**: 1000
- **기능**: 자동 저장의 지연 시간을 설정하며, 자동 저장이 '지연 후'로 설정된 경우에만 적용됩니다.

### 워크플로우 삭제 시 확인 표시
- **기본값**: 활성화됨
- **기능**: 사이드바에서 워크플로우를 삭제할 때 확인 대화상자를 표시할지 여부를 제어하며, 중요한 워크플로우를 실수로 삭제하지 않도록 방지합니다.

### 워크플로우에서 캔버스 위치와 확대/축소 수준 저장 및 복원
- **기본값**: 활성화됨
- **기능**: 워크플로우에서 캔버스 위치와 확대/축소 수준을 저장하고 복원해, 워크플로우를 다시 열 때 이전 뷰 상태를 복원합니다.

### 열린 워크플로우 위치
- **옵션**: 사이드바, 상단바, 상단바(두 번째 행)
- **기본값**: 상단바
- **기능**: 열린 워크플로우 탭의 표시 위치를 제어하며, 현재는 사이드바, 상단바, 상단바(두 번째 행)만 지원합니다.

### 워크플로우 저장 시 파일명 프롬프트 표시
- **기본값**: 활성화됨
- **기능**: 워크플로우를 저장할 때 파일명 입력 프롬프트를 표시할지 여부를 제어하며, 사용자가 워크플로우 파일명을 맞춤 설정할 수 있도록 합니다.

### 워크플로우 저장 시 노드 ID 정렬
- **기본값**: 비활성화됨
- **기능**: 워크플로우 저장 시 노드 ID를 정렬할지 여부를 결정하며, 워크플로우 파일 형식을 더욱 표준화하고 버전 관리에 편리하게 만듭니다.

### 누락된 노드 경고 표시
- **기본값**: 활성화됨
- **기능**: 워크플로우에서 누락된 노드에 대한 경고를 표시할지 여부를 제어하며, 사용자가 워크플로우에서 사용할 수 없는 노드를 식별하도록 돕습니다.

### 누락된 모델 경고 표시
- **기본값**: 활성화됨
- **기능**: 모델 파일 로딩 시 프롬프트를 위해 워크플로우 파일의 위젯 값에 모델 링크 정보를 추가하는 것을 지원합니다. 활성화되면 현지에 해당 모델 파일이 없을 경우 워크플로우에서 누락된 모델에 대한 경고가 표시됩니다.

### 워크플로우 지우기 시 확인 요구
- **기본값**: 활성화됨
- **기능**: 워크플로우를 지울 때 확인 대화상자를 표시할지 여부를 제어하며, 실수로 워크플로우 내용이 지워지는 것을 방지합니다.

### 노드 ID를 워크플로우에 저장
- **기본값**: 활성화됨
- **기능**: 워크플로우 저장 시 노드 ID를 저장할지 여부를 제어하며, 워크플로우 파일 형식을 더욱 표준화하고 버전 관리에 편리하게 만듭니다.
