노드 2.0은 이제 Comfy 데스크톱, 포터블 및 안정적인 릴리스에서 사용할 수 있습니다. 이번 업데이트를 통해 노드 시스템이 LiteGraph.js 캔버스 렌더링에서 Vue 기반 아키텍처로 전환되어 더 빠른 반복과 풍부한 상호작용을 실현했습니다.

## 왜 이런 변화가 필요한가요?

기존 캔버스 렌더링 시스템은 개발의 병목 현상으로 자리 잡았습니다. 작은 UI 변경만으로도 깊은 수정이 필요했고, 구현하는 데 며칠이 걸릴 수도 있었습니다. 이는 커뮤니티의 피드백에 신속히 대응하는 속도를 저하시켰으며, 개발자들이 노드를 맞춤화하기 어렵게 만들었습니다.

새로운 Vue 기반 시스템은 다음을 가능하게 합니다:

- **더 빠른 기능 개발** - UI 변경사항을 보다 신속히 구현할 수 있습니다.
- **더 풍부한 상호작용** - 동적 위젯, 확장 가능한 노드, 더 나은 구성 요소들
- **더 유연한 기반** - 미래 도구와 맞춤화에 더 나은 지원

## 현재 상태와 알려진 문제점

우리는 적극적으로 개선 작업을 진행하고 있으며, 현재 한계에 대해 투명하게 공유하고자 합니다:

| 분야 | 상태 |
|------|--------|
| **성능** | 여전히 캔버스 수준의 성능을 목표로 최적화 중입니다. 특정 시나리오에서는 다르게 동작할 수 있습니다. |
| **호환성** | 일부 커스텀 노드는 완벽한 지원을 위해 업데이트가 필요할 수 있습니다. |
| **특수 사례** | 극단적인 확대/축소 수준이나 매우 큰 워크플로우 내에서는 약간의 시각적 문제가 나타날 수 있습니다. |

우리는 피드백을 수집해 개선 우선순위를 정하고 있습니다. LiteGraph.js 렌더링 시스템은 계속 사용 가능하며, 언제든지 두 시스템 간에 전환할 수 있습니다.

## 사용 방법

### 노드 2.0 활성화하기

1. [cloud.comfy.org](https://cloud.comfy.org)를 방문하거나 Comfy 데스크톱, 포터블 또는 안정적인 릴리스의 최신 버전으로 업데이트하세요.
2. ComfyUI를 실행할 때 상단 배너의 **사용해 보기**를 클릭하세요.

### 테스트할 내용

일상적인 워크플로우에서 노드 2.0을 사용해 보고 우리에게 피드백을 공유해 주세요:

- **기본 상호작용** - 드래그, 크기 조정, 선택
- **일상적인 워크플로우** - 실행, 파라미터, 커스텀 노드
- **성능** - 특히 복잡한 워크플로우에서 확인해 주세요.

## 피드백 제공 방법

문제를 발견했거나 상호작용 개선을 위한 제안이 있으신가요?

- **지원 센터** - [ComfyUI 지원 센터](https://support.comfy.org)
- **GitHub 이슈** - [ComfyUI_frontend에서 이슈 생성](https://github.com/Comfy-Org/ComfyUI_frontend/issues)

<Tip>
문제를 보고할 때 재현 단계, 스크린샷, 시스템 정보를 포함하면 매우 도움이 됩니다.
</Tip>

## 다시 전환하기

문제가 발생한 경우:

1. **ComfyUI 로고**를 클릭해 메뉴를 열고
2. **노드 2.0**을 토글해 이전 인터페이스로 돌아가세요.

![ComfyUI 메뉴에서 노드 2.0 토글 위치](/images/interface/nodes_2.0.jpg)

여러분의 피드백은 개선 우선순위를 정하는 데 도움이 됩니다. 설정에서 노드 2.0을 활성화하고 새로운 노드 디자인을 탐험해 보세요!
