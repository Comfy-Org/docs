`파트너 노드`를 사용하려면 크레딧이 필요하므로, 계정 시스템은 크레딧 관리와 인증 지원을 제공하여 사용자가 최첨단 폐쇄형 모델 API에 편리하게 접근할 수 있도록 합니다.

현재 다음과 같은 로그인 방식을 지원합니다:

- 이메일 로그인
- 구글 로그인
- 깃허브 로그인
- API 키 로그인 (화이트리스트에 등록되지 않은 사이트 인증용)

이 문서에서 관련된 로그인 요구사항과 설명을 제공하겠습니다.

## ComfyUI 버전 요구사항

계정 시스템을 사용하려면 최소한 [ComfyUI v0.3.0](https://github.com/Comfy-Org/ComfyUI/releases/tag/v0.3.30)을 사용해야 할 수 있습니다. 해당 프론트엔드 버전이 최소 `1.17.11` 이상인지 확인하세요. 때때로 프론트엔드가 설치에 실패해 이전 버전으로 되돌아가는 경우가 있으니, `설정` -> `정보`에서 프론트엔드 버전이 `1.17.11`보다 높은지 확인해 주세요.

일부 지역에서는 네트워크 제한으로 인해 로그인 API에 정상적으로 접속하지 못해 시간 초과나 실패가 발생할 수 있습니다. 로그인하기 전에 **네트워크 환경이 해당 API에 대한 접근을 제한하지 않는지 반드시 확인**하고, 구글이나 깃허브와 같은 사이트에 접속할 수 있는지 확인해 주세요.

<Tip>
아직 빠른 반복 업데이트 중이므로 관련 기능이 변경될 수 있습니다. 특별한 상황이 없다면 최신 버전으로 업데이트해 관련 기능을 지원받으시기 바랍니다.
</Tip>

## 네트워크 요구사항

ComfyUI 계정에 로그인하려면 안전한 네트워크 환경에 있어야 합니다:

- `127.0.0.1` 또는 `localhost`로부터만 접속을 허용합니다.
- `--listen` 파라미터를 사용해 로컬 네트워크를 통해 파트너 노드에 접근하는 것은 지원되지 않습니다.
- 비SSL 인증서를 사용하거나 `https`로 시작하지 않는 사이트에서는 성공적으로 로그인하지 못할 수 있습니다.
- 화이트리스트에 등록되지 않은 사이트에서는 로그인하지 못할 수 있습니다(다만 현재는 API 키를 이용해 로그인할 수 있습니다).
- 당사 서비스에 정상적으로 연결될 수 있는지 확인하세요(일부 지역에서는 프록시가 필요할 수 있습니다).

## 로그인 후 상태

로그인 후 ComfyUI 인터페이스 상단 메뉴바에 로그인 버튼이 표시됩니다. 이 버튼을 통해 해당 로그인 인터페이스를 열고 설정 메뉴에서 해당 계정의 로그아웃을 진행할 수 있습니다.

## 자주 묻는 질문

<AccordionGroup>
  <Accordion title="로그인 기기 제한이 있나요?">
   로그인 기기를 제한하지 않습니다. 어떤 기기에서도 계정에 로그인할 수 있지만, 다른 기기에서도 계정 정보에 접근할 수 있으니 공공 장소의 기기에서는 로그인하지 마세요.
  </Accordion>

  <Accordion title="LAN 환경에서 어떻게 로그인하나요?">
  현재 LAN 환경에서는 API 키 로그인만 지원됩니다. LAN을 통해 ComfyUI 서비스에 접근하는 경우 API 키를 사용해 로그인해 주세요.
  </Accordion>
   <Accordion title="왜 일부 웹사이트에서 로그인할 수 없나요?">
  저희 로그인 서비스에는 화이트리스트가 적용되어 일부 서버에 배포된 ComfyUI에서는 로그인하지 못할 수 있습니다. 이런 경우 API 키 로그인을 사용해 해결할 수 있습니다.
  </Accordion>
</AccordionGroup>
