Legacy Desktop이 이미 설치되어 있다면 Comfy Desktop이 이를 감지하고 마이그레이션하여 Chooser 뷰에 새로운 Standalone 설치를 만듭니다.

## 마이그레이션되는 항목

Legacy Desktop은 단일 설치 앱이었습니다. Comfy Desktop은 다중 설치 관리자입니다. 마이그레이션하면:

- **워크플로와 설정** — 새로운 Standalone 설치로 복사됩니다.
- **커스텀 노드** — 검사되어 새로운 환경에 재설치됩니다.
- **모델** — 기존 모델 폴더가 공유 디렉터리로 추가됩니다(파일은 그대로 유지).
- **입력 파일 및 출력** — 새로운 설치의 공유 디렉터리로 복사됩니다.
- **Python 환경** — 새 Python 환경이 생성됩니다(Legacy의 번들 Python은 재사용되지 않음).
- **ComfyUI 버전** — 최신 버전이 새 Standalone에 설치됩니다.

Legacy Desktop 설치 자체는 마이그레이션 중에 **수정되지 않습니다**. 백업으로 시스템에 남아 있습니다.

## 마이그레이션 후 변경 사항

마이그레이션이 완료되면 새 Standalone 설치는 다른 Comfy Desktop 설치와 동일하게 작동합니다.

- **실행** — Chooser 뷰에서 새 설치 카드의 재생 버튼을 클릭하여 ComfyUI를 시작합니다.
- **포트** — Legacy Desktop은 기본적으로 `8000` 포트를 사용했습니다. 새 설치도 이 포트 설정(및 기타 커스텀 시작 인자)을 이어받습니다.
- **관리** — Manage 패널을 사용하여 설치를 업데이트, 스냅샷, 구성합니다.
- **업데이트** — Standalone 설치는 Manage 패널을 통해 독립적으로 업데이트할 수 있습니다. Legacy Desktop과 달리 업데이트할 때마다 새 빌드를 다운로드할 필요가 없습니다.
- **다중 인스턴스** — 각각 고유한 ComfyUI 버전과 구성으로 여러 개의 Standalone 설치를 나란히 운영할 수 있습니다.
- **공유 저장소** — 기본적으로 모델은 모든 Standalone 설치에서 공유됩니다. 입력 및 출력 디렉터리는 기존 legacy 작업 공간을 가리키도록 고정되어 기존 파일이 그대로 유지됩니다.

## 이전 링크가 여전히 작동하나요?

Legacy Desktop은 기본적으로 `8000` 포트에서 실행되었습니다. 새 설치도 legacy 구성에서 **포트를 이어받기** 때문에 `http://localhost:8000`이 여전히 작동할 것입니다. `--port 8188`과 같은 커스텀 CLI 인자가 구성되어 있었다면 이것도 이어받습니다.

URL 단축기, 북마크, 또는 다른 주소로 Legacy Desktop 앱을 참조하는 외부 통합을 사용 중이라면 새 설치의 Startup Args 탭을 확인하여 적절히 업데이트해야 할 수 있습니다.

## 단계별 마이그레이션

<Steps>
<Step title="Comfy Desktop 설치">
[공식 다운로드 페이지](https://comfy.org/download)에서 Comfy Desktop을 다운로드하여 설치합니다.
</Step>

<Step title="Comfy Desktop 시작">
Comfy Desktop을 처음 열면 시스템의 Legacy Desktop 설치를 자동으로 감지하고 마이그레이션 프롬프트를 표시합니다.
</Step>

<Step title="마이그레이션 확인">
**지금 마이그레이션**을 클릭하여 진행합니다. Comfy Desktop이 다음을 수행합니다:

1. 기존 설치 검사(커스텀 노드, 모델, 워크플로)
2. 새 Python 환경과 최신 ComfyUI로 새로운 Standalone 설치 생성
3. 사용자 데이터(워크플로, 설정 등) 복사
4. 입력 파일 및 출력 복사
5. 모델 폴더를 공유 디렉터리로 추가
6. 새 환경에 커스텀 노드 설치

완료 후 Chooser 뷰에 새 Standalone 설치 카드가 나타납니다.
</Step>

<Step title="확인">
새 설치를 시작하고 다음을 확인합니다:
- 워크플로가 사용 가능한지
- 커스텀 노드가 설치되었는지
- 모델에 접근할 수 있는지
- 서버가 이어받은 URL에서 실행 중인지(기본값 `http://localhost:8000`)
</Step>
</Steps>

## 문제 해결

### Legacy Desktop이 자동으로 감지되지 않는 경우

마이그레이션 프롬프트가 나타나지 않으면 수동으로 마이그레이션할 수 있습니다:

1. Chooser 뷰에서 **+** 카드를 클릭합니다.
2. **Legacy Desktop**을 선택합니다.
3. Legacy Desktop 설치를 선택하고 안내를 따릅니다.

Chooser에서 기존 설치가 전혀 감지되지 않은 경우 Manage 패널을 통해 직접 추가할 수도 있습니다. 자세한 내용은 [**기존 설치 추적**](/ko/installation/desktop/usage/instance-management#tracking-an-existing-installation)을 참조하세요.

### 마이그레이션 실패

- Manage 패널의 Console 탭에서 **로그를 확인**하여 오류 내용을 확인합니다.
- **Legacy 설치가 손상되지 않았는지 확인**합니다: 마이그레이션은 기존 설치에서 읽어옵니다.
- **다시 시도**합니다: 네트워크 문제는 두 번째 시도에서 해결될 수 있습니다.

### 마이그레이션 후 모델이 누락된 경우

모델은 공유 디렉터리 경로로 추가됩니다(복사되지 않음). 모델이 나타나지 않으면:

1. 마이그레이션된 설치의 **Manage** 패널을 엽니다.
2. **Settings → Model directories**로 이동합니다.
3. 이전 모델 폴더가 나열되어 있는지 확인합니다. 없으면 수동으로 추가합니다.
