## 일반 질문

<Accordion title="Comfy Desktop과 레거시 데스크톱의 차이점은 무엇인가요?">
Comfy Desktop은 **멀티 설치 관리자** 역할을 하는 완전히 새로 작성된 버전입니다. 단일 설치 애플리케이션이었던 레거시 데스크톱과 달리, Comfy Desktop을 사용하면 여러 ComfyUI 버전을 나란히 설치, 관리 및 실행할 수 있습니다. 또한 스냅샷 및 모델 다운로드 관리 기능도 추가되었습니다.

레거시 데스크톱 사용자라면 [마이그레이션 가이드](/ko/installation/desktop/usage/migrate)를 확인하여 기존 설정을 이전하세요.
</Accordion>
<Accordion title="Comfy Desktop은 오픈 소스인가요?">
네! 소스 코드는 [GitHub](https://github.com/Comfy-Org/Comfy-Desktop)에서 확인할 수 있습니다.
</Accordion>
<Accordion title="Comfy Desktop이 Portable/Manual 설치를 대체하나요?">
Comfy Desktop은 여러 설치 옵션 중 하나입니다. 필요에 따라 **Portable (Windows)** 설치, **Manual Install** 또는 **Comfy Cloud**를 계속 사용할 수 있습니다. Desktop은 사용 편의성과 원클릭 업데이트, 스냅샷 관리 같은 기능으로 인해 대부분의 사용자에게 권장됩니다.
</Accordion>

## 설치 및 설정

<Accordion title="시스템 요구 사항은 무엇인가요?">
전체 내용은 [시스템 요구 사항](/ko/installation/system_requirements)을 참조하세요.

- **Windows:** Windows 10 이상, CUDA 지원 NVIDIA GPU
- **macOS:** macOS 13 이상 (Ventura), Apple Silicon (M1 이상)
- **Linux:** Ubuntu 22.04 이상, NVIDIA GPU와 CUDA 권장
- **디스크:** 독립형 설치당 최소 4.85 GB
- **RAM:** 최소 8 GB, 16 GB 권장
</Accordion>
<Accordion title="관리자 권한 없이 Comfy Desktop을 설치할 수 있나요?">
**Windows**에서는 독립형 설치에 관리자 권한이 필요하지 않습니다. **macOS**에서는 초기 시스템 전체 설치에 관리자 권한이 필요합니다. **Linux**에서는 시스템 종속성에 관리자 권한이 필요하지만 앱 자체는 사용자 공간에서 실행됩니다.
</Accordion>
<Accordion title="Comfy Desktop은 어떻게 다운로드하나요?">
공식 [ComfyUI 다운로드 페이지](https://comfy.org/download)에서 최신 버전을 다운로드하세요.
</Accordion>
<Accordion title="레거시 데스크톱과 Comfy Desktop을 모두 설치할 수 있나요?">
네, 공존할 수 있습니다. 하지만 설치 항목들을 새로운 Comfy Desktop으로 마이그레이션하고 결국 레거시 데스크톱을 제거하는 것이 좋습니다. [마이그레이션 가이드](/ko/installation/desktop/usage/migrate)를 참조하세요.
</Accordion>

## 사용법

<Accordion title="새 ComfyUI 설치를 추가하려면 어떻게 하나요?">
Comfy Desktop 앱을 열고 **설치 추가**를 클릭한 다음 유형을 선택하세요.

- **Standalone:** 번들 Python으로 새로 설치
- **Git Clone:** 기존 git 저장소 연결
- **Portable:** 포터블 설치 가져오기 (Windows)
- **Cloud:** Comfy Cloud에 연결
</Accordion>
<Accordion title="여러 ComfyUI 버전을 동시에 실행할 수 있나요?">
네! Comfy Desktop은 이를 위해 설계되었습니다. 각 설치는 자체 Python과 의존성을 가진 격리된 환경에서 실행됩니다. 별도의 창에서 동시에 실행할 수 있습니다.
</Accordion>
<Accordion title="모델, 커스텀 노드, 설정은 어디에 저장되나요?">
각 설치는 독립적입니다.

- **Standalone:** 모든 데이터는 설치 폴더 내에 있습니다.
- **Git Clone:** 데이터는 저장소 내의 ComfyUI `models/`, `custom_nodes/`, `user/` 폴더에 있습니다.
</Accordion>
<Accordion title="스냅샷은 어떻게 작동하나요?">
Comfy Desktop은 모든 업데이트 전에 자동으로 스냅샷을 생성합니다. 스냅샷은 다음을 캡처합니다:
- 커스텀 노드와 버전
- 모델 파일
- 설정 및 구성

앱의 **스냅샷** 섹션에서 모든 스냅샷을 찾아보고 복원할 수 있습니다.
</Accordion>
### Comfy Desktop 중지/종료 방법

실행 중인 ComfyUI 인스턴스를 중지하려면 (앱을 열린 상태로 유지하면서 Python 백엔드 중지):

1. 인스턴스 카드에서 **⋮ (kebab) 메뉴** 클릭
2. **중지** 선택
3. 대화상자에서 **확인**

이렇게 하면 ComfyUI Python 프로세스가 중지되지만 데스크톱 앱 창은 열린 상태로 유지되므로 언제든지 다시 시작할 수 있습니다. 데스크톱 앱을 완전히 종료하려면 시스템 트레이 아이콘을 마우스 오른쪽 버튼으로 클릭하고 **종료**를 선택하세요. 창을 닫으면 마지막 창 닫기 설정에 따라 Dashboard로 돌아갈 수 있으므로, 시스템 트레이 방법을 사용해야 완전히 종료됩니다.

## 문제 해결

<Accordion title="설치 후 앱이 시작되지 않습니다">
1. **Windows:** 관리자 권한으로 한 번 실행해 보세요.
2. **macOS:** 보안 경고가 표시되면 **시스템 설정 → 개인정보 보호 및 보안**으로 이동하여 **그래도 열기**를 클릭하세요.
3. **Linux:** `fuse2`가 설치되어 있는지 확인하세요 (`sudo apt install fuse2`)
4. 최신 버전은 [다운로드 페이지](https://comfy.org/download)에서 확인하세요.
</Accordion>
<Accordion title="GPU가 감지되지 않습니다">
- NVIDIA 드라이버가 최신인지 확인하세요.
- Linux에서는 CUDA 툴킷이 설치되어 있는지 확인하세요.
- 설치 유형이 GPU를 지원하는지 확인하세요 (Standalone 및 Git Clone은 지원; 클라우드 연결은 원격 GPU 사용).
</Accordion>
<Accordion title="설치가 업데이트에 실패합니다">
- 인터넷 연결을 확인하세요.
- 앱 메뉴를 통해 먼저 수동 스냅샷을 생성해 보세요.
- 문제가 계속되면 이전 스냅샷으로 복원하고 다시 시도하세요.
</Accordion>
<Accordion title="Comfy Desktop을 어떻게 제거하나요?">
Comfy Desktop과 설치는 디스크에서 분리되어 있습니다. 완전히 제거하려면:
1. 앱 내에서 **설치 삭제** (오른쪽 클릭 → 제거)하거나 설치 폴더를 수동으로 삭제하세요.
2. 시스템의 표준 방법을 사용해 **앱 제거** (Windows의 프로그램 추가/제거, macOS의 휴지통으로 드래그, Linux의 패키지 관리자).
</Accordion>
<Accordion title="Comfy Desktop을 헤드리스 또는 서버 모드로 실행할 수 있나요?">
아니요. Comfy Desktop은 로컬 GUI 애플리케이션으로 설계되었으며, 헤드리스 모드, 서버 전용 모드 또는 원격 조작을 지원하지 않습니다. ComfyUI 인스턴스를 시작하고 관리하려면 그래픽 데스크톱 환경이 필요합니다.

헤드리스 또는 서버 배포가 필요하다면 **수동 설치** 또는 **Windows Portable 빌드**를 사용하세요.
</Accordion>
