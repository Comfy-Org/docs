최근 버전의 새로운 기능: 한 번에 라우팅, 인증 및 에이전트 기술을 처리하는 단일 대화형 마법사입니다.

```bash
comfy setup
```

라우팅 대상(로컬 또는 클라우드) 선택, **브라우저를 통한 로그인(OAuth)**, 프로젝트 디렉터리 선택, 그리고 선택적으로 에이전트 기술 설치 과정을 안내합니다. 이것이 **권장**되는 경로입니다. 복사할 키 없이 브라우저 로그인 창을 열어줍니다.

```bash
comfy setup --where cloud
```

<Note>
  **비대화형(CI 전용).** 브라우저 OAuth는 대화형 세션이 필요합니다. CI, devcontainer 및 브라우저를 사용할 수 없는 스크립트 설치의 경우 API 키를 대신 전달하세요:

  ```bash
  comfy setup --where cloud --api-key comfyui-... --non-interactive
  ```
</Note>

| 플래그 | 목적 |
| --- | --- |
| `--where local\|cloud` | 라우팅 대상; 프롬프트 생략 |
| `--project-dir` | 워크플로, 입력 및 출력 디렉터리 |
| `--api-key` | *(선택 사항)* 헤드리스/CI용 Comfy Cloud API 키이며, `--where cloud`를 암시합니다. |
| `-y, --non-interactive` | 프롬프트 없음. 모든 것을 플래그로 처리합니다. |
| `--skip-skills` | 에이전트 기술을 설치하지 않음 |
| `--skip-verify` | 연결 확인 건너뛰기 |
