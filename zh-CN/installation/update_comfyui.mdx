## 如何更新 ComfyUI?

<Tabs>
  <Tab title="ComfyUI 桌面版">
    ComfyUI 桌面版采用自动更新机制，确保您始终使用最新版本。

    ### 自动更新设置
    请确保在设置中已经启用自动更新功能：

    ![ComfyUI 桌面版设置](/images/desktop/comfyui-desktop-update-setting.jpg)

    ### 手动检查更新
    您也可以手动检查是否有可用的更新：
    1. 点击菜单栏中的 `菜单`
    2. 选择 `帮助`
    3. 点击 `检查更新`

    ![ComfyUI 桌面版检查更新](/images/desktop/desktop_check_for_updates.jpg)

    <Note>桌面版会自动处理所有更新过程，包括 ComfyUI 核心代码和相关依赖的更新</Note>
  </Tab>
  <Tab title="ComfyUI 便携版">
    ComfyUI 便携版提供了便捷的批处理脚本来完成更新操作。 

    ### 更新脚本位置
    在便携版安装目录下的 `update` 文件夹中，您可以找到以下更新脚本：

    ```
    ComfyUI_windows_portable
    └─ 📂update
       ├── update.py
       ├── update_comfyui.bat                           // 更新到最新开发版本
       ├── update_comfyui_stable.bat                    // 更新到最新稳定版本
       └── update_comfyui_and_python_dependencies.bat   // 更新依赖（问题修复时使用）
    ```

    ### 更新步骤
    <Steps>
      <Step title="选择更新脚本">
        根据您的需要选择合适的更新脚本：
        - **推荐使用** `update_comfyui_stable.bat` 更新到最新稳定版本
        - 如果想体验最新功能，可使用 `update_comfyui.bat` 更新到开发版本
        - 仅在遇到运行问题时使用 `update_comfyui_and_python_dependencies.bat`
      </Step>
      <Step title="执行更新">
        双击选择的批处理文件，等待更新完成
      </Step>
      <Step title="重启 ComfyUI">
        更新完成后，重新启动 ComfyUI 即可使用新版本
      </Step>
    </Steps>

    <Warning>更新过程中请保持网络连接稳定，如果在中国大陆地区可能需要配置网络代理</Warning>
  </Tab>
  <Tab title="ComfyUI 手动安装">
    手动安装的 ComfyUI 需要通过 Git 命令来完成更新操作。

    ### 更新前准备
    确保您的系统已安装 [Git](https://git-scm.com/downloads) 并且 ComfyUI 是通过 Git 克隆安装的。

    ### 更新步骤
    <Steps>
      <Step title="激活虚拟环境">
        首先激活 ComfyUI 的 Python 虚拟环境（如果使用了虚拟环境）：
        ```bash
        # 对于 conda 环境
        conda activate comfyui
        
        # 对于 venv 环境
        # Windows
        venv\Scripts\activate
        # macOS/Linux  
        source venv/bin/activate
        ```
      </Step>
      <Step title="拉取最新代码">
        进入 ComfyUI 安装目录并拉取最新代码：
        ```bash
        cd <ComfyUI安装路径>
        git pull
        ```
      </Step>
      <Step title="更新依赖">
        安装或更新 ComfyUI 的依赖包：
        ```bash
        pip install -r requirements.txt
        ```
        <Warning>
        请确保当前处于 ComfyUI 的虚拟环境中，避免污染系统级 Python 环境
        </Warning>
      </Step>
      <Step title="重启 ComfyUI">
        更新完成后重新启动 ComfyUI：
        ```bash
        python main.py
        ```
      </Step>
    </Steps>

    ### 切换版本（可选）
    如果需要切换到特定版本，可以使用以下命令：
    ```bash
    # 查看提交历史
    git log --oneline
    
    # 切换到特定提交
    git checkout <commit-hash>
    
    # 返回最新版本
    git checkout master
    ```

    <Tip>建议定期更新以获得最新功能和安全修复，同时推荐使用稳定版本以确保系统稳定性</Tip>
  </Tab>
</Tabs>

## 在更新 ComfyUI 时都需要更新什么内容？

目前 ComfyUI 的更新主要需要确保两部分内容：
1. 更新 ComfyUI 的代码
2. 更新 ComfyUI 的核心依赖

另外很有必要说明的一点是，开发版本(nightly) 和 稳定版本(release) 的区别：
- **开发版本(nightly)**：最新 commit 的代码，你可以体验到我们最新提供的一些功能，但是也有可能存在一些潜在的问题
- **稳定版本(release)**：是基于稳定版本构建，通常会滞后于开发版本，但是稳定性更高，我们会在相关功能发布稳定后对稳定版本进行支持

目前较多用户总是在更新过程中处于 release 版本，

## ComfyUI 的核心依赖更新

目前除了一些必要的 Python 依赖之外， ComfyUI 的一些核心功能也是单独拆分成 python 包的形式,我们会随着功能发布同步更新这些依赖项目。

### ComfyUI 核心依赖包

```
comfyui-frontend-package   # ComfyUI 前端功能
comfyui-workflow-templates # ComfyUI 工作流模板  
comfyui-embedded-docs      # ComfyUI 节点的帮助文档
```

目前这三个依赖项目分别在不同的仓库中维护：
- [ComfyUI_frontend](https://github.com/Comfy-Org/ComfyUI_frontend/) - 前端界面和交互功能
- [workflow_templates](https://github.com/Comfy-Org/workflow_templates) - 预置工作流模板
- [comfyui-embedded-docs](https://github.com/Comfy-Org/embedded-docs) - 节点帮助文档

### 常见的依赖更新问题

经常有用户只是使用 `git pull` 命令来更新 ComfyUI 的代码，但**忽略了依赖更新**，导致出现以下问题：

<Warning>
- 前端功能缺失或显示异常
- 工作流模板无法加载
- 节点帮助文档过时或缺失
- 新功能无法正常使用
</Warning>

### 为什么要按照 requirements.txt 指定版本安装？

<Warning>
**重要原则：**ComfyUI 的核心依赖版本是经过严格测试和验证的。请始终按照 `requirements.txt` 中指定的版本安装，而不是升级到最新版本。
</Warning>

**原因说明：**
1. **兼容性保证**：指定版本与 ComfyUI 代码完全兼容
2. **稳定性验证**：这些版本组合经过了测试验证
3. **功能匹配**：确保前端功能与后端代码版本匹配
4. **避免冲突**：防止依赖包之间的版本冲突

### 如何正确更新核心依赖

<Tabs>
  <Tab title="便携版用户">
    **推荐方法：** 使用 `update_comfyui.bat`
    这个脚本会同时更新 ComfyUI 代码和所有 Python 依赖包。
    
    **手动更新依赖：**
    ```bash
    # 在便携版目录下打开命令行
    .\python_embeded\python.exe -m pip install -r ComfyUI\requirements.txt
    ```
  </Tab>
  <Tab title="手动安装用户">
    **标准更新流程：**
    ```bash
    # 1. 激活虚拟环境
    conda activate comfyui  # 或其他环境名
    
    # 2. 更新代码
    cd <ComfyUI安装路径>
    git pull
    
    # 3. 更新依赖（重要步骤）
    pip install -r requirements.txt
    ```
    
    **强制重新安装核心依赖：**
    ```bash
    # 强制重新安装所有依赖（按照 requirements.txt 指定版本）
    pip install -r requirements.txt --force-reinstall
    ```
  </Tab>
  <Tab title="桌面版用户">
    桌面版通常会自动处理依赖更新。如果遇到问题：
    1. **检查自动更新设置**是否启用
    2. **手动触发更新**：菜单 → 帮助 → 检查更新
    3. **重新安装桌面版**（极端情况下）
  </Tab>
</Tabs>

### 依赖更新故障排除

如果依赖更新失败，请按以下步骤排查：

<Steps>
  <Step title="检查网络连接">
    确保能够访问 PyPI 或配置国内镜像：
    ```bash
    # 使用清华大学镜像
    pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
    ```
  </Step>
  <Step title="清理缓存">
    清除 pip 缓存避免冲突：
    ```bash
    # 清理 pip 缓存
    pip cache purge
    
    # 清理 Python 缓存
    删除 __pycache__ 文件夹
    ```
  </Step>
  <Step title="检查权限问题">
    Windows 用户可能需要管理员权限：
    ```bash
    # 使用 --user 参数避免权限问题
    pip install -r requirements.txt
    ```
  </Step>
  <Step title="逐个安装核心包">
    如果批量安装失败，可尝试逐个安装。**首先查看 requirements.txt 中的版本要求：**
    
    ```bash
    # 查看 requirements.txt 文件内容
    cat requirements.txt | grep comfyui
    
    # 示例输出：
    # comfyui-frontend-package==1.17.11
    # comfyui-workflow-templates==1.0.0
    # comfyui-embedded-docs==1.0.0
    ```
    
    **然后按照指定版本安装：**
    ```bash
    pip install comfyui-frontend-package==1.17.11
    pip install comfyui-workflow-templates==1.0.0
    pip install comfyui-embedded-docs==1.0.0
    ```
    
    <Warning>
    请使用 requirements.txt 中指定的确切版本号，不要自行升级到最新版本，这可能导致兼容性问题。
    </Warning>
  </Step>
</Steps>

### 验证依赖安装是否成功

更新完成后，**必须验证版本是否与 requirements.txt 要求一致**：

<Steps>
  <Step title="查看已安装版本">
    ```bash
    # 查看已安装的核心包版本
    pip list | grep comfyui
    
    # 输出示例：
    # comfyui-frontend-package    1.17.11
    # comfyui-workflow-templates  1.0.0
    # comfyui-embedded-docs       1.0.0
    ```
  </Step>
  <Step title="对比 requirements.txt 要求">
    ```bash
    # 查看 requirements.txt 中的版本要求
    cat requirements.txt | grep comfyui
    
    # 应该输出相同的版本号：
    # comfyui-frontend-package==1.17.11
    # comfyui-workflow-templates==1.0.0
    # comfyui-embedded-docs==1.0.0
    ```
  </Step>
  <Step title="版本匹配检查">
    <Warning>
    **确保版本完全匹配！**如果版本不一致，请重新安装指定版本：
    ```bash
    pip install comfyui-frontend-package==1.17.11 --force-reinstall
    ```
    </Warning>
  </Step>
</Steps>

您也可以直接查看在线的 [ComfyUI/requirements.txt](https://github.com/comfyanonymous/ComfyUI/blob/master/requirements.txt) 来确认最新的版本要求。

<Tip>
**最佳实践：**
1. 每次更新 ComfyUI 代码后都要执行 `pip install -r requirements.txt`
2. 严格按照 requirements.txt 中指定的版本安装依赖
3. 验证安装的版本与要求版本完全匹配
4. 避免手动升级依赖包到最新版本
</Tip>

## 常见问题

### 为什么我更新后找不到新功能？

这是最常见的问题之一。主要原因是**开发版本(nightly)和稳定版本(release)的区别**：

<Tabs>
  <Tab title="开发版本 (nightly)">
    - **特点**：包含最新的 commit 代码
    - **优势**：可以第一时间体验到最新功能和改进
    - **风险**：可能存在未发现的 bug 或不稳定因素
    - **适合人群**：开发者、测试用户、想要体验最新功能的用户
  </Tab>
  <Tab title="稳定版本 (release)">
    - **特点**：经过测试和验证的稳定代码
    - **优势**：稳定性高，适合生产环境使用
    - **劣势**：功能更新会有延迟，可能落后开发版本数周或数月
    - **适合人群**：需要稳定性的用户、生产环境用户
  </Tab>
</Tabs>

### 如何选择合适的版本？

<Warning>
如果您总是使用稳定版本更新，那么您可能需要等待较长时间才能体验到我们最新发布的功能。
</Warning>

**推荐策略：**

1. **追求稳定性**：使用稳定版本，接受功能更新的延迟
2. **追求新功能**：使用开发版本，但要做好遇到问题的准备
3. **混合使用**：平时使用稳定版本，想体验新功能时临时切换到开发版本

### 如何切换到开发版本？

<Tabs>
  <Tab title="便携版用户">
    使用 `update_comfyui.bat` 而不是 `update_comfyui_stable.bat`：
    ```
    # 开发版本（最新功能）
    double-click: update_comfyui.bat
    
    # 稳定版本
    double-click: update_comfyui_stable.bat
    ```
  </Tab>
  <Tab title="手动安装用户">
    ```bash
    # 切换到开发版本
    git checkout master
    git pull
    
    # 切换到最新稳定版本
    git fetch --tags
    git checkout $(git describe --tags `git rev-list --tags --max-count=1`)
    ```
  </Tab>
  <Tab title="桌面版用户">
    桌面版通常基于稳定版本构建。如果您需要最新功能，建议：
    1. 等待桌面版更新
    2. 或者使用便携版/手动安装来体验最新功能
  </Tab>
</Tabs>

### 更新后出现错误怎么办？

1. **检查依赖**：运行 `pip install -r requirements.txt` 确保所有依赖都已更新
2. **清理缓存**：删除 `__pycache__` 文件夹和 `.pyc` 文件
3. **检查自定义节点**：某些自定义节点可能与新版本不兼容
4. **回退版本**：如果问题严重，可以回退到之前的稳定版本
