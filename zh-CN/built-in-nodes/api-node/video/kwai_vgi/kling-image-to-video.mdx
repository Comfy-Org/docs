Kling Image to Video 节点允许你使用Kling的图像到视频转换技术，将静态图像转变为动态、流畅的视频内容。

## 节点功能

此节点连接到Kling的图像到视频API，让用户能够基于输入图像创建动态视频。它保留原始图像的视觉特征，同时根据文本提示词添加自然的动态效果。该节点支持各种参数控制，包括提示词引导、视频持续时间和宽高比设置。

## 参数说明

### 基本参数

| 参数 | 类型 | 默认值 | 说明 |
|-----|-----|-------|------|
| start_frame | 图像 | - | 要转换为视频的输入图像 |
| prompt | 字符串 | "" | 描述视频动作和内容的文本提示词 |
| negative_prompt | 字符串 | "" | 指定不希望在视频中出现的元素 |
| cfg_scale | 浮点数 | 7.0 | 配置缩放值，控制对提示词的遵循程度 |
| model_name | 选择项 | "ModelName.image2video" | 使用的图像到视频转换模型 |
| aspect_ratio | 选择项 | "AspectRatio.ar_16_9" | 输出视频的宽高比 |
| duration | 选择项 | "Duration.seconds_4" | 生成视频的持续时间 |
| mode | 选择项 | "Mode.cinematic" | 视频生成模式 |

### 输出

| 输出 | 类型 | 说明 |
|-----|-----|------|
| VIDEO | 视频 | 生成的视频结果 |

## 工作原理

Kling Image to Video 节点首先分析输入图像的内容和结构，然后结合用户提供的文本提示词，为图像添加动态效果。节点将请求（包含图像数据和参数）发送到Kling的API服务器，处理完成后返回生成的视频结果。

用户可以通过调整提示词、负面提示词、配置缩放值等参数来精确控制生成结果。不同的模型和模式选项适合不同类型的图像内容和所需的动态效果。
