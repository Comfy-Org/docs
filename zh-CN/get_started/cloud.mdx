Comfy Cloud 提供了一个托管解决方案，让您无需本地安装或 GPU 硬件即可运行 ComfyUI 工作流。本指南将帮助您开始使用 Comfy Cloud。

## 开始使用

### 1. 加入私有测试版

Comfy Cloud 目前处于私有测试阶段。我们正在根据提交逐步授予访问权限。请提交申请以加入我们的等候名单。

### 2. 访问 Comfy Cloud

一旦您获得访问权限，打开您的网络浏览器并访问 [www.comfy.org/cloud](https://www.comfy.org/cloud)，然后使用您的 Comfy 账户登录。您可以立即开始使用。

### 3. 使用模板

使用 Comfy Cloud 与使用本地 ComfyUI 完全相同。如果您是 ComfyUI 的新手，这里有一些快速提示：

#### a. 点击左侧边栏的模板图标

#### b. 点击您想要运行的任何模板

为云用户特别设计的五个重点工作流包括：

- **文本生成图像**：从文本提示生成图像
- **图像编辑**：通过文本提示编辑图像，例如"改变头发颜色"
- **图像生成视频**：将您的任何图像转换为 5 秒视频
- **图像生成 3D**：将您的任何图像转换为 3D 模型
- **文本生成歌曲**：基于您的文本提示生成 AI 歌曲

#### c. 提供您的输入

由于模板中的所有模型都已准备就绪，您只需提供图像输入或提示。检查 `Load Image` 节点或 `CLIP Text Encode` 节点以提供您的输入。

#### d. 运行工作流

如果一切加载正确，点击"Run"图标或使用快捷键"Ctrl + Enter"来运行工作流。

### 4. 简化的工作流

如果工作流对您来说仍然太复杂，这里有一些简化的即用型工作流。您可以下载它们并将它们拖入 Comfy Cloud 中立即使用。

## 管理您的工作流

### 检查工作流状态

点击左侧边栏的队列图标以打开面板并查看所有工作流的状态。

### 保存您的输出

目前，如果输出超出队列历史长度，Cloud 不支持访问所有输出。（这将在后续更新中得到支持）。请手动保存输出以避免在测试期间丢失它们。

### 增加队列历史大小

1. 点击徽标打开下拉菜单
2. 点击设置打开设置面板
3. 搜索 `history` 设置
4. 增加 `Queue history size`

## 云端版本 vs 本地版本

主要区别在于您如何访问 ComfyUI：

- **开源 ComfyUI**：在您自己的设备上安装和运行
- **Comfy Cloud**：在线使用，零设置麻烦

两个版本将始终具有相同的功能。我们同样致力于开发两者。唯一的区别是 Comfy Cloud 在我们强大的服务器上运行，因此您的计算机不需要强大的 GPU。

ComfyUI 将始终免费在本地运行，开源 ComfyUI 和云版本将始终保持功能对等。Cloud 是我们创造收入和维持 ComfyUI 长期发展的方式。

## Comfy Cloud 解决的常见挑战

本地 ComfyUI 安装经常面临这些挑战：

- **环境设置问题**：复杂的 Python 环境配置
- **依赖冲突**：版本不匹配和包冲突
- **硬件限制**：没有足够强大的 GPU 限制了您可以做的事情
- **维护开销**：持续的更新和故障排除

使用 Comfy Cloud，您无需担心任何这些问题。它已准备好立即使用，预装了流行的模型和自定义节点，为您提供所需的强大 GPU 和模型，让您专注于创作，而不是解决冲突。

## 当前限制

### 有限的自定义模型和节点

我们目前对模型和自定义节点的支持有限，但我们正在努力在未来支持更多。

## 故障排除

<AccordionGroup>
  <Accordion title="为什么我无法访问 Comfy Cloud？">
    目前，产品仍处于测试阶段。我们将逐步开放权限。只有白名单上的用户才会被邀请。如果获得批准，我们将发送电子邮件通知。
  </Accordion>

  <Accordion title="我可以上传自己的模型吗？">
    目前还不行。这将很快得到支持。
  </Accordion>

  <Accordion title="我可以安装自定义节点吗？">
    目前，用户无法在 Comfy Cloud 上自行安装自定义节点，但我们将逐步扩展对更多自定义节点的支持。
  </Accordion>

  <Accordion title="云测试版本有管理器吗？">
    云测试版本不支持用户安装自定义插件，因此没有管理器相关功能。我们稍后会添加对插件的支持。
  </Accordion>

  <Accordion title="如何检查正在运行的工作流的状态？">
    您可以在队列面板中检查工作流执行状态。由于工作流在平台上并发执行，可能会有一些延迟。
  </Accordion>

  <Accordion title="如何检查执行历史？">
    由于云版本的限制，您只能在队列历史中查看，最多 256 条记录。建议测试用户手动保存内容。
  </Accordion>
</AccordionGroup>
