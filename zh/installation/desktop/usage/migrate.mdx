如果你正在使用旧版 **Desktop Legacy**（单实例），Comfy Desktop 可以自动检测并迁移你的安装。迁移会创建一个新的独立安装，携带现有的自定义节点、工作流、设置和用户数据。

## 迁移原理

Desktop Legacy 是单实例桌面应用。Comfy Desktop 是多实例管理器。迁移时：

1. **创建新的独立安装** — 使用自己的 Python 环境和 ComfyUI 版本
2. **自定义节点被复制** — 到新安装
3. **模型被链接（而非复制）** — 保留在原位置
4. **工作流和设置被带入** — 无缝继续使用

你的 Desktop Legacy 安装不会在迁移中被修改。

## 分步迁移

### 步骤 1 — 安装 Comfy Desktop

从[官方下载页面](https://comfy.org/download)下载并安装 Comfy Desktop。

### 步骤 2 — 启动 Comfy Desktop

首次打开 Comfy Desktop 时，它会自动检测系统中的 Desktop Legacy 安装。你会看到迁移提示。

### 步骤 3 — 选择"立即迁移"

点击 **Migrate Now** 开始迁移：

1. 检测 Desktop Legacy 安装
2. 创建新的独立安装
3. 复制自定义节点、工作流和设置
4. 链接模型目录

迁移完成后，新的独立安装卡片会出现在 Chooser 视图中。

### 步骤 4 — 验证

启动新安装确认一切正常：
- 所有工作流应该可用
- 自定义节点应该已安装
- 模型应该可以访问

## 迁移后

迁移成功后，可以继续使用新的独立安装。Desktop Legacy 安装保留不变，作为备份。

## 迁移内容

| 项目 | 是否迁移 | 方式 |
|------|:-------:|------|
| 工作流 | ✅ | 复制到新安装 |
| 自定义节点 | ✅ | 在新环境中重新安装 |
| 模型 | ✅ | 通过共享目录链接（不复制） |
| 设置 | ✅ | 旧版设置迁移过来 |
| Python 环境 | 🆕 | 全新环境，内置 Python |
| ComfyUI 版本 | 🆕 | 安装最新版本 |

## 故障排查

### Desktop Legacy 未自动检测

如果 Comfy Desktop 未检测到你的 Desktop Legacy 安装，可以手动添加：

1. 在 Chooser 视图中点击 **+** 卡片
2. 选择 **Legacy Desktop** 作为来源类型
3. Comfy Desktop 会搜索现有安装
4. 按提示完成迁移

### 迁移失败

如果迁移过程中出现错误：

- **检查日志** — 查看平台特定日志目录获取详情
- **确认 Desktop Legacy 安装完整** — 迁移需要读取现有安装
- **重试** — 如果是网络超时等临时错误，重启重试可能成功

### 迁移后模型缺失

模型在迁移中是被链接（而非复制）的。如果模型未出现：

1. 打开已迁移安装的 **Manage** 面板
2. 进入 **Settings → Model directories**
3. 确认旧模型目录已列出
4. 如未列出，手动添加
