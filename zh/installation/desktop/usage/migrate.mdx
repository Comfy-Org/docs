<Info>
本页介绍从 **Desktop Legacy**（旧版单实例桌面应用）迁移到 Comfy Desktop。首次使用 Comfy Desktop 的用户可以跳过此页。
</Info>

如果你已经在使用 Desktop Legacy，Comfy Desktop 可以检测到它并引导你完成迁移，在 Chooser 视图中创建一个新的独立安装。

## 迁移内容

Desktop Legacy 是单实例应用。Comfy Desktop 是多实例管理器。迁移时：

- **工作流和设置** — 复制到新的独立安装
- **自定义节点** — 扫描后在新环境中重新安装
- **模型** — 原有模型文件夹被添加为共享目录（文件保留在原位置）
- **输入文件和输出结果** — 复制到新安装的共享目录
- **Python 环境** — 创建全新的 Python 环境（不会沿用 Legacy 自带的 Python）
- **ComfyUI 版本** — 新安装使用最新版 ComfyUI

Desktop Legacy 安装本身**不会被修改**，保留在系统中作为备份。

## 迁移后的变化

迁移完成后，新的独立安装与其他 Comfy Desktop 安装完全一样：

- **启动方式** — 在 Chooser 视图中点击新安装卡片的播放按钮启动 ComfyUI。服务器使用默认端口（通常为 8188）。
- **管理方式** — 通过 Manage 面板进行更新、快照和配置。
- **更新方式** — 独立安装可以直接在 Manage 面板中更新。不再像 Desktop Legacy 那样每次更新都要下载新安装包。
- **多实例支持** — 可以同时拥有多个独立安装，每个使用不同的 ComfyUI 版本和配置。
- **共享存储** — 模型、输入文件和输出结果默认在所有独立安装间共享。

## 原来的链接还能用吗？

旧版桌面的链接（如 `http://localhost:17888`）在新安装中**不再适用**。新安装的默认链接为 `http://localhost:8188`（或在启动参数中配置的其他端口）。

如果你有保存的书签、工作流中引用了旧链接、或者外部集成指向了旧端口，需要更新为新的安装链接。原来的 Desktop Legacy 安装仍然保留，可作为备用。

## 分步迁移

<Steps>
<Step title="安装 Comfy Desktop">
从[官方下载页面](https://comfy.org/download)下载并安装 Comfy Desktop。
</Step>

<Step title="启动 Comfy Desktop">
首次打开 Comfy Desktop，它会自动检测系统中的 Desktop Legacy 安装并提示迁移。
</Step>

<Step title="确认迁移">
点击 **Migrate Now** 开始迁移：

1. 扫描现有安装（自定义节点、模型、工作流）
2. 创建新的独立安装，包含全新的 Python 环境和最新 ComfyUI
3. 复制用户数据（工作流、设置等）
4. 复制输入文件和输出结果
5. 将模型文件夹添加为共享目录
6. 在新环境中安装自定义节点

完成后，新的独立安装卡片会出现在 Chooser 视图中。
</Step>

<Step title="验证">
启动新安装，确认：
- 工作流正常可用
- 自定义节点已安装
- 模型可以访问
- 服务器运行在新的链接上（默认 `http://localhost:8188`）
</Step>
</Steps>

## 故障排查

### 未自动检测到 Desktop Legacy

如果未出现迁移提示，可以手动迁移：

1. 在 Chooser 视图点击 **+** 卡片
2. 选择 **Legacy Desktop**
3. 选择你的 Desktop Legacy 安装，按提示操作

### 迁移失败

- **检查日志**：在 Manage 面板的 Console 标签页查看错误详情
- **确认 Legacy 安装完整** — 迁移需要读取现有安装
- **重试** — 网络问题可能重试后就能解决

### 迁移后模型找不到

模型是添加为共享目录路径（不复制）。如果模型未出现：

1. 打开已迁移安装的 **Manage** 面板
2. 进入 **Settings → Model directories**
3. 检查旧模型文件夹是否已列出。如未列出，手动添加。
