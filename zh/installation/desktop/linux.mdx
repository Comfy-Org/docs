**Comfy Desktop** 是一个多实例管理器，让您从一个启动器运行多个 ComfyUI 实例。本指南介绍如何在 Linux 上安装和卸载 Comfy Desktop。

## 系统要求

- **系统：** 基于 Debian 的发行版（推荐 Ubuntu 22.04+）
- **GPU：** 推荐使用独立 GPU（NVIDIA / AMD）以获得更好性能，但不是必须
- **磁盘空间：** 每个安装至少推荐 15 GB

## 安装

### .deb 包（Debian/Ubuntu）

```bash
sudo apt install ./comfyui-desktop_*.deb
```

### AppImage（其他发行版）

```bash
chmod +x ./comfyui-desktop-*.AppImage
./comfyui-desktop-*.AppImage --no-sandbox
```

安装后从应用菜单启动 Comfy Desktop。首次打开会显示 **欢迎** 页面，在此创建您的第一个安装。

## 卸载

### 卸载 Comfy Desktop

**Debian/Ubuntu：**
```bash
sudo apt remove comfyui-desktop
```

**AppImage：** 直接删除 AppImage 文件即可。

### 删除安装数据

Comfy Desktop 和它的安装在磁盘上是分开存储的。要完全清理：

1. **从应用内删除单个安装**，或手动删除安装文件夹
2. **删除应用数据和配置目录：**

| 数据 | 路径 |
|------|------|
| 配置 | `~/.config/Comfy Desktop` |
| 数据 | `~/.local/share/Comfy Desktop` |
| 缓存 | `~/.cache/Comfy Desktop` |
| 状态 | `~/.local/state/Comfy Desktop` |

## 数据路径

Comfy Desktop 遵循 XDG Base Directory 规范：

| 数据 | Linux 路径 |
|------|-----------|
| 配置 | `~/.config/Comfy Desktop` |
| 数据 | `~/.local/share/Comfy Desktop` |
| 缓存 | `~/.cache/Comfy Desktop` |
| 状态 | `~/.local/state/Comfy Desktop` |
| 安装目录 | 用户在设置时选择 |

## 故障排除

### AppImage 需要 fuse2

如果 AppImage 无法运行，安装 `fuse2`：

```bash
sudo apt install fuse2
```

### 镜像错误

如果镜像下载失败，展开镜像设置配置自定义的 Python、PyPI 和 PyTorch 镜像源。

### 识别错误

如果安装失败，您会看到以下界面

![Comfy Desktop 安装失败](/images/desktop/win-comfyui-desktop-7.jpg)

建议按照以下步骤查找错误原因：
1. 点击 `显示终端` 查看错误输出
2. 点击 `打开日志` 查看安装日志
3. 访问 [Comfy 官方论坛](https://forum.comfy.org/) 搜索类似错误报告
4. 点击 `重新安装` 重试

在提交反馈之前，建议将 **错误输出** 和 **日志文件** 提供给 AI 工具（如 GPT）进行分析

![Comfy Desktop 安装失败 - 错误日志](/images/desktop/win-comfyui-desktop-8.jpg)
![Comfy Desktop 安装失败 - GPT 反馈](/images/desktop/win-comfyui-desktop-9.jpg)

如上图所示，让 GPT 分析对应错误的原因，或完全卸载 Comfy Desktop 后重新安装。

### 反馈安装失败

如果在安装过程中遇到任何错误，请检查是否有类似的错误报告，或通过以下方式向我们提交错误：

- [GitHub Issues](https://github.com/Comfy-Org/desktop/issues)
- [Comfy 官方论坛](https://forum.comfy.org/)

提交错误报告时，请确保包含以下日志和配置文件，以帮助我们定位和排查问题：

1. **日志文件**

| 文件名 | 描述 | 位置 |
|--------|------|------|
| main.log | 包含 Electron 进程的桌面应用和服务器启动日志 | ~/.config/Comfy Desktop/logs |
| comfyui.log | 包含 ComfyUI 正常运行的相关日志，如核心 ComfyUI 进程终端输出 | ~/.config/Comfy Desktop/logs |

![Comfy Desktop 日志文件位置](/images/desktop/win-comfyui-desktop-10-logs.jpg)

2. **配置文件**

| 文件名 | 描述 | 位置 |
|--------|------|------|
| extra_model_paths.yaml | 包含 ComfyUI 搜索模型和自定义节点的额外路径 | ~/.config/Comfy Desktop |
| config.json | 包含应用配置。此文件不应直接编辑 | ~/.config/Comfy Desktop |

![Comfy Desktop 配置文件位置](/images/desktop/win-comfyui-desktop-11-config.jpg)
