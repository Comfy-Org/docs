## 常见问题

### Comfy Desktop 和旧版 Desktop (Legacy) 有什么区别？

Comfy Desktop 是全新重写的版本，是一个**多实例管理器**。与旧版 Desktop（单一安装）不同，Comfy Desktop 允许你同时安装、管理和运行多个 ComfyUI 版本。还增加了快照和模型下载管理等功能。

如果你在使用旧版 Desktop Legacy，请查看[迁移指南](/zh/installation/desktop/migrate-from-legacy)。

### Comfy Desktop 是开源的吗？

是的！源代码在 [GitHub](https://github.com/Comfy-Org/Comfy-Desktop) 上可用。

### Comfy Desktop 会取代便携版/手动安装吗？

Comfy Desktop 是多种安装方式之一。你仍然可以根据需要选择**便携版（Windows）**、**手动安装**或 **Comfy Cloud**。Desktop 因易用性和一键更新、快照管理等特色功能而被推荐给大多数用户。

## 安装与设置

### 系统要求是什么？

- **Windows：** Windows 10+，支持 CUDA 的 NVIDIA GPU
- **macOS：** macOS 13+ (Ventura)，Apple Silicon (M1+)
- **Linux：** Ubuntu 22.04+，推荐支持 CUDA 的 NVIDIA GPU
- **磁盘：** 每个独立安装至少 4.85 GB
- **内存：** 最低 8 GB，推荐 16 GB

### 没有管理员权限可以安装 Comfy Desktop 吗？

**Windows** 上独立安装不需要管理员权限。**macOS** 上初次安装需要管理员权限。**Linux** 上系统依赖需要管理员权限，但应用本身运行在用户空间。

### 如何下载 Comfy Desktop？

从官方 [ComfyUI 下载页面](https://comfy.org/download) 下载最新版本。

### 可以同时安装旧版 Desktop Legacy 和 Comfy Desktop 吗？

可以共存。但我们建议将实例迁移到新的 Comfy Desktop，然后逐步卸载旧版。请参考[迁移指南](/zh/installation/desktop/migrate-from-legacy)。

## 使用

### 如何添加新的 ComfyUI 安装？

打开 Comfy Desktop，点击 **添加安装**，选择类型：

- **Standalone** — 全新安装，自带 Python
- **Git Clone** — 关联现有 Git 仓库
- **Portable** — 导入便携版安装（Windows）
- **Cloud** — 连接 Comfy Cloud

### 能同时运行多个 ComfyUI 版本吗？

可以！Comfy Desktop 就是为此设计的。每个安装运行在独立的隔离环境中，拥有自己的 Python 和依赖。它们可以同时在独立窗口中启动。

### 我的模型、自定义节点和设置存储在哪里？

每个安装是自包含的：

- **Standalone：** 所有数据在安装文件夹内
- **Git Clone：** 数据在仓库的 `models/`、`custom_nodes/` 和 `user/` 文件夹中

### 快照如何工作？

Comfy Desktop 在每次更新前自动创建快照。快照包含：
- 自定义节点及其版本
- 模型文件
- 设置和配置

你可以从应用的 **快照** 部分浏览和恢复任意快照。

## 故障排除

### 安装后应用无法启动

1. **Windows：** 尝试以管理员身份运行一次
2. **macOS：** 如果看到安全警告，前往 **系统设置 → 隐私与安全性**，点击 **仍要打开**
3. **Linux：** 确保已安装 `fuse2`（`sudo apt install fuse2`）
4. 查看[下载页面](https://comfy.org/download)获取最新版本

### GPU 检测不到

- 确保 NVIDIA 驱动是最新的
- Linux 上检查 CUDA toolkit 是否安装
- 确认安装类型支持 GPU（Standalone 和 Git Clone 支持；Cloud 使用远程 GPU）

### 安装更新失败

- 检查网络连接
- 先通过应用菜单手动创建快照
- 如果问题持续，恢复到之前的快照再试

### 如何卸载 Comfy Desktop？

Comfy Desktop 和它的安装在磁盘上是分开的。完整卸载：

1. **从应用内删除安装**（右键 → 移除）或手动删除安装文件夹
2. **卸载应用** — 使用系统标准方式（Windows 的添加/删除程序，macOS 拖到废纸篓，Linux 包管理器）
