如果你正在使用旧版 **Desktop Legacy**（单实例版本），Comfy Desktop 可以自动检测并迁移你的安装。迁移会创建一个新的独立安装，包含你现有的自定义节点、工作流、设置和用户数据。

## 迁移原理

旧版 Desktop 是一个单实例桌面应用。Comfy Desktop 是一个多实例管理器。迁移时：

1. **创建一个新的独立安装** — 带有自己的 Python 环境和 ComfyUI 版本
2. **复制你的自定义节点** — 到新安装
3. **链接你的模型**（不复制）— 它们保留在原位
4. **继承工作流和设置** — 让你从中断的地方继续

你的旧版 Desktop 安装不会被修改。

## 迁移步骤

### 第一步 — 安装 Comfy Desktop

从[官方下载页面](https://dl.todesktop.com/241130tqe9q3y)下载并安装 Comfy Desktop。

### 第二步 — 启动 Comfy Desktop

首次打开 Comfy Desktop 时，它会自动检测系统上的任何旧版 Desktop 安装。你会看到迁移提示。

### 第三步 — 点击"立即迁移"

点击 **Migrate Now** 开始迁移过程。Comfy Desktop 将：

1. 检测你的旧版 Desktop 安装
2. 创建一个新的独立安装
3. 复制自定义节点、工作流和设置
4. 链接你的模型目录

迁移完成后，新的独立安装卡片将出现在 Chooser 视图中。

### 第四步 — 验证

启动新安装确认一切正常：
- 所有工作流应可用
- 自定义节点应已安装
- 模型应可访问

## 迁移后

迁移成功后，你可以继续使用新的独立安装。你的旧版 Desktop 安装保持不变，作为备份。

## 迁移内容

| 项目 | 是否迁移？ | 方式 |
|------|:--------:|------|
| 工作流 | ✅ | 复制到新安装 |
| 自定义节点 | ✅ | 在新环境中重新安装 |
| 模型 | ✅ | 通过共享目录链接（不复制） |
| 设置 | ✅ | 旧版设置被继承 |
| Python 环境 | 🆕 | 全新的内置 Python |
| ComfyUI 版本 | 🆕 | 安装最新版本 |

## 为什么迁移？

Comfy Desktop 相比旧版 Desktop 的优势：

- **多实例管理** — 同时运行不同版本
- **快照** — 每次更新前自动备份
- **一键更新** — 点击一次获取最新 ComfyUI
- **云集成** — 原生支持 Comfy Cloud
- **Git 克隆支持** — 适合需要跟踪分支的开发者
- **远程连接** — 连接到其他机器上的 ComfyUI 实例
