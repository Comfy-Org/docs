Seedance 2.0 首尾帧生成视频节点（`ByteDance2FirstLastFrameNode`）可以在起始图像和结束图像之间生成完整的视频。你提供首帧和末帧，Seedance 2.0 创建连接它们的运动与过渡。

<ReqHint/>
<UpdateReminder/>

## 工作流概览

首尾帧工作流使用四个节点：

1. **LoadImage（×2）** — 加载首帧和末帧图片
2. **ByteDance2FirstLastFrameNode** — 接收两张图片加文本提示，输出 VIDEO
3. **SaveVideo** — 将生成的视频保存到磁盘

## 节点参数

| 参数 | 说明 |
| :--- | :--- |
| `model` | 模型名称。默认：`Seedance 2.0` |
| `prompt` | 视频的文字描述。用于描述过渡、镜头运动和场景 |
| `resolution` | 输出分辨率。可选：`480p`、`720p`、`1080p` |
| `aspect_ratio` | 画面比例。可选：`16:9`、`9:16`、`1:1`、`4:5` |
| `duration` | 视频时长（秒） |
| `generate_audio` | 是否生成与视频同步的音频 |
| `seed` | 随机种子 |
| `cfg` | 无分类器引导尺度 |

## 输入

| 输入 | 类型 | 说明 |
| :--- | :--- | :--- |
| `first_frame` | IMAGE | 起始帧图片（直接图片输入） |
| `last_frame` | IMAGE | 结束帧图片（直接图片输入） |
| `first_frame_asset_id` | STRING | 已验证真人首帧的 Asset ID |
| `last_frame_asset_id` | STRING | 已验证真人末帧的 Asset ID |

## 运行工作流

1. **加载图片** — 使用 LoadImage 节点加载首帧和末帧图片
2. **编写提示词** — 描述过渡和整体场景。提示词有助于引导运动、镜头和风格
3. **设置参数** — 选择分辨率、画面比例、时长及音频设置
4. **运行工作流** — 点击"加入队列"或使用 `Ctrl(Cmd) + Enter`
5. **查看结果** — 生成的视频出现在 SaveVideo 节点中

## 提示

- **选择差异明显的帧** — 首末帧之间的反差越大，生成的过渡越精彩
- **提示词作为引导** — 提示词描述的是过渡效果，而不仅仅是场景。请包含镜头运动方向（例如"无人机向前飞，然后旋转180度"）
- **真人 Asset ID** — 使用已验证的真人肖像时，连接 `asset_id` 字符串而非直接图片输入
