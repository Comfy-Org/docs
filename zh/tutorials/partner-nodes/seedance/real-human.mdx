**Seedance 2.0 真人模式** 现已直接在 ComfyUI 中可用。你可以通过 ComfyUI 工作流上传肖像，完成一次性验证，然后生成带有该真人一致身份和自然表情的视频 — 无需离开 ComfyUI。

有了 ComfyUI 中的真人支持，你不再局限于虚拟角色。Seedance 2.0 的可控性和原生音画同步现在扩展到真人，身份和表演在整段视频中保持一致。

## 有何不同

- **真人，电影级控制** — 控制镜头运动、灯光和场景变化，同时保持主体真实性
- **身份一致性** — 面部特征和整体外观在运动、镜头移动和转场中保持稳定
- **原生音画同步** — 视频和音频（对话、环境音、音乐）一起生成，唇形同步和表情更可靠
- **多参考引导** — 混合文本与最多 **9 张图片**、**3 个视频**和 **3 个音频片段**，锁定外观、运动和节奏

## ComfyUI 中的验证流程

在 Seedance 2.0 生成中出现真人之前需要完成验证。这是为了防止身份冒用和未经授权使用他人肖像，由字节跳动处理，符合新兴的 AI 透明度法规。整个流程通过 ComfyUI 中的 **ByteDance Create Image/Video Asset** 节点驱动。

### 首次运行（需验证）

1. **上传肖像** — 在 ComfyUI 中添加 **ByteDance Create Image/Video Asset** 节点并上传肖像图片或视频
2. **运行工作流** — ComfyUI 发送素材进行处理，并返回字节跳动的验证链接
3. **完成活体检测** — 在手机或浏览器中打开链接，完成活体检测（通常少于 1 分钟）
4. **保存 ID** — 验证通过后，节点输出两个 ID：
   - **Group ID** — 代表已验证的本人；保存此 ID 以便后续复用同一身份时无需重新验证
   - **Asset ID** — 绑定当前这张图片/视频；接入 Seedance 2.0 视频生成节点即可生成该真人的视频

### 再次上传（无需验证）

1. 用 **ByteDance Create Image/Video Asset** 节点上传该真人的新照片或视频
2. 在节点中填入已保存的 **Group ID**
3. 运行时字节跳动会比对新人脸与首次验证记录；确认为同一人后，新素材自动激活可用

## 可用工作流

### 参考生成视频真人模式

使用已验证的肖像（加上可选的其他参考图片、视频或音频）驱动带有持续真人身份的 Seedance 2.0 视频。

`ByteDance2ReferenceNode` 上的 `asset_1` 输入接受 Asset ID 字符串。请注意，使用真人 Asset ID 时，直接的 `image` 和 `video` 输入不支持真人 — 请使用 Asset ID 输入代替。

### 首尾帧生成视频真人模式

提供已验证的首帧和末帧，生成之间的视频，同时保持真人的身份。

`ByteDance2FirstLastFrameNode` 上的 `first_frame_asset_id` 和 `last_frame_asset_id` 输入接受 Asset ID 字符串。使用 Asset ID 时，不能同时使用对应帧的直接图片输入。

## 重要说明

- **Group ID** 由单个账号验证后仅限该账号使用，不支持跨账号使用
- 确保用于创建资产的图片/视频中只包含**一个角色**
- Asset ID 相当于一个图片输入。对于首帧，如果使用了 Asset ID 输入，就不能同时使用图片输入。末帧同理
- 资产限制：图片最多 9 张，视频最多 3 个，音频最多 3 个
