**ElevenLabsSpeechToText** 节点使用 ElevenLabs Scribe 模型将音频转录为文字。它输出转录文本、检测到的语言代码以及包含时间信息的逐词 JSON 数据。

<ReqHint/>
<UpdateReminder/>

## 支持模型

| 模型 | 描述 |
| :---- | :---------- |
| `scribe_v2` | 高精度语音转文字转录 |

### 语音转文字输出

节点产生三个输出：

| 输出 | 类型 | 描述 |
| :----- | :--- | :---------- |
| **text** | STRING | 完整的转录文本 |
| **language_code** | STRING | 检测到的语言代码 |
| **words_json** | STRING | 逐词转录及其时间信息（JSON） |

### 语音转文字参数

| 参数 | 描述 |
| :-------- | :---------- |
| **model_id** | 转录模型选择（`scribe_v2`） |
| **tag_audio_events** | 标记音频事件，如笑声、掌声 |
| **num_speakers** | 说话人分离的说话人数（0 = 自动） |
| **timestamps_granularity** | 时间戳粒度：none, word, character |
| **diarize** | 启用/禁用说话人分离 |
| **output_format** | 结果文本的输出格式 |

使用模板作为参考时的操作步骤：

1. 通过 **LoadAudio** 加载音频文件（如 `voice_demo.mp3`）
2. （可选）使用 **RecordAudio** 直接录制音频
3. 将音频连接到 **ElevenLabsSpeechToText**
4. 选择模型（`scribe_v2`）
5. 配置可选参数：
   - 启用 **diarize** 进行多说话人转录
   - 将 **timestamps_granularity** 设为"word"获取逐词时间戳
   - 设置 **num_speakers** 指定说话人数（0 为自动检测）
6. 点击 **运行** 或按 `Ctrl(cmd) + Enter`
7. 在 **PreviewAny** 节点中查看转录文本
8. 在 **words_json** PreviewAny 节点中检查逐词 JSON 数据

### 补充说明

- **RecordAudio** 节点默认处于绕过状态，请用 `Ctrl-B` 启用进行实时录制
- **words_json** 输出包含每个单词的开始和结束时间戳，适用于字幕生成或对齐
- **language_code** 会自动检测，可用于下游多语言处理
