<Tip>
对 ComfyUI 进行故障排查时，请手动进行检查。简单的重新安装可能会掩盖根本原因。
</Tip>

## 基本信息

在报告错误之前，请确保包含以下信息：

- **操作系统**：Windows/Mac/Linux
- **安装方式**：桌面应用/便携版/手动安装
- **运行环境**：ComfyUI 版本号
- **显卡型号**：显卡型号与显存容量
- **自定义节点**：使用[/customnode API 路由](https://github.com/Comfy-Org/ComfyUI/blob/master/server.py)导出已安装的节点列表
- **模型**：使用的模型
- **桌面应用问题（如适用）**：桌面应用版本号

在报告错误之前，请运行以下命令确保所有内容都是最新的：

```bash
git pull
comfy node update all
```

## 命令行参数

```bash
# 使用 CPU 运行
comfy launch --cpu

# 使用低显存模式（将文本编码器移到 CPU）
comfy launch --lowvram

# 使用中等显存模式
comfy launch --medvram

# 使用缓存经典模式
comfy launch --cache-classic
```

## 常见问题

<Tabs>
  <TabItem value="launch" label="启动问题" default>
    ComfyUI 无法启动或启动后立即崩溃。

    ```bash
    # 检查 Python 和 PyTorch 是否正确安装
    python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
    ```

    如果 PyTorch 报告 CUDA 不可用：
    1. 更新您的 GPU 驱动
    2. 重新安装与您的 CUDA 版本匹配的 PyTorch
  </TabItem>
  <TabItem value="api" label="网络与 API 问题">
    ### 前端问题

    如果页面显示空白，请打开开发者控制台。如果控制台显示类似 `Loading` 的内容卡住，请尝试以下方法：

    - **硬刷新**：`Ctrl + F5`（Windows）或 `Cmd + Shift + R`（Mac）
    - **清空 IndexedDB**
      - 打开开发者工具 → 应用（Application）→ IndexedDB → 右键 → 清空
    - **切换网络**
      - 使用 `--listen 0.0.0.0` 启动 ComfyUI，将监听所有网络接口
      - 有些网络可能会阻止本地连接

    ### 防火墙

    - 确保您的防火墙允许连接 `127.0.0.1:8188`
    - 如果您使用 `--listen 0.0.0.0`，请确保您的防火墙允许所需的端口
  </TabItem>
  <TabItem value="desc" label="桌面应用问题（Windows）">
    ### 不支持的设备

    如果您看到"不支持的设备"错误，请更新您的显卡驱动。

    ### 安装失败

    如果桌面应用安装失败，请尝试：
    1. 以管理员身份运行安装程序
    2. 关闭杀毒软件
    3. 确保您有足够的磁盘空间

    ### 维护页面

    如果您看到维护页面，桌面应用正在进行自动更新。请等待更新完成。

    ### 缺少模型

    如果缺少模型，您可以通过桌面应用的"模型"标签下载它们。
  </TabItem>
  <TabItem value="hardware" label="特定硬件问题">
    ### Intel GPU

    如果您使用 Intel GPU，请确保：
    - 已安装最新 Intel GPU 驱动
    - 使用正确的 PyTorch 版本：`pip install torch --index-url https://download.pytorch.org/whl/xpu`
  </TabItem>
</Tabs>
