About 页面是 ComfyUI 设置系统中的一个信息展示面板，用于显示应用程序版本信息、相关链接和系统统计数据，这些设置在向我们提交反馈问题时，可以提供给我们一些非常关键的信息。

![about](/images/interface/setting/settings-about.jpg)

### 版本信息徽章

About 页面显示以下核心版本信息：

- **ComfyUI 版本**：显示后端 ComfyUI 的版本号，格式取决于分发方式：
  - **OSS（开源版）**：显示 `ComfyUI <commit hash>`，链接到 [GitHub 主仓库](https://github.com/Comfy-Org/ComfyUI)
  - **Desktop**：显示 `ComfyUI v<版本号>`，链接到同一个 GitHub 仓库
  - **Cloud**：显示 `ComfyUI <commit hash>`，带云图标链接到 [comfy.org](https://www.comfy.org)
- **ComfyUI_frontend 版本**：显示前端界面的版本号，链接到前端 GitHub 仓库
- **Templates 版本**：如果 `systemStats.system.installed_templates_version` 可用，则显示 Templates 徽章：
  - 标签：`Templates v<版本号>`
  - 链接：[https://pypi.org/project/comfyui-workflow-templates/](https://pypi.org/project/comfyui-workflow-templates/)
  - 图标：`pi pi-book`
  - 如果已安装版本与所需版本不一致，徽章将以 **danger** 严重级别（红色）显示
- **Discord 社区**：提供 ComfyOrg Discord 服务器的链接
- **ComfyOrg**：链接到 ComfyOrg 官方网站

徽章的显示顺序为：ComfyUI → Frontend → Templates（如已安装）→ Discord → ComfyOrg。

<Tip>
由于这里的版本信息主要是对应稳定版本的信息，如果你正在使用的是 nightly 版本，那么这里并不会显示对应的 commit hash。如果你正在使用 nightly 版本，可以在对应的 ComfyUI 主目录中使用 `git log` 命令来查看对应的 commit hash 等信息。
另外一个常见的问题是，不同的依赖包可能会在更新过程中失败回滚。
</Tip>

### 自定义节点徽章

如果安装了自定义节点，About 页面还会显示自定义节点提供的额外徽章信息。这些徽章由各个自定义节点通过 `aboutPageBadges` 属性注册。

### 系统统计信息

页面底部显示详细的系统统计信息，根据分发方式略有不同：

**OSS / 通用版本：**
- **OS**：操作系统标识
- **Python 版本**：已安装的 Python 版本
- **嵌入式 Python**：嵌入式 Python 详细信息（如适用）
- **Pytorch 版本**：PyTorch 框架版本
- **启动参数 (argv)**：启动 ComfyUI 时使用的命令行参数
- **总内存 (RAM Total)**：系统总可用内存
- **空闲内存 (RAM Free)**：当前空闲内存
- **Templates 版本**：如果已安装版本过旧，会以 danger 样式显示

**Cloud 版本：**
- **Cloud 版本**：云平台版本
- **ComfyUI 版本**：后端 ComfyUI 版本
- **前端版本**：前端界面版本
- **Templates 版本**：工作流模板版本

**设备信息**（检测到兼容硬件时显示）：
- 设备名称和类型（如 GPU 型号）
- **总 VRAM / 空闲 VRAM**：显存统计
- **Torch 总 VRAM / Torch 空闲 VRAM**：PyTorch 管理的显存统计
- 多设备以 **TabView** 显示，每个设备一个标签页

底部提供 **复制系统信息** 按钮，可将所有系统信息以格式化文本块复制到剪贴板，方便在报告问题时分享。

### 版本更新通知

`ShowVersionUpdates` 设置（位于 Comfy → 通知偏好设置下）控制是否在 About 页面及界面其他位置显示版本更新通知。

## 扩展开发者指南

扩展开发者可以通过在扩展配置中添加 `aboutPageBadges` 属性来向 About 页面添加自定义徽章：

```javascript
app.registerExtension({
  name: 'MyExtension',
  aboutPageBadges: [
    {
      label: 'My Extension v1.0.0',
      url: 'https://github.com/myuser/myextension',
      icon: 'pi pi-github'
    }
  ]
})
```
