此节点将 SeedVR2 视频潜空间变量分割成较小的时序块，以便在可用 VRAM 内逐一处理。它会根据您的 GPU 内存自动计算最佳块大小，或允许您手动指定块大小，并按顺序输出这些块以供处理。

## ## 输入

| 参数 | 描述 | 数据类型 | 是否必填 | 范围 |
|------|------|----------|----------|------|
| `latent` | 要分割的 VAE 编码后的 SeedVR2 潜空间变量。 | LATENT | 是 | - |
| `temporal_overlap` | 相邻块之间共享并在合并时进行交叉淡化的潜空间帧数；0 表示无重叠（默认值：0）。 | INT | 否 | 0 至 16384 |
| `chunking_mode` | "manual" 模式精确使用 `frames_per_chunk` 值；"auto" 模式自动预测适合空闲 VRAM 的最大块。 | COMBO | 是 | "auto"<br />"manual" |

当 `chunking_mode` 设置为 "manual" 时，会出现一个额外参数：

| 参数 | 描述 | 数据类型 | 是否必填 | 范围 |
|------|------|----------|----------|------|
| `frames_per_chunk` | 每个时序块的像素帧数。必须为 4n+1 值（1, 5, 9, 13, 17, 21, ...）（默认值：21）。 | INT | 是 | 1 至 16384 |

注意：`frames_per_chunk` 参数仅在 `chunking_mode` 设置为 "manual" 时出现。该值必须满足公式 `(frames_per_chunk - 1) % 4 == 0`，即必须是以下数值之一：1, 5, 9, 13, 17, 21 等。

## ## 输出

| 输出名 | 描述 | 数据类型 |
|--------|------|----------|
| `latents` | 按顺序排列的时序块。 | LATENT |
| `temporal_overlap` | 相邻块之间的有效潜空间帧重叠数，用于合并 SeedVR2 潜空间变量。 | INT |
