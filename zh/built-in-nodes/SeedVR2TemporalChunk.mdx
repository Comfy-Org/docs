此节点将 SeedVR2 视频潜变量分割成较小的重叠时间块，这些块可在可用 VRAM 内逐一处理。这些块设计为同时传递给 Apply SeedVR2 Conditioning 节点和采样器潜变量输入，随后使用 Merge SeedVR2 Latents 节点重新组合。

## 输入
| 参数 | 描述 | 数据类型 | 是否必需 | 范围 |
|-----------|-------------|-----------|----------|-------|
| `latent` | 要分割的 VAE 编码 SeedVR2 潜变量 | LATENT | 是 | - |
| `temporal_overlap` | 相邻块之间共享并在合并时交叉淡化的潜变量帧数；0 表示无重叠（默认值：0） | INT | 否 | 0 至 16384 |
| `chunking_mode` | manual = 精确使用 frames_per_chunk；auto = 预测适合空闲 VRAM 的最大块 | COMBO | 是 | "auto"<br />"manual" |

当 `chunking_mode` 设置为 "manual" 时，以下参数可用：

| 参数 | 描述 | 数据类型 | 是否必需 | 范围 |
|-----------|-------------|-----------|----------|-------|
| `frames_per_chunk` | 每个时间块的像素帧数（必须为 4n+1：1, 5, 9, 13, ...）（默认值：21） | INT | 是 | 1 至 16384，步长 4 |

**参数约束说明：**
- 输入 `latent` 必须是形状为 (B, C, T, H, W) 且恰好有 4 个潜变量通道的五维视频潜变量
- 使用 "manual" 模式时，`frames_per_chunk` 必须为 4n+1 值（1, 5, 9, 13, 17, 21, ...）
- `temporal_overlap` 自动限制为小于块大小
- 在 "auto" 模式下，节点根据可用空闲 VRAM 计算最佳块大小

## 输出
| 输出名 | 描述 | 数据类型 |
|-------------|-------------|-----------|
| `latents` | 按顺序排列的时间块 | LATENT |
| `temporal_overlap` | 相邻块之间的有效潜变量帧重叠数，用于 Merge SeedVR2 Latents | INT |
