# 概述

LoraSave 节点可从模型差异中提取并保存 LoRA（低秩适应）文件。它可以处理扩散模型差异、文本编码器差异或两者同时处理，将其转换为指定秩和类型的 LoRA 格式。生成的 LoRA 文件将保存到输出目录中供后续使用。

## 输入

| 参数 | 描述 | 数据类型 | 必填 | 取值范围 |
| --- | --- | --- | --- | --- |
| `文件名前缀` | 输出文件名的前缀（默认值："loras/ComfyUI_extracted_lora"） | STRING | 是 | - |
| `排名` | LoRA 的秩值，控制大小和复杂度（默认值：8） | INT | 是 | 1-4096 |
| `lora类型` | 要创建的 LoRA 类型（默认值："standard"） | COMBO | 是 | `"standard"`<br />`"locon"`<br />`"loha"`<br />`"lokr"`<br />`"dylora"` |
| `偏差差异` | 是否在 LoRA 计算中包含偏置差异（默认值：True） | BOOLEAN | 是 | - |
| `模型差异` | 要转换为 LoRA 的 ModelSubtract 输出 | MODEL | 否 | - |
| `文本编码器差异` | 要转换为 LoRA 的 CLIPSubtract 输出 | CLIP | 否 | - |

**注意：** 节点运行时，必须至少提供 `model_diff` 或 `text_encoder_diff` 中的一个。如果两者都省略，节点将不会产生任何输出。

## 输出

| 输出名称 | 描述 | 数据类型 |
| --- | --- | --- |
| - | 此节点将 LoRA 文件保存到输出目录，但不会通过工作流返回任何数据 | - |
