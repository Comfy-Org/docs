## 概述

**Comfy Cloud MCP 服务器**通过[模型上下文协议（MCP）](https://modelcontextprotocol.io)将 AI 代理连接到 [Comfy Cloud](https://cloud.comfy.org)。连接后，你可以通过代理对话生成图像、视频、音频和3D内容，搜索模型、节点和模板，并运行 ComfyUI 工作流。

服务器远程运行在 `cloud.comfy.org/mcp`。工作流在 Comfy Cloud GPU 上执行，因此你无需本地 GPU。你的代理通过 HTTPS 与服务器通信；服务器将工具调用转换为经过身份验证的请求，代表你发送给 Comfy Cloud。

任何 **MCP 兼容客户端**都可以连接到相同的服务器 URL。**Claude Code**、**Claude Desktop**、**Cursor** 和 **Codex** 在下方有优先设置指导。**Windsurf**、**Amp** 和其他代理使用相同的 URL，搭配 OAuth 或 API 密钥。

## 安装 Comfy Cloud MCP

<Info>
  你需要有一个 [Comfy Cloud](https://cloud.comfy.org) 账户才能连接。如果你还没有，请[注册](https://cloud.comfy.org)。设置期间的 OAuth 登录会使用你的 Comfy 账户。
</Info>

Comfy Cloud MCP 运行于：

```
https://cloud.comfy.org/mcp
```

选择你的客户端：

## 您的代理能做什么

您无需自行调用 MCP 工具。您的代理会根据您的请求自动选择合适的工具。斜杠命令和提示（如下所示）是引导代理完成常见使用例的快捷键，但自然语言也同样有效（例如“生成一张猫宇航员的图像”、“放大这张照片”、“找一份 Wan万相 2.2 视频模板”）。

典型流程：

1. **发现**可用资源（使用 `search_templates`、`search_models`、`search_nodes`，或对画面风格的问题使用 `cql`）。
2. **运行**生成任务：如果找到匹配的预构建模板则使用 `run_template`，自定义工作流则使用 `submit_workflow`（需要输入图像时配合 `upload_file`），或使用 `partner_generate` 调用 Flux、Grok、Gemini、OpenAI、Ideogram、Seedance 等合作伙伴模型。
3. **等待并获取输出**（`wait_for_job`，然后 `get_output` 返回一条下载指令，由您的代理在 shell 中运行）。

服务器倾向于优先匹配 [预构建模板](https://comfy.org/workflows)，而不是从头构建工作流，这样往往能更快地获得更好的结果。

## MCP 工具

这些是您的代理连接后可访问的工具。名称与 MCP 客户端日志和调试中显示的名称保持一致。

### 发现

| 工具 | 描述 |
| --- | --- |
| `search_templates` | 按文本、标签、媒体类型或模型从 [comfy.org](https://comfy.org/workflows) 搜索预构建的工作流模板 |
| `get_template` | 获取模板的完整工作流 JSON，用于检查或手动编辑 |
| `get_template_schema` | 查看模板的哪些参数可以在运行时覆盖 |
| `search_models` | 按文本、类型、基础模型或来源搜索模型目录 |
| `search_nodes` | 按文本、类别或输入/输出类型搜索可用节点。还能列出**子图蓝图**（预连线的节点组，如文本转图像或移除背景） |
| `get_node` | 获取特定节点的完整输入说明，包括完整的选项列表 |
| `cql` | 运行 [CQL](https://github.com/Comfy-Org/cql) 图查询以回答结构性问题（例如，哪些节点生成图像，或节点类型之间的路径）。当需要特定节点的完整输入说明时，请使用 `search_nodes` |
| `get_prompting_guide` | 各模型系列的提示词风格与推荐设置（步数、cfg、采样器、分辨率） |

### 生成

| 工具 | 描述 |
| --- | --- |
| `run_template` | 按名称运行预构建模板，可选择覆盖参数——有匹配模板时的首选路径 |
| `submit_workflow` | 提交 ComfyUI API 格式的工作流在 Comfy Cloud 上执行 |
| `partner_generate` | 使用合作伙伴 API 模型生成（Flux/BFL、Grok、Gemini、OpenAI、Ideogram、Seedream/Seedance 等） |
| `upload_file` | 上传输入图像或文件用于工作流中（例如，与 LoadImage 配合使用） |
| `apply_slots` | 将参数覆盖应用到工作流的内部值，并返回修改后的工作流 |

### 作业与批处理

| 工具 | 描述 |
| --- | --- |
| `get_job_status` | 轮询已提交工作流的执行状态 |
| `wait_for_job` | 等待作业完成，而无需反复轮询 |
| `get_output` | 从已完成的工作流中检索输出的图像、视频或音频 |
| `use_previous_output` | 通过将一个运行的输出重用作另一个运行的输入来串联工作流 |
| `cancel_job` | 取消待定或正在运行的作业 |
| `get_queue` | 检查正在运行和待定的作业数量 |
| `submit_batch`、`get_batch_status`、`get_batch_output`、`wait_for_batch` | 在一次调用中提交多个生成任务，稍后统一收集结果——批处理 ID 在多个会话间保持有效 |

### 已保存的工作流

| 工具 | 描述 |
| --- | --- |
| `list_saved_workflows` | 浏览您保存在 Comfy Cloud 中的工作流 |
| `get_saved_workflow` | 检查已保存工作流的节点、输入和可自定义的设置 |
| `save_workflow` | 将工作流保存到您的 Comfy Cloud 账户 |
| `update_workflow` | 就地更新现有工作流并生成新版本（早期版本仍可恢复） |
| `run_saved_workflow` | 按文件名运行已保存的工作流——服务器会自动将其从编辑器格式转换为可执行格式 |

### 共享工作流

| 工具 | 描述 |
| --- | --- |
| `share_workflow` | 发布一个已保存的工作流，并返回一个任何人都可以打开的 `?share=<id>` 链接 |
| `import_shared_workflow` | 将分享链接或纯分享 ID 解析为工作流 JSON，可选择保存到您的账户 |

**Hub 链接分享 ID：** `comfy.org/workflows/<slug>-<hex>` Hub 链接中尾部的连字符分隔的十六进制令牌就是分享 ID。例如，`comfy.org/workflows/topaz-starlight-upscale-1c77e82713b7` 的分享 ID 是 `1c77e82713b7`。将该令牌作为 `share_id` 传给 `import_shared_workflow`。`share_url` 参数只接受类似 `https://cloud.comfy.org/?share=...` 的 `?share=<id>` 查询参数链接，不接受 Hub 页面链接。

### 应用与链接

| 工具 | 描述 |
| --- | --- |
| `create_app` | 将已保存的工作流转换为应用模式（App Mode）应用——带有选定输入和输出、简化的“运行此工作流”视图 |
| `get_app_mode_url` | 获取一个稳定链接，可将工作流作为可运行的应用打开 |
| `get_workflow_canvas_url` | 获取一个直接在 Comfy Cloud 画布上打开工作流的链接，随时可查看、编辑或运行 |

### 账户与会话

| 工具 | 描述 |
| --- | --- |
| `get_billing_status` | 查看你的积分余额、订阅等级和账单链接 |
| `get_server_info` | 确认代理连接到的是哪台服务器：环境、主机、版本和认证状态 |
| `submit_feedback` | 获取测试版反馈调查问卷链接 |
| `report_session_summary` | 与 Comfy 团队分享匿名化的会话摘要——**仅在你明确同意的情况下**；代理必须先征求你的同意，且摘要中不包含任何提示词、文件路径或个人信息 |

## 积分与消费

探索功能免费：`search_templates`、`search_models` 和 `search_nodes` 只需一个 [Comfy 账户](https://cloud.comfy.org) 即可使用。运行生成会消耗 Comfy Cloud 积分，且需要订阅或积分余额。

实际花费金钱的工具是**消费门槛限定**的。当生成使用付费合作伙伴模型（`partner_generate`）或包含付费 API 节点的工作流/模板（`run_template`、`run_saved_workflow`、`submit_batch`）时，服务器会阻止执行，直到你的智能体确认你同意花费积分——因此，在这些运行之前，智能体会先与你确认。免费的本地工作流永远不会提示。

你可以随时向智能体询问你的余额（“我还有多少积分？”）——它会通过 `get_billing_status` 回答。

## 输出与下载

MCP 服务器在云端运行，**绝不会将文件写入你的机器**。当生成完成时，你的代理会调用 `get_output`，它会返回：

1. 一个**临时签名下载 URL**（在短时间内有效）。
2. 一条可直接运行的 **shell 指令**（macOS 和 Linux 上为 `curl`，Windows 上为 `curl.exe`）。

你的代理应在终端中运行该指令。指令中包含了目标路径和文件名。

<Warning>
  请**逐字运行**返回的指令。请勿重新编码或编辑签名的 URL。签名位于查询字符串中，一旦修改 URL 就会失效。
</Warning>

如果你的 MCP 客户端无法运行 shell 指令（某些仅 GUI 的环境），请复制该指令并自行在终端中运行。

## 已知限制

Comfy Cloud MCP 为早期版本。以下为已知的限制，团队正在努力解决：

**工作流**

- **编辑器格式到可执行格式的转换尚不完善。** 已保存的工作流可以通过 `run_saved_workflow` 按文件名运行（服务器会自动转换），但某些节点配置的转换仍不够完美。
- **通过 `submit_workflow` 生成的资产可能未嵌入工作流元数据。** 在 ComfyUI 中打开时，可能无法重新打开其原始工作流。
- **工作流构建依赖于智能体的准确性。** 复杂的多节点工作流可能需要重试或优化。

**文件处理**

- **输出需要经过 Shell 下载步骤。** 请参阅[输出与下载](#outputs-and-downloads)。
- **上传大小限制**可能因 MCP 客户端而异。某些客户端对文件上传施加了自己的限制。

**身份验证**

- **OAuth 或 API 密钥。** Claude Code 和 Claude Desktop 使用一次性浏览器 OAuth 流程。无头客户端可以通过 `X-API-Key` 标头传递 Comfy Cloud API 密钥作为替代。计划为无法打开浏览器的客户端提供设备码 OAuth 流程。

## 常见问题

<AccordionGroup>
  <Accordion title="支持哪些客户端？">
    任何支持远程 HTTP 传输的 MCP 兼容客户端都可以连接到 `https://cloud.comfy.org/mcp`。我们为 **Claude Code**、**Claude Desktop**、**Cursor** 和 **Codex** 提供了最优先的配置说明。**Windsurf**、**Amp**、**OpenClaw** 和其他代理使用相同的 URL，通过 OAuth 或 API 密钥连接。参见[安装 Comfy Cloud MCP](#install-comfy-cloud-mcp)。
  </Accordion>
  <Accordion title="是否需要 API 密钥？">
    对于支持 OAuth 的交互式客户端（Claude Code、Claude Desktop、Cursor、Codex 等）不需要。仅在没有浏览器的无头或 CI 环境中才需要 API 密钥。请参见[安装 Comfy Cloud MCP](#install-comfy-cloud-mcp) 下的 **其他客户端** 标签。
  </Accordion>
  <Accordion title="为什么我的代理在生成前会先询问我？">
    付费生成受消费确认保护：当一次运行会消耗 Comfy Cloud 积分时（使用合作伙伴模型，或包含付费 API 节点的工作流/模板），服务器会要求你明确同意后才执行。请参阅[积分与消费](#credits-and-spending)。
  </Accordion>
  <Accordion title="在 Claude Desktop 中能否使用斜杠指令？">
    不能。斜杠指令是 Claude Code 插件的一部分。Claude Desktop 可以连接到同一个 MCP 服务器（如果你用自然语言提问或使用提示选择器，这些工具也能正常工作），但它不支持 Claude Code 插件或斜杠指令。
  </Accordion>
  <Accordion title="我输入了 /comfy 或 /cloud 但没有反应。">
    没有 `/comfy` 或 `/cloud` 指令。指令根据连接方式显示在以下两个前缀之一下：

    - **插件（推荐）：** `/comfy-cloud:generate-image`、`/comfy-cloud:generate-video`……输入 `/comfy-cloud:` 即可看到所有指令。
    - **直接连接（无插件）：** `/mcp__comfy-cloud__generate-image`……输入 `/mcp__` 即可查看。

    无论哪种方式，你也可以直接用自然语言提问（例如“生成一张……的图像”）。MCP 工具由模型调用，无需斜杠指令。
  </Accordion>
  <Accordion title="登录时没有打开浏览器。">
    在 Claude Code 中，运行 `/mcp`，选择 **comfy-cloud**，然后选择 **Authenticate**。在 Claude Desktop 中，重新打开“自定义 → 连接器”中的连接器，并触发登录。
  </Accordion>
  <Accordion title="如何在本地使用代理运行 Comfy？">
    Comfy Cloud MCP 是我们的托管服务，但自 ComfyUI 开源以来，本地代理工作流就已存在。请参阅 Agent Tools 概述中的[关于本地 ComfyUI？](/zh/agent-tools#what-about-local-comfyui)部分，了解 Cloud 首先发布的原因、当前可用的社区 MCP 服务器，以及另一个选择——[Comfy CLI](/zh/agent-tools/comfy-cli)。
  </Accordion>
</AccordionGroup>
