[comfy-cli](https://github.com/Comfy-Org/comfy-cli) 是 Comfy 的命令行工具。与 [MCP 服务器](/zh/agent-tools) 一起，它为智能体和开发者提供了另一种从 UI 外部驱动 Comfy 的方式。

当你需要**终端或脚本访问**时使用它：批处理任务、CI 流水线、快速的一次性生成，或与 Claude Code 技能配合使用。`comfy generate` 子命令（测试版）调用与在工作流中连接的相同的托管[合作节点](/zh/tutorials/partner-nodes/overview)，但作为一次性 CLI 调用。

<Info>
  **`comfy generate` 处于测试版。** 标志名称、模型别名和输出格式可能发生变化。底层合作端点稳定。请在 [comfy-cli GitHub 仓库](https://github.com/Comfy-Org/comfy-cli/issues) 提交反馈。
</Info>

## 如何与 MCP 配合使用

| | Comfy Cloud MCP | Comfy CLI（`comfy generate`） |
|---|---|---|
| **接口** | 与 MCP 客户端（Claude Code、Claude Desktop 等）对话 | 终端命令和脚本 |
| **适用场景** | 交互式智能体工作流、模板搜索、完整的 Cloud 工作流 | 自动化、批量运行、CI、Shell 管道 |
| **认证** | OAuth（客户端）或 API 密钥（无头模式） | [Comfy API 密钥](/zh/development/api-development/getting-an-api-key) |
| **输出** | 返回至聊天会话 | 保存到磁盘（`--download`）或 JSON（`--json`） |

有关 MCP 设置、安装、工具列表和认证方法，请参阅 [Comfy Cloud MCP](/zh/agent-tools/cloud)。

要教会智能体如何从终端驱动 Comfy，请浏览或贡献 [Comfy Skills](https://github.com/Comfy-Org/comfy-skills/) 仓库中的技能。

## `comfy generate` 的先决条件

1. [创建 Comfy API 密钥](/zh/development/api-development/getting-an-api-key)
2. 为你的账户[添加积分](/zh/interface/credits)
3. 可选：[浏览合作节点及每次调用定价](/zh/tutorials/partner-nodes/pricing)

一次性设置密钥：

```bash
export COMFY_API_KEY=comfyui-...   # 或在每次调用时传递 --api-key
```

## 第一次生成

```bash
comfy generate flux-pro \
    --prompt "a cat on the moon, cinematic lighting" \
    --width 1024 --height 1024 \
    --download cat.png
```

CLI 会上传本地文件输入、提交任务、轮询直至准备就绪，并将结果保存到 `cat.png`。

## 可用模型

`comfy generate` 涵盖图像和视频合作模型。查看你的 CLI 版本中已安装的模型：

```bash
comfy generate list                            # 全部模型
comfy generate list --category text-to-video   # 按类别筛选
comfy generate list --partner kling            # 按合作伙伴筛选
comfy generate schema flux-kontext             # 单个模型的参数
```

常用别名包括：

| 类别 | 示例 |
|---|---|
| **图像** | `flux-pro`、`flux-kontext`、`nano-banana`（Gemini Flash 图像）、`grok`、`grok-edit`、`ideogram-edit`、`dalle`、`recraft`、`stability` |
| **视频** | `seedance`、`kling`、`luma`、`runway`、`pika`、`vidu`、`hailuo`、`moonvalley`、`grok-video` |

运行 `comfy generate list` 获取你环境中的完整列表。别名和标志在测试版期间可能发生变化。

## 使用示例

### 文生图

```bash
comfy generate nano-banana \
    --prompt "a watercolor of a sleeping fox" \
    --download fox.png
```

### 图像编辑（参考图像）

直接传递本地路径。CLI 通过 Comfy 存储上传（或根据合作伙伴要求进行 base64 编码内联）：

```bash
comfy generate nano-banana \
    --prompt "add a top hat" \
    --image ./cat.png \
    --download edited.png

comfy generate flux-kontext \
    --prompt "add a top hat and a monocle" \
    --input_image ./photo.jpg \
    --download out.png
```

上传一次并重用签名 URL：

```bash
comfy generate upload ./photo.jpg
# 会打印一个用于 --input_image 的签名 URL
```

<Note>
  上传的参考资产在 **24 小时** 后自动删除。对于长时间运行的流水线，请在每次任务前重新上传。参阅 [CLI 参考](/zh/comfy-cli/reference#upload)。
</Note>

### 文生视频

视频任务是异步的。默认情况下 CLI 会阻塞并轮询直至就绪：

```bash
comfy generate seedance \
    --prompt "a hummingbird hovering over a flower" \
    --resolution 1080p --duration 5 \
    --download hummingbird.mp4

comfy generate kling \
    --prompt "a paper boat drifting on a river at dusk" \
    --duration 5 \
    --download boat.mp4
```

传递 `--async` 可立即返回任务 ID，然后恢复：

```bash
comfy generate luma --prompt "neon koi swimming through clouds" --aspect_ratio 16:9 --async
comfy generate resume luma <job_id> --download out.mp4
```

### 使用 JSON 编写脚本

对于流水线，`--json` 输出原始 API 响应：

```bash
comfy generate dalle --prompt "a watercolor whale" --json | jq '.data[0].url'
```
